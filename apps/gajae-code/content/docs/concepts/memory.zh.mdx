title: 记忆

---

记忆让 Gajae-Code 无需手动操作即可跨会话延续知识。启用后，智能体从过往会话中**保留（retain）**持久信号，并在每个新会话中**召回（recall）**一份项目范围的紧凑摘要——技术决策、反复出现的工作流和陷阱，否则每次都要重新摸索。

记忆**默认禁用**。在 `/settings` 或 `config.yml` 中启用：

```yaml
memories:
  enabled: true
```

## 记忆在循环中的位置

记忆是会话的首尾环节，而非驱动者：

- **召回（会话开始）**——若当前项目存在记忆摘要，会作为 **Memory Guidance** 块注入系统提示。
- **保留（后台）**——启动时（或按需）运行流水线，从过往会话提取知识并整合到磁盘，供下次使用。

记忆被视为**启发式上下文**，而非权威状态。智能体被要求：

- 把记忆用于流程与既往决策，而非当前仓库状态的真相来源；
- 当记忆改变计划时引用记忆工件路径，并在行动前与当前仓库证据配对；
- 当记忆与仓库状态或用户指令冲突时优先后者，把冲突的记忆视为陈旧。

## 读取记忆工件

智能体可用 `read` 工具与 `memory://` URL 直接读取记忆文件：

| URL                                    | 内容                |
| -------------------------------------- | ------------------- |
| `memory://root`                        | 启动时注入的紧凑摘要 |
| `memory://root/MEMORY.md`              | 完整长期记忆文档     |
| `memory://root/skills/<name>/SKILL.md` | 生成的 skill 手册    |

## `/memory` 斜杠命令

| 子命令                | 效果                          |
| --------------------- | ----------------------------- |
| `view`                | 显示当前记忆注入载荷           |
| `clear` / `reset`     | 删除所有记忆数据与生成工件      |
| `enqueue` / `rebuild` | 强制下次启动时运行整合         |

## 保留的工作方式

保留流水线在启动时或经斜杠命令分两阶段运行。

<Steps>
<Step>
**逐会话提取。** 对每个自上次处理后发生变化的过往会话，模型读取会话历史并提取持久信号——技术决策、约束、已解决的失败、反复工作流。过新、过旧或当前活动的会话被跳过。每次提取产出一个原始记忆块与简短概要。
</Step>
<Step>
**整合。** 第二轮模型通读所有逐会话提取，并向磁盘写出三份输出：`MEMORY.md`（精选长期记忆文档）、`memory_summary.md`（会话开始时注入的紧凑文本）、`skills/`（可复用的流程手册，各自独立子目录）。该阶段使用 lease 防止并发进程重复运行，并自动清理上次运行的陈旧 skill 目录。
</Step>
</Steps>

所有输出在写盘前都会做密钥扫描。

## 运行记忆的模型

记忆搭载[模型角色系统](/zh/docs/concepts/models-and-providers)：

| 阶段           | 角色      | 目的           |
| -------------- | --------- | -------------- |
| 提取（阶段 1） | `default` | 逐会话知识提取 |
| 整合（阶段 2） | `smol`    | 跨会话综合     |

若请求的角色未配置，记忆模型解析回退到 `default`、活动会话模型、registry 中第一个模型。

## 配置

| 设置                                   | 默认值  | 说明                                  |
| -------------------------------------- | ------- | ------------------------------------- |
| `memories.enabled`                     | `false` | 主开关                                |
| `memories.maxRolloutAgeDays`           | `30`    | 超过此天数的会话不处理                 |
| `memories.minRolloutIdleHours`         | `12`    | 比此更近活动的会话被跳过               |
| `memories.maxRolloutsPerStartup`       | `64`    | 单次启动处理会话数上限                 |
| `memories.summaryInjectionTokenLimit`  | `5000`  | 注入系统提示的摘要最大 token 数        |

并发、lease 时长、token 预算等更多调优项在 config 中供高级使用。完整面见[配置](/zh/docs/reference/configuration)。
