description: Gajae-Code가 지난 세션에서 지속 가능한 지식을 자동으로 추출하고, 새 세션마다 압축 요약을 불러오는 방식.
---

메모리는 Gajae-Code가 수작업 없이 세션을 넘나들며 지식을 이어가게 해 줍니다. 활성화하면 에이전트는 지난 세션에서 지속 가능한 신호를 **보존(retain)** 하고, 프로젝트 범위의 압축 요약을 새 세션마다 **불러옵니다(recall)** — 기술 결정, 반복 워크플로, 함정처럼 매번 다시 알아내야 할 것들을 말이죠.

메모리는 **기본 비활성화** 입니다. `/settings`나 `config.yml`에서 켭니다:

```yaml
memories:
  enabled: true
```

## 메모리가 루프에서 차지하는 자리

메모리는 세션을 주도하기보다 세션의 앞뒤를 감쌉니다:

- **불러오기(세션 시작)** — 현재 프로젝트의 메모리 요약이 있으면 시스템 프롬프트에 **Memory Guidance** 블록으로 주입됩니다.
- **보존(백그라운드)** — 시작 시(또는 수동으로) 파이프라인이 돌며 지난 세션에서 지식을 추출하고, 다음 실행을 위해 디스크에 통합합니다.

메모리는 권위 있는 상태가 아니라 **휴리스틱 컨텍스트** 로 취급됩니다. 에이전트는 다음을 따르도록 지시받습니다:

- 메모리는 현재 repo 상태의 진실 출처가 아니라 프로세스와 과거 결정에 사용,
- 메모리가 계획을 바꿀 때는 메모리 아티팩트 경로를 인용하고, 행동 전에 현재 repo 증거와 짝지음,
- 메모리와 충돌할 때는 repo 상태와 사용자 지시를 우선하고, 충돌하는 메모리는 낡은 것으로 간주.

## 메모리 아티팩트 읽기

에이전트는 `read` 도구와 `memory://` URL로 메모리 파일을 직접 읽을 수 있습니다:

| URL                                    | 내용                          |
| -------------------------------------- | ----------------------------- |
| `memory://root`                        | 시작 시 주입되는 압축 요약     |
| `memory://root/MEMORY.md`              | 장기 메모리 전체 문서          |
| `memory://root/skills/<name>/SKILL.md` | 생성된 스킬 플레이북           |

## `/memory` 슬래시 명령

| 하위 명령              | 효과                                        |
| --------------------- | ------------------------------------------- |
| `view`                | 현재 메모리 주입 페이로드 표시               |
| `clear` / `reset`     | 모든 메모리 데이터와 생성 아티팩트 삭제      |
| `enqueue` / `rebuild` | 다음 시작 시 통합을 강제 실행                |

## 보존이 동작하는 방식

보존 파이프라인은 시작 시 또는 슬래시 명령으로 두 단계로 실행됩니다.

<Steps>
<Step>
**세션별 추출.** 마지막 처리 이후 변경된 각 지난 세션에 대해, 모델이 세션 히스토리를 읽고 지속 가능한 신호 — 기술 결정, 제약, 해결된 실패, 반복 워크플로 — 를 추출합니다. 너무 최근·너무 오래된·현재 활성 세션은 건너뜁니다. 각 추출은 원시 메모리 블록과 짧은 개요를 만듭니다.
</Step>
<Step>
**통합.** 두 번째 모델 패스가 모든 세션별 추출을 읽고 세 가지 출력을 디스크에 씁니다:

- `MEMORY.md` — 정제된 장기 메모리 문서,
- `memory_summary.md` — 세션 시작 시 주입되는 압축 텍스트,
- `skills/` — 재사용 가능한 절차 플레이북, 각각 자체 하위 디렉터리에.

이 단계는 동시 프로세스가 중복 실행하지 않도록 lease를 쓰며, 이전 실행의 낡은 스킬 디렉터리는 자동 정리됩니다.
</Step>
</Steps>

모든 출력은 디스크에 쓰기 전에 비밀 값(secret) 스캔을 거칩니다.

## 메모리를 실행하는 모델

메모리는 [모델 역할 시스템](/ko/docs/concepts/models-and-providers)에 편승합니다:

| 단계                | 역할      | 목적                  |
| ------------------- | --------- | --------------------- |
| 추출(1단계)         | `default` | 세션별 지식 추출       |
| 통합(2단계)         | `smol`    | 세션 간 종합           |

요청한 역할이 설정되지 않았으면 메모리 모델 해석은 `default`, 그다음 활성 세션 모델, 그다음 레지스트리의 첫 모델로 폴백합니다.

## 설정

| 설정                                   | 기본값  | 설명                                              |
| -------------------------------------- | ------- | ------------------------------------------------- |
| `memories.enabled`                     | `false` | 메인 스위치                                        |
| `memories.maxRolloutAgeDays`           | `30`    | 이보다 오래된 세션은 처리하지 않음                  |
| `memories.minRolloutIdleHours`         | `12`    | 이보다 최근에 활성이던 세션은 건너뜀                |
| `memories.maxRolloutsPerStartup`       | `64`    | 한 번 시작에서 처리하는 세션 수 상한               |
| `memories.summaryInjectionTokenLimit`  | `5000`  | 시스템 프롬프트에 주입되는 요약의 최대 토큰 수      |

동시성, lease 지속 시간, 토큰 예산 등 추가 튜닝 노브가 고급 용도로 config에 있습니다. 전체 표면은 [설정](/ko/docs/reference/configuration)을 보세요.
