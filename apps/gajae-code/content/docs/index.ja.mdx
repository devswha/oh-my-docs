**red-claw コーディングエージェント harness (`gjc`)。** 明確なインタビュー、堅牢な計画、tmux ネイティブ実行、永続的な検証。

> 注意：Gajae-Code は実験的な初期ベータプロジェクトです。粗い部分があるため、重要な作業に使う前に出力を確認してください。

## 概要

Gajae-Code は npm レジストリに `gajae-code` として公開され、このパッケージが `gjc` バイナリをインストールします。公開エージェントの表面を意図的に小さく保ちつつ、その周囲のランタイムを信頼できるものにします。セッション状態、worktree 隔離、tmux オーケストレーション、モデルルーティング、ツール実行、永続的な証拠が、作業に必要なときに揃っています。

肥大したデフォルト skill 群ではなく、1 つの方法をより良くすることで harness を改善します。

## 1 つの有用なループ

```text
deep-interview -> ralplan -> ultragoal
                         └─ 並列 tmux ワーカーが役立つ場合は team 実行を選択
```

- **`deep-interview`** — 計画やコード変更の前に意図を明確にします。
- **`ralplan`** — 変更前に方針を立て批評します。
- **`$ultragoal`** — 実装・改訂・検証・証拠のまとめまで進めます。
- **`$team`** — 並列ワーカーが有益な場合のみ加える任意の実行モードで、必須の引き継ぎ手順ではありません。

この順序は隠れた自動化ではなく操作の流れを説明します。エージェントは何を変え、何を改訂し、どの検査を実行し、どの証拠が結果を裏付けるかを報告します。

## インストール

```bash
bun install -g gajae-code
```

このパッケージは `gjc` バイナリをインストールします。scoped パッケージ `@gajae-code/coding-agent` でも入手できます。その後、推奨の tmux ベース環境で起動します：

```bash
gjc --tmux
```

## ワークフロー

4 つのデフォルトワークフロースキル：`deep-interview`、`ralplan`、`team`、`ultragoal`。

4 つの組み込みロールエージェント：`executor`、`architect`、`planner`、`critic`。
