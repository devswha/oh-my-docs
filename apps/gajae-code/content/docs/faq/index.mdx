## Install and Environment

  </details>
  <details className="group px-4 py-3">

## First Use

<section className="not-prose my-6 divide-y divide-fd-border rounded-lg border border-fd-border bg-fd-card">
  <details className="group px-4 py-3">
    <summary className="flex cursor-pointer list-none items-center justify-between gap-4 font-medium text-fd-foreground [&::-webkit-details-marker]:hidden"><span>Which flow should I learn first?</span><span className="text-fd-muted-foreground transition-transform group-open:rotate-45">+</span></summary>
    <p className="mt-3 text-sm leading-7 text-fd-muted-foreground">Learn the loop: <code>deep-interview</code> for clarity, <code>ralplan</code> for the reviewed plan, and <code>ultragoal</code> for durable execution and evidence.</p>
  </details>
  <details className="group px-4 py-3">
    <summary className="flex cursor-pointer list-none items-center justify-between gap-4 font-medium text-fd-foreground [&::-webkit-details-marker]:hidden"><span>When should I use rlm?</span><span className="text-fd-muted-foreground transition-transform group-open:rotate-45">+</span></summary>
    <p className="mt-3 text-sm leading-7 text-fd-muted-foreground">Use <code>gjc rlm</code> for exploratory research instead of code mutation. It writes notebook and report artifacts under <code>.gjc/_session-&lt;gjcSessionId&gt;/rlm/&lt;rlmSessionId&gt;/</code>.</p>
  </details>
  <details className="group px-4 py-3">
    <summary className="flex cursor-pointer list-none items-center justify-between gap-4 font-medium text-fd-foreground [&::-webkit-details-marker]:hidden"><span>Is team required?</span><span className="text-fd-muted-foreground transition-transform group-open:rotate-45">+</span></summary>
    <p className="mt-3 text-sm leading-7 text-fd-muted-foreground">No. <code>team</code> is optional and useful only when parallel tmux workers genuinely help. Single-threaded work can stay in the normal loop.</p>
  </details>
</section>

## Running and Limits
