**red-claw 코딩 에이전트 하네스 (`gjc`).** 또렷한 인터뷰, 견고한 계획, tmux 네이티브 실행, 지속 가능한 검증.

> 참고: Gajae-Code는 실험적인 베타 초기 프로젝트입니다. 거친 부분이 있을 수 있으니 중요한 작업에 쓰기 전에 결과를 확인하세요.

## 무엇인가요

Gajae-Code는 npm 레지스트리에 `gajae-code`로 배포됩니다. 이 패키지가 `gjc` 바이너리를 설치합니다.

공개 에이전트 표면은 작게 유지하면서 그 주변 런타임을 단단하게 만듭니다. 작업에 필요할 때 다음이 모두 준비되어 있습니다:

- 세션 상태와 worktree 격리
- tmux 오케스트레이션과 모델 라우팅
- 도구 실행과 지속 가능한 증거

거대한 기본 스킬 더미 대신, 하나의 방법을 더 잘 만드는 방향으로 하네스를 개선합니다.

## 하나의 쓸모 있는 루프

```text
deep-interview -> ralplan -> ultragoal
                         └─ 병렬 tmux 워커가 도움이 될 때만 team 실행
```

- **`deep-interview`** — 계획이나 코드 변경 전에 의도를 명확히 합니다.
- **`ralplan`** — 변경 전에 접근을 세우고 비평합니다.
- **`$ultragoal`** — 구현·수정·검증·증거 요약까지 작업을 끌고 갑니다.
- **`$team`** — 병렬 워커가 이득일 때만 더하는 선택적 실행 모드입니다. 필수 핸드오프 단계가 아닙니다.

이 순서는 숨은 자동화가 아니라 운영자 흐름을 설명합니다. 에이전트는 다음을 보고합니다:

- 무엇을 바꿨는지
- 발견 후 무엇을 수정했는지
- 어떤 검사를 돌렸는지
- 어떤 증거가 결과를 뒷받침하는지

## 설치

```bash
bun install -g gajae-code
```

이 패키지는 `gjc` 바이너리를 설치합니다. 스코프 패키지 `@gajae-code/coding-agent`로도 받을 수 있습니다.

권장하는 tmux 기반 환경으로 시작합니다:

```bash
gjc --tmux
```

## 워크플로우 표면

기본 워크플로우 스킬 네 가지:

| 스킬             | 하는 일                                          |
| ---------------- | ------------------------------------------------ |
| `deep-interview` | 계획·코드 변경 전에 모호함을 제거합니다.         |
| `ralplan`        | 변경 전에 계획을 세우고 비평합니다.              |
| `team`           | 필요할 때 tmux 기반 병렬 실행을 조율합니다.      |
| `ultragoal`      | 구현·수정·검증·증거 요약까지 목표를 추적합니다.  |

번들된 역할 에이전트 네 가지:

| 에이전트    | 하는 일                                  |
| ----------- | ---------------------------------------- |
| `executor`  | 범위가 정해진 구현·수정·리팩터링.        |
| `architect` | 읽기 전용 아키텍처·코드 리뷰 평가.       |
| `planner`   | 읽기 전용 순서 설계·수용 기준 정의.      |
| `critic`    | 읽기 전용 계획 비평·실행 가능성 검토.    |
