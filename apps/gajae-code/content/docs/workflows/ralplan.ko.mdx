`ralplan`은 [작동 루프](/ko/docs/workflows)의 중간 단계입니다. 명확해진 요청에서 계획을 세우고, 파일이 변경되기 전에 그 계획을 비평합니다. "문제를 이해했다"와 "파일을 바꾸기 시작한다" 사이의 관문입니다.

## 하는 일

`ralplan`은 한 스킬에서 두 가지를 합니다:

1. **계획 수립** — 명확해진 요청(이상적으로는 [`deep-interview`](/ko/docs/workflows/deep-interview)의 출력)을 구체적이고 검토 가능한 계획으로 바꿉니다: 순서, 만들 변경, 대조할 수용 기준.
2. **계획 비평** — 변경이 시작되기 전에 그 계획의 빈틈, 위험한 가정, 실행 가능성을 검토합니다.

핵심은 비평입니다. 약한 계획을 여기서 잡는 게 구현 중간에 발견하는 것보다 훨씬 저렴합니다.

## 사용법

GJC 세션 안에서 `/skill:<name>`으로 실행합니다:

```text
/skill:ralplan turn the clarified bug report into a reviewed fix plan
```

명확해진 문제 정의를 넘기면, 승인하고 실행으로 넘길 수 있는 검토된 계획을 돌려줍니다.

## 변경 전의 관문

```text
deep-interview -> ralplan -> ultragoal
```

`ralplan`이 수용할 만한 계획을 내놓기 전까지는 어떤 하류 단계도 파일을 변경해서는 안 됩니다. 승인된 계획은 [`ultragoal`](/ko/docs/workflows/ultragoal)이 구현·수정·검증으로 끌고 가는 brief가 됩니다:

```bash
gjc ultragoal create-goals --brief-file <approved-plan>
```

## 관여하는 에이전트

`ralplan`은 읽기 전용 역할 [에이전트](/ko/docs/agents)를 활용해 계획·비평을 실행과 분리합니다:

- [`planner`](/ko/docs/agents/planner) — 읽기 전용 시퀀싱과 수용 기준.
- [`critic`](/ko/docs/agents/critic) — 읽기 전용 계획 비평과 실행 가능성 검토.
- [`architect`](/ko/docs/agents/architect) — 계획이 구조를 건드릴 때 읽기 전용 아키텍처·코드 리뷰 평가.
