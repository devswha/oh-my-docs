---

Gajae-Code는 기본 워크플로우 스킬 네 가지를 제공합니다. GJC 세션 안에서 `/skill:<name>`으로 실행합니다.

| 스킬             | 하는 일                                          |
| ---------------- | ------------------------------------------------ |
| `deep-interview` | 계획·코드 변경 전에 모호함을 제거합니다.         |
| `ralplan`        | 변경 전에 계획을 세우고 비평합니다.              |
| `team`           | 필요할 때 tmux 기반 병렬 실행을 조율합니다.      |
| `ultragoal`      | 구현·수정·검증·증거까지 목표를 추적합니다.       |

## deep-interview

계획이나 변경 전에 의도를 명확히 합니다 — 진짜 버그·기능, 영향 범위, 비목표, 수용 기준. 요청이 모호하거나 영향 범위가 불분명할 때 여기서 시작하세요.

```text
/skill:deep-interview clarify the bug, affected behavior, non-goals, and acceptance checks
```

## ralplan

명확해진 요청을 검토된 계획으로 바꾸고, 변경 전에 그 계획을 비평합니다. "문제를 이해했다"와 "파일을 바꾸기 시작한다" 사이의 관문입니다.

```text
/skill:ralplan turn the clarified bug report into a reviewed fix plan
```

## ultragoal

승인된 계획을 구현·발견 후 수정·검증·최종 증거 요약까지 끌고 갑니다. CLI로 구동합니다:

```bash
gjc ultragoal create-goals --brief-file <approved-plan>
gjc ultragoal complete-goals
```

## team (선택)

tmux 기반 병렬 워커를 조율합니다 — 작업을 여러 pane으로 나눠서 정말 이득일 때만 쓰세요. 필수 단계가 아니라 실행 모드입니다.

```bash
gjc team 2:executor "split implementation and verification for this bug fix"
```

## 전체 흐름

구체적인 버그 수정 패스가 루프를 엮습니다:

```text
/skill:deep-interview clarify the bug, affected behavior, non-goals, and acceptance checks
/skill:ralplan turn the clarified bug report into a reviewed fix plan
gjc ultragoal create-goals --brief-file <approved-plan>
# 병렬 작업일 때만 선택:
gjc team 2:executor "split implementation and verification for this bug fix"
gjc ultragoal complete-goals
```

이는 숨은 자동화가 아니라 운영자 시퀀스를 설명합니다. 에이전트는 무엇을 바꿨고, 발견 후 무엇을 수정했고, 어떤 검사를 돌렸고, 어떤 증거가 수정을 뒷받침하는지 보고합니다.
