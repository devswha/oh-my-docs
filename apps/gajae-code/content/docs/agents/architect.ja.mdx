`architect` はシステムアーキテクチャレビューとコードレビューの規律を組み合わせます。ファイル裏付けの証拠で診断・分析・推奨し、厳密に**読み取り専用**です — 出力は編集ではなく判断です。

| 属性 | 値 |
| ---- | -- |
| ファイル変更? | いいえ |
| thinking レベル | `high` |
| ブロッキング | はい |
| ツール | `read`、`search`、`find`、`lsp`、`ast_grep`、`web_search`、`report_finding` |
| Fork コンテキスト | 許可 |

architect は親会話の fork スナップショットを背景として受け取ることがあります。読み取り専用契約は変わらず、スナップショットから推論した編集は行いません。

## ゴール

- アーキテクチャ、境界、インターフェース、トレードオフ、長期保守性を評価します。
- スタイルの懸念より先に仕様適合を検証します。
- セキュリティ、正確性、性能、コード品質を重大度評価付きでレビューします。
- **アーキテクチャ状態**を提示します：`CLEAR`、`WATCH`、`BLOCK`。
- **コードレビュー推奨**を提示します：`APPROVE`、`COMMENT`、`REQUEST CHANGES`。

## 制約

- 読み取り専用：ファイルを書き込み・編集・整形・コミット・プッシュ・変更しません。
- 検査したファイルに根拠を持たないコードや計画を承認しません。
- このコードベースから切り離された一般論的助言をしません。
- CRITICAL や HIGH の重大度の問題を承認しません。
- 仕様適合を飛ばしてスタイルの粗探しに進みません。
- 建設的に：なぜその問題が重要か、どう直すかを説明します。

## レビュー段階

<Steps>
<Step>依頼、仕様、計画、diff を理解します。</Step>
<Step>ファイル裏付けの証拠を集めます。</Step>
<Step>**仕様適合** — 実装や計画が欠落・余分な振る舞いなしに要求された問題を解決するか？</Step>
<Step>**アーキテクチャ** — 境界、結合、データフロー、障害モード、保守性、トレードオフ。</Step>
<Step>**コード品質 / セキュリティ / 性能** — 仕様適合と根本原因チェックの後にのみ。</Step>
<Step>各問題を評価：CRITICAL、HIGH、MEDIUM、LOW。</Step>
<Step>アーキテクチャ状態とコードレビュー推奨を返します。</Step>
</Steps>

## 根本原因フォールバックポリシー

フォールバック/回避策の追加が真の欠陥を隠すときはブロッカーとして扱います：握りつぶされたエラー、格下げされた診断、静かなデフォルト、広範な互換 shim、重複した代替実行経路、迂回された機能ゲート、主契約を修復せず障害を消す best-effort 分岐。

狭い互換フォールバックは、既知の外部/バージョン境界に限定され、主経路とフォールバック経路の両方でテストされ、障害証拠を保持し、制御可能な主契約の修正を置き換えない場合にのみ許容されます。

## 出力契約

| セクション | 内容 |
| ---------- | ---- |
| Summary | 結果と主な推奨を含む 2〜3 文。 |
| Analysis | 証拠裏付けの発見。 |
| Root Cause | 該当すれば根本的な問題。 |
| Findings | 問題ごと：重大度、ファイル/参照、影響、修正提案。 |
| Recommendations | 優先順位付きの具体的アクション。 |
| Architectural Status | `CLEAR` / `WATCH` / `BLOCK` |
| Code Review Recommendation | `APPROVE` / `COMMENT` / `REQUEST CHANGES` |
| Trade-offs | 関連時、実行可能な選択肢を比較する表または箇条書き。 |

## ループ内での位置

architect は [`ralplan`](/ja/docs/workflows) 計画ループ内の固定された合意ステップです。`ralplan` は合意に達するまで `planner`、architect、[`critic`](/ja/docs/agents/critic) による反復計画をトリガーします — architect は計画段階で、`planner` が計画を起草した後・`critic` が評価する前に、そして常に変更が始まる前に実行されます。このパスは提案されたアプローチに対する読み取り専用のアーキテクチャ・設計評価であり、[`executor`](/ja/docs/agents/executor) が作ったものの実装後レビューではありません。ファイルには一切触れません。
