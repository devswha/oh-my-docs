Gajae-Code は境界が明確な 4 つのロールエージェントを同梱します。2 つは書き込み、2 つは読み取り専用です。

| エージェント | 役割                                   | 変更? |
| ----------- | -------------------------------------- | ----- |
| `executor`  | 範囲の定まった実装・修正・リファクタ。 | はい  |
| `architect` | 読み取り専用のアーキテクチャ・レビュー。| いいえ |
| `planner`   | 読み取り専用の順序付け・受け入れ基準。 | いいえ |
| `critic`    | 読み取り専用の計画批評・実行可能性審査。| いいえ |

## executor

範囲の定まった実装を行います：コードを書き、修正を適用し、リファクタを実行します。ファイルを変更する唯一のロールで、計画段階で設定された範囲内で作業します。

## architect

アーキテクチャとコードレビューのための読み取り専用の評価ロールです。ファイルに触れずに設計や既存コードベースを評価します — 出力は編集ではなく判断です。

## planner

作業を順序付け、受け入れ基準を定義する読み取り専用ロールです。意図を明示的な「完了」条件を持つ順序付き計画に変え、それを `ralplan` が批評します。

## critic

変更前に計画の妥当性と実行可能性を批評する読み取り専用ロールです。ループ内の懐疑者として、抜け、リスク、書かれたままでは実行されない手順を探します。

## ループ内での位置

ロールエージェントは[ワークフローループ](/ja/docs/workflows)に対応します：planner と critic は `deep-interview` / `ralplan` で計画を形成し検証し、executor は `ultragoal` の下で実行し、architect は設計や変更に独立した評価が必要なときに読み取り専用のレビューを提供します。これらのロールの並列実行は `team` で調整できます。
