Gajae-Code 内置四个边界清晰的角色智能体。两个写、两个只读。

| 智能体      | 角色                                   | 修改? |
| ----------- | -------------------------------------- | ----- |
| `executor`  | 有边界的实现、修复与重构。             | 是    |
| `architect` | 只读的架构与代码评审。                 | 否    |
| `planner`   | 只读的排序与验收标准。                 | 否    |
| `critic`    | 只读的计划评审与可执行性审查。         | 否    |

## executor

完成有边界的实现工作：写代码、应用修复、执行重构。这是唯一改动文件的角色，并在规划阶段设定的范围内工作，而非无限开放。

## architect

用于架构与代码评审的只读评估角色。不触碰文件即可评估设计或现有代码库——产出是判断，而非编辑。

## planner

排序工作并定义验收标准的只读角色。它把意图转为带明确"完成"条件的有序计划，再由 `ralplan` 评审。

## critic

在任何变更前评审计划的合理性与可执行性的只读角色。它是循环中的怀疑者，寻找漏洞、风险和按字面无法执行的步骤。

## 在循环中的位置

角色智能体对应[工作流循环](/zh/docs/workflows)：planner 与 critic 在 `deep-interview` / `ralplan` 中塑造并压力测试计划；executor 在 `ultragoal` 下执行；architect 在设计或变更需要独立评估时提供只读评审。这些角色的并行运行可用 `team` 协调。
