## 共通の取り決め

- **内部 URL。** 内容ツール（`read`、`search`、`ast-grep`）はルーターがバッキングリソースに解決できる場合 `agent://`、`artifact://`、`memory://`、`gjc://`、`rule://`、`issue://`、`pr://` を受け付けます。内部 URL に glob は不可。
- **ハッシュラインアンカー。** `read`、`search`、`ast-grep` は `LINEhh|text` アンカー（例：`41th|def alpha():`）を出力し、`edit` は `|` 左側の `LINEhh` トークンのみ消費します。`:raw` 読み取りと不変リソースではアンカーが抑制されます。
- **プレビューしてから resolve。** 一部のツール（特に `ast-edit`）は先にプレビューし強制 `resolve` アクションを予約します。`resolve(action: "apply", ...)` を呼ぶまで書き込みません。
- **上限と切り詰め。** 出力はバイト/行で上限が掛かり（`search` は先頭 `100` 件、`ast-grep` はデフォルト `50` 件）、結果に上限・切り詰めの到達が報告されます。

## 設定・資格情報が必要なツール

一部のツールはバックエンド設定後に動作します。有効化するキー・環境変数・資格情報は[リファレンス](/ja/docs/reference)を参照。

- `web_search` と `browser` は provider 設定に依存。
- `github` は `gh` CLI とそのキャッシュを使用。
- `lsp` は `lsp.enabled` でゲートされ、プロジェクトごとに言語サーバーを探索/設定。
- `search_tool_bm25` は `tools.discoveryMode !== "off"` のときのみ出現。
- アクティブな `edit` variant は `GJC_EDIT_VARIANT`、`ast-edit` のファイル範囲は `GJC_MAX_AST_FILES` で決まる。
