## 通用约定

- **内部 URL。** 内容工具（`read`、`search`、`ast-grep`）在路由器能解析到后端资源时接受 `agent://`、`artifact://`、`memory://`、`gjc://`、`rule://`、`issue://`、`pr://`。内部 URL 不允许 glob。
- **哈希行锚点。** `read`、`search`、`ast-grep` 输出 `LINEhh|text` 锚点（如 `41th|def alpha():`）；`edit` 仅消费 `|` 左侧的 `LINEhh` 令牌。`:raw` 读取和不可变资源会抑制锚点。
- **先预览后 resolve。** 部分工具（尤其 `ast-edit`）先预览并排入强制 `resolve` 动作；在 `resolve(action: "apply", ...)` 前不写入。
- **限制与截断。** 输出按字节/行设限（`search` 显示前 `100` 个匹配，`ast-grep` 默认 `50`），结果会报告是否触达限制或截断。

## 需要配置或凭据的工具

部分工具需先配置后端。启用它们的键、环境变量与凭据见[参考](/zh/docs/reference)。

- `web_search` 与 `browser` 依赖 provider 设置。
- `github` 使用 `gh` CLI 及其缓存。
- `lsp` 受 `lsp.enabled` 限制，按项目探测/配置语言服务器。
- `search_tool_bm25` 仅在 `tools.discoveryMode !== "off"` 时出现。
- 活动 `edit` variant 由 `GJC_EDIT_VARIANT` 决定，`ast-edit` 文件范围由 `GJC_MAX_AST_FILES` 决定。
