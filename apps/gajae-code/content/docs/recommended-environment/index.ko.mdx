Gajae-Code는 터미널 우선 코딩 에이전트 하네스입니다. 설치 전에 셸에서 Bun을 실행할 수 있고, 작업 저장소가 Git 아래에 있으며, 최소 하나의 모델 provider를 인증할 수 있는지 확인하세요.

## 운영체제

| OS | 추천 | 메모 |
| --- | --- | --- |
| macOS | 추천 | 로컬 개발 환경으로 좋습니다. Apple Silicon macOS는 실험적 `computer-use` 도구의 지원 호스트이기도 합니다. |
| Linux | 추천 | tmux 기반 터미널 세션, CI에 가까운 셸, 원격 개발에 잘 맞습니다. |
| Windows | 가능하면 WSL2 | 일부 터미널 흐름은 네이티브 Windows에서도 가능하지만, tmux와 셸 동작은 보통 WSL2가 더 부드럽습니다. |

## 설치 전 확인

| 확인 | 권장 상태 |
| --- | --- |
| Bun | `PATH`에 설치되어 있어야 합니다. 문서화된 설치 명령은 `bun install -g gajae-code`입니다. |
| Git | diff, worktree, checkpoint를 검토할 수 있도록 Git 저장소에서 시작하세요. |
| tmux | 권장 실행 방식인 `gjc --tmux`나 `team`을 쓸 계획이면 설치하세요. |
| Provider 인증 | 모델을 고르기 전에 API 키, OAuth 자격 증명, provider preset 중 하나를 준비하세요. |
| 비밀 값 | 키는 셸, `~/.gjc/agent/.env`, 또는 다른 private env 파일에 두고 커밋하지 마세요. |

기본 도구부터 확인합니다.

```bash
bun --version
git --version
tmux -V
```

## 작성자의 주관적 추천

- **Git checkout에서 시작하세요.** Gajae-Code는 실제 diff를 읽고 위험한 작업을 격리할 수 있을 때 가장 유용합니다.
- **처음에는 `gjc --tmux`를 쓰세요.** tmux 경로는 team 실행과 긴 작업에 안정적인 터미널 표면을 제공합니다.
- **provider 하나를 명확히 설정하세요.** 모델 라우팅을 실험하기 전에 provider key 하나를 export하거나 preset 하나를 추가하세요.
- **Windows 중심 환경은 WSL2를 쓰세요.** tmux, 셸 경로, 프로젝트 파일 동작이 문서화된 흐름에 더 가깝습니다.

첫 실행은 [시작하기](/ko/docs/getting-started)로 이어가세요.
