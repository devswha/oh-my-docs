本節は Gajae-Code（`gjc`）のランタイムリファレンスです。`~/.gjc` 配下のファイル、環境変数の解決方法、各詳細ページへのリンクを扱います。

## 設定ファイル

Gajae-Code は `~/.gjc` から設定を読み込みます（ルートは `GJC_CONFIG_DIR` で変更）：

| パス                       | 用途                                |
| -------------------------- | ----------------------------------- |
| `~/.gjc/config.yml`        | 一般的なランタイム設定（例：リトライ予算）。|
| `~/.gjc/agent/models.yml`  | provider、モデル、ロール、ルーティング。|
| `~/.gjc/agent/.env`        | agent スコープの環境変数。          |
| `~/.gjc/.env`              | 設定ルートの環境フォールバック。    |
| `~/.env`                   | home の環境フォールバック。         |

`GJC_CODING_AGENT_DIR` は `GJC_CONFIG_DIR` とは独立に agent ディレクトリ（既定 `~/.gjc/agent`）全体を上書きできます。

## .env の解決順

多くのランタイム参照は `$env` を使い、`.env` ファイルを次の順で読み込みます — 各段は未設定のキーのみ埋めます：

<Steps>
  <Step>既存のプロセス環境（`Bun.env`）。</Step>
  <Step>プロジェクト `.env`（`$PWD/.env`）。</Step>
  <Step>agent `.env`（`~/.gjc/agent/.env`、`GJC_CONFIG_DIR` / `GJC_CODING_AGENT_DIR` を尊重）。</Step>
  <Step>設定ルート `.env`（`~/.gjc/.env`、`GJC_CONFIG_DIR` を尊重）。</Step>
  <Step>home `.env`（`~/.env`）。</Step>
</Steps>
