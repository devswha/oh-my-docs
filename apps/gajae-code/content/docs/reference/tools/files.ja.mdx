title: ファイルツール

---

ファイルファミリーは内容の読み取りと変更の適用を扱います。`read` は単一の `path` 文字列でほぼ何でも開き、`write` は作成・上書き、`edit` は複数の variant で範囲指定の行編集、`ast-edit` は構造的書き換えを行います。

## `read`

単一の `path` 文字列でファイル、ディレクトリ、アーカイブ、SQLite DB、内部リソース、画像、ドキュメント、URL を読みます。末尾のセレクタが切り出し範囲を決めます。

| 入力 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `path` | `string` | はい | ファイルパス、内部 URL、Web URL。`:50-100` や `:raw` などのセレクタで終わってよい。 |

### セレクタ

| 接尾辞 | 意味 |
| --- | --- |
| `:raw` | 原文モード。構造要約と行プレフィックスを無効化。 |
| `:N` / `:LN` | 1 始まり `N` 行目から、終端開放。 |
| `:A-B` / `:LA-LB` | 1 始まりの包含行範囲。 |
| `:A+C` / `:LA+LC` | `A` から `C` 行。 |
| `:range:raw` / `:raw:range` | 同じ行選択で原文出力。 |

行番号は 1 始まり（`:0` は無効）。URL セレクタは `:raw`、`:N`、`:A-B`、`:A+C` のみで `L` プレフィックスはなし。

### 開けるもの

- **テキストファイル** — 要約が有効で十分小さい（`<= 2 MiB`、`<= 20_000` 行）場合、省略区間を `...` に置換した構造要約と復元セレクタの案内を返す。そうでなければ行範囲ストリーム読み取り。
- **ディレクトリ** — 新しい順のツリー（深さ `2`、ディレクトリあたり最大 `12`）、サイズと経過付き。
- **アーカイブ** — `.tar`、`.tar.gz`、`.tgz`、`.zip` を `archive.ext:inner/path[:lines]` で。
- **SQLite** — `db.sqlite` で一覧、`:table` でスキーマ+サンプル、`:table:key` で 1 行、`?limit=&offset=&order=&where=` でクエリ、`?q=SELECT ...` で原文の読み取り専用 SQL。
- **ドキュメント** — `.pdf`、`.doc(x)`、`.ppt(x)`、`.xls(x)`、`.rtf`、`.epub` をテキスト変換。
- **Jupyter** — `.ipynb` を編集可能な `# %% [code] cell:N` テキストに（`:raw` を除く）。
- **画像** — インライン画像ブロック（最大 `20 MiB`）、`inspect_image.enabled` ならメタデータのみ。
- **内部 URL** — `agent://`、`artifact://`、`memory://`、`gjc://`、`rule://`、`issue://`、`pr://`。
- **Web URL** — `http://`、`https://`、`www.`；読みやすいテキストに描画、`:raw` で本文フォールバック。

<Callout type="info">
ハッシュラインモードの非 raw テキスト読み取りは `LINEhh|text`（例：`41th|def alpha():`）でプレフィックスされ、これらのアンカーを後の `edit` が消費します。単独の `/` はファイルシステムルートでなくセッション cwd に解決されます。
</Callout>

## `write`

ファイル、アーカイブエントリ、SQLite 行を作成または上書きします。

| 入力 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `path` | `string` | はい | 通常パスはファイル書き込み。`archive.ext:inner/path` はアーカイブエントリ。`db.sqlite:table` は行挿入、`db.sqlite:table:key` は更新・削除。 |
| `content` | `string` | はい | 完全置換内容。SQLite の非削除書き込みは JSON5 オブジェクト必須。行キー付きの空内容は行を削除。 |

variant：通常ファイル（上書き）、アーカイブエントリ（1 エントリ置換後にアーカイブ全体を再書き込み、親ディレクトリ作成はこの経路のみ）、SQLite 挿入（`db.sqlite:table` + JSON5 オブジェクト）、SQLite 更新/削除（`db.sqlite:table:key`、非空で更新、空/空白で削除）。

<Callout type="warn">
`write` は自動生成と検出されたファイルの上書きを拒否します。プロンプトは日常編集での `write`（代わりに `edit`）と、明示要求のない `*.md`/`README` 作成を禁じます。SQLite DB は `write` で新規作成されず、DB は既存である必要があります。
</Callout>

## `edit`

ソース編集を適用します。デフォルトモードは **ハッシュライン** パッチ言語で、単一の `input` 文字列で渡します。

| 入力 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `input` | `string` | はい | 1 つ以上の編集セクション。最初の非空行は `§PATH` 必須。任意の `*** Begin Patch` / `*** End Patch` 包絡はあれば無視。 |

### ハッシュラインパッチ言語

| トークン | 意味 |
| --- | --- |
| `§PATH` | セクションヘッダ — 対象ファイル。 |
| `»ANCHOR` | アンカー行の後に挿入。 |
| `«ANCHOR` | アンカー行の前に挿入。 |
| `≔A..B` | 範囲の置換・削除（ペイロード有=置換、無=削除）。 |
| `≔A` | `≔A..A` の略記。 |
| `BOF` / `EOF` | ファイル先頭/末尾アンカー。 |

アンカーは `read`/`search` 出力の `LINEhh` トークン（例：`41th`）です。`|` の左側だけをコピーします。パーサは先頭・末尾アンカーのハッシュを現在のファイルと照合し、古ければ不一致を投げて直近の読み取りスナップショットに基づく復元を試みます。

```text
§src/a.ts
»4fb
const added = true;
```

### 編集 variant

アクティブモードはツールペイロード外で `resolveEditMode()` と `GJC_EDIT_VARIANT` 環境変数により決まり、デフォルトは `hashline`。

| variant | 役割 |
| --- | --- |
| `hashline` | デフォルト — 上記の行アンカーパッチ言語。 |
| `replace` | 厳密/曖昧な旧-新テキスト置換。 |
| `patch` | 構造化 JSON diff-hunk モード。 |
| `apply_patch` | 自由形式 `*** Begin Patch` 包絡、patch モード項目に展開。 |
| `vim` | 永続モーダル編集バッファ。 |
| `atom` | Atom スタイルの編集モード。 |

<Callout type="info">
複数 op パッチは元ファイルのスナップショットに対し解析されます — 先行 op の後で後続アンカーを振り直さないでください。下から上へ適用されます。ペイロードなしの `≔A..B` は範囲を削除し、その場で空行にするには明示的な空ペイロード行を 1 つ入れます。
</Callout>

## `ast-edit`

ネイティブ ast-grep でソースファイルに構造的書き換えをプレビュー・適用します。

| 入力 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `ops` | `{ pat: string; out: string }[]` | はい | 1 つ以上の書き換え規則。`pat` は非空必須、重複 `pat` は失敗、空 `out` は一致ノードを削除。 |
| `paths` | `string[]` | はい | ファイル、ディレクトリ、glob、バッキングファイルのある内部 URL。内部 URL に glob は禁止。 |

[`ast-grep`](/ja/docs/reference/tools/search) のパターン文法を共有します：`$NAME` はノード捕捉、`$_` は束縛なしで 1 つ一致、`$$$NAME` / `$$$` は 0 個以上。メタ変数名は大文字でノード全体を表す必要があり、`pat` の捕捉が `out` に置換されます。

<Steps>
<Step>**まずプレビュー。** `ast-edit` は常にドライランし、ファイル別に `-before` / `+after` を表示します。まだ書き込みません。</Step>
<Step>**resolve で適用。** 強制 `resolve` アクションが予約されます。`resolve(action: "apply", ...)` で書き込み、`discard` で破棄します。</Step>
<Step>**陳腐化チェック。** 適用は書き換えセットを実行し直し、合計がプレビューと一致しなければ適用を拒否します。</Step>
</Steps>

ファイル範囲は `GJC_MAX_AST_FILES`（デフォルト `1000`）で制限されます。ディレクトリスキャンは `.gitignore` に従い、glob が明示しない限り `node_modules` を飛ばします。重なる編集は実行を中止し、構文エラーのファイルはスキップされます。

## どれをいつ使うか

| 目的 | ツール |
| --- | --- |
| 内容・範囲・DB・URL を確認 | `read` |
| 新規ファイル作成または全置換 | `write` |
| その場で数行を変更 | `edit` |
| 複数ファイルにわたる構文パターン書き換え | `ast-edit` |
