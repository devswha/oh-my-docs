本节是 Gajae-Code（`gjc`）的运行时参考，涵盖 `~/.gjc` 下的文件、环境变量的解析方式，以及通往各详细页面的链接。

## 配置文件

Gajae-Code 从 `~/.gjc` 读取配置（用 `GJC_CONFIG_DIR` 覆盖根目录）：

| 路径                       | 用途                              |
| -------------------------- | --------------------------------- |
| `~/.gjc/config.yml`        | 通用运行时设置（如重试预算）。    |
| `~/.gjc/agent/models.yml`  | provider、模型、角色与路由。      |
| `~/.gjc/agent/.env`        | agent 范围环境变量。              |
| `~/.gjc/.env`              | config 根目录环境变量回退。       |
| `~/.env`                   | 主目录环境变量回退。              |

`GJC_CODING_AGENT_DIR` 可独立于 `GJC_CONFIG_DIR` 完整覆盖 agent 目录（默认 `~/.gjc/agent`）。

## .env 解析顺序

多数运行时查找使用 `$env`，按以下顺序加载 `.env` 文件——每一步只填充尚未设置的键：

<Steps>
  <Step>已有进程环境（`Bun.env`）。</Step>
  <Step>项目 `.env`（`$PWD/.env`）。</Step>
  <Step>agent `.env`（`~/.gjc/agent/.env`，遵循 `GJC_CONFIG_DIR` / `GJC_CODING_AGENT_DIR`）。</Step>
  <Step>config 根 `.env`（`~/.gjc/.env`，遵循 `GJC_CONFIG_DIR`）。</Step>
  <Step>主目录 `.env`（`~/.env`）。</Step>
</Steps>
