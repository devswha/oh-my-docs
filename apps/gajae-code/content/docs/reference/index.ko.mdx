이 섹션은 Gajae-Code(`gjc`) 런타임 레퍼런스입니다. `~/.gjc` 아래 파일, 환경변수 해석 방식, 그리고 설정·자격증명·키바인딩·테마 상세 페이지로 가는 링크를 다룹니다.

## 설정 파일

Gajae-Code는 `~/.gjc`에서 설정을 읽습니다 (루트는 `GJC_CONFIG_DIR`로 변경):

| 경로                       | 용도                                    |
| -------------------------- | --------------------------------------- |
| `~/.gjc/config.yml`        | 일반 런타임 설정 (예: 재시도 예산).     |
| `~/.gjc/agent/models.yml`  | provider, 모델, 역할, 라우팅.           |
| `~/.gjc/agent/.env`        | agent 범위 환경변수.                    |
| `~/.gjc/.env`              | config 루트 환경변수 폴백.              |
| `~/.env`                   | 홈 환경변수 폴백.                       |

`GJC_CODING_AGENT_DIR`는 `GJC_CONFIG_DIR`와 무관하게 agent 디렉터리(기본 `~/.gjc/agent`) 전체를 덮어쓸 수 있습니다.

## .env 해석 순서

대부분의 런타임 조회는 `$env`를 사용하며, `.env` 파일을 다음 순서로 로드합니다 — 각 단계는 아직 설정되지 않은 키만 채웁니다:

<Steps>
  <Step>기존 프로세스 환경(`Bun.env`).</Step>
  <Step>프로젝트 `.env`(`$PWD/.env`).</Step>
  <Step>agent `.env`(`~/.gjc/agent/.env`, `GJC_CONFIG_DIR` / `GJC_CODING_AGENT_DIR` 반영).</Step>
  <Step>config 루트 `.env`(`~/.gjc/.env`, `GJC_CONFIG_DIR` 반영).</Step>
  <Step>홈 `.env`(`~/.env`).</Step>
</Steps>

파싱된 각 `.env` 파일 안에서 `GJC_*` 키는 그 파일의 `GJC_*` 키로 미러링됩니다.
