这是 Gajae-Code 运行时读取的环境变量分类参考。所有 `*_API_KEY`、`*_TOKEN`、OAuth/bearer 凭据与云凭据都是**机密**——切勿记录或提交。

`.env` 解析顺序见[参考](/zh/docs/reference)；重试预算与行为开关的上下文见[配置](/zh/docs/reference/configuration)。

## provider 凭据

provider 密钥经由 `getEnvApiKey()` 使用。每个 provider 的认证顺序：CLI `--api-key` → 已存凭据 → 已存 OAuth → 环境变量 → `models.yml` 的 `apiKey`。完整 provider 列表很长（70+ 项），以下为常见项——只为你使用的 provider 设置密钥。

| 变量                                          | provider                          |
| --------------------------------------------- | --------------------------------- |
| `ANTHROPIC_OAUTH_TOKEN` / `ANTHROPIC_API_KEY` | Anthropic（OAuth 令牌优先）       |
| `OPENAI_API_KEY`                              | OpenAI 系列 provider              |
| `GEMINI_API_KEY`                              | Google Gemini                     |
| `OPENROUTER_API_KEY`                          | OpenRouter                        |
| `ZAI_API_KEY`                                 | z.ai (GLM)；也用于 z.ai 网络搜索  |
| `MINIMAX_API_KEY` / `MINIMAX_CODE_API_KEY`    | MiniMax / MiniMax Code            |
| `DEEPSEEK_API_KEY`                            | DeepSeek                          |
| `XAI_API_KEY`                                 | xAI                               |
| `GROQ_API_KEY` / `MISTRAL_API_KEY`            | Groq / Mistral                    |
| `CEREBRAS_API_KEY` / `FIREWORKS_API_KEY` / `TOGETHER_API_KEY` | Cerebras / Fireworks / Together |
| `HUGGINGFACE_HUB_TOKEN` / `HF_TOKEN`          | Hugging Face                      |
| `MOONSHOT_API_KEY`                            | Moonshot                          |
| `QWEN_OAUTH_TOKEN` / `QWEN_PORTAL_API_KEY`    | Qwen Portal（OAuth 令牌优先）     |

还支持更多 provider，包括 Synthetic、NVIDIA、NanoGPT、Venice、LiteLLM、LM Studio、Ollama、llama.cpp、Xiaomi、Azure OpenAI、OpenCode、Qianfan、ZenMux、vLLM、Cursor、Vercel/Cloudflare AI Gateway、Alibaba Coding Plan、Kilo、Ollama Cloud、GitLab Duo 等。本地运行时（LM Studio、Ollama、llama.cpp、vLLM）通常无需认证；若主机要求，任意非空令牌即可。

### Auth broker 与凭据排序

| 变量 | 用途 |
| --- | --- |
| `GJC_AUTH_BROKER_URL` | 远程 auth broker 的 base URL。设置后，本地 SQLite 凭据会被绕过，已保存的 API/OAuth 凭据通过 broker 解析。 |
| `GJC_AUTH_BROKER_TOKEN` | broker 端点的 bearer token（`/v1/healthz` 除外）。可回退到 `auth.broker.token` 或 `<config-dir>/auth-broker.token`。 |
| `GJC_CREDENTIAL_RANKING_MODE` | 多账号 OAuth 选择策略。默认 `balanced`；`earliest-reset` 优先选择 quota reset 最早的账号。 |

Copilot 令牌链：`COPILOT_GITHUB_TOKEN` → `GH_TOKEN` → `GITHUB_TOKEN`。

## provider base URL 覆盖

内置 provider base URL 解析顺序：`models.yml` 的 `baseUrl` → provider 专属 base URL env → 内置默认。

| provider          | 变量                                                                            |
| ----------------- | ------------------------------------------------------------------------------- |
| OpenAI            | `OPENAI_BASE_URL`                                                              |
| Anthropic         | `ANTHROPIC_BASE_URL`                                                           |
| Google Gemini     | `GOOGLE_BASE_URL`、`GEMINI_BASE_URL`                                           |
| 任意 provider id  | 派生 `<PROVIDER_ID>_BASE_URL`（大写化，非字母数字 → `_`；如 `my-proxy` → `MY_PROXY_BASE_URL`）|

内置 `openai` provider 保留其内置 Responses 传输：`OPENAI_BASE_URL` 改变主机但仍调用 `<baseUrl>/responses`。若代理只支持 Chat Completions，请改用 `api: openai-completions` 的自定义 `models.yml` provider。本地运行时发现 base URL 也可覆盖：`OLLAMA_BASE_URL`（`http://127.0.0.1:11434`）、`LM_STUDIO_BASE_URL`（`http://127.0.0.1:1234/v1`）、`LLAMA_CPP_BASE_URL`（`http://127.0.0.1:8080`）。

## 网络搜索密钥

每个密钥启用相应的 `web_search` provider：`EXA_API_KEY`（Exa）、`BRAVE_API_KEY`（Brave）、`PERPLEXITY_API_KEY` / `PERPLEXITY_COOKIES`（Perplexity）、`TAVILY_API_KEY`（Tavily）、`KAGI_API_KEY`（Kagi）、`JINA_API_KEY`（Jina）、`PARALLEL_API_KEY`（Parallel）、`ZAI_API_KEY`（z.ai）、`SEARXNG_ENDPOINT`（+ `SEARXNG_TOKEN`，SearXNG）。

`SEARXNG_BASIC_USERNAME` / `SEARXNG_BASIC_PASSWORD` 提供 HTTP Basic 认证。SearXNG 也读取 `~/.gjc/agent/config.yml` 中等价的 `searxng.*` 设置，env 变量为回退。Anthropic 网络搜索认证链以 `ANTHROPIC_SEARCH_API_KEY`（+ 可选 `ANTHROPIC_SEARCH_BASE_URL`）开始。

## 存储与行为开关

| 变量                   | 效果                                                                       |
| ---------------------- | -------------------------------------------------------------------------- |
| `GJC_CONFIG_DIR`       | home 下 config 根目录名（默认 `.gjc`）。                                   |
| `GJC_CODING_AGENT_DIR` | agent 目录完整覆盖（默认 `~/<GJC_CONFIG_DIR 或 .gjc>/agent`）。            |
| `GJC_NO_PTY`           | `1` 时禁用 bash 工具的交互 PTY 路径。                                      |
| `GJC_NOTIFICATIONS`    | `off` / `0` / `false` 抑制桌面通知。                                       |
| `GJC_NO_TITLE`         | 设置时禁用自动会话标题生成。                                               |
| `GJC_EDIT_VARIANT`     | 强制 edit 工具 variant：`patch`、`replace`、`hashline`、`atom`、`vim`、`apply_patch`。|
| `GJC_TIMING`           | 把分层计时跨度树打印到 stderr（`x` = 打印后退出，`full` = 列出全部模块加载）。|

## 机密变量

将以下内容按机密处理——切勿记录或提交：

- provider/API 密钥与 OAuth/bearer 凭据（所有 `*_API_KEY`、`*_TOKEN`、OAuth access/refresh 令牌）。
- 云凭据（`AWS_*`；`GOOGLE_APPLICATION_CREDENTIALS` 可能指向服务账号材料）。
- 搜索/provider 认证变量（`EXA_API_KEY`、`BRAVE_API_KEY`、`PERPLEXITY_API_KEY`、Anthropic 搜索密钥）。
- Foundry mTLS 材料（`ANTHROPIC_MODEL_CODE_CLIENT_CERT`、`ANTHROPIC_MODEL_CODE_CLIENT_KEY`、指向私有 CA 包的 `NODE_EXTRA_CA_CERTS`）。
