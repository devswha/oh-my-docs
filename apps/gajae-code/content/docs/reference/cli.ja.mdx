## Launch

| Command | 用途 |
| --- | --- |
| `gjc --tmux` | 推奨の tmux-backed interactive session を開始。 |
| `gjc` | tmux session を作成・attach せず直接開始。 |
| `gjc --tmux --worktree my-task-branch` | branch-like task name で GJC-managed sibling Git worktree を作成または使用。 |
| `gjc @screenshot.png "What should I change?"` | image input と一緒に開始。 |

`--worktree` は filesystem path ではありません。既存の worktree directory がある場合は、その中へ `cd` して `gjc --tmux` を実行してください。

## Provider setup

```bash
gjc setup provider --preset minimax
gjc setup provider --preset minimax-cn
gjc setup provider --preset glm
```

TUI 内では次を使います。

```text
/provider add --preset glm
/provider add zai
```

model は TUI では `/model`、CLI では `--model` / `--models` で選びます。

## Workflow commands

```text
/skill:deep-interview clarify the bug, affected behavior, non-goals, and acceptance checks
/skill:ralplan turn the clarified bug report into a reviewed fix plan
gjc ultragoal create-goals --brief-file <approved-plan>
gjc ultragoal complete-goals
```

parallel tmux workers が役立つときだけ `team` を使います。

```bash
gjc team 2:executor "split implementation and verification for this bug fix"
```

## Research mode

```bash
gjc rlm
```

`rlm` は notebook-style research/REPL loop を開き、artifact を `.gjc/_session-<gjcSessionId>/rlm/<rlmSessionId>/` 配下に書きます。

## Auth broker

```bash
gjc auth-broker
```

OAuth refresh token を local machine だけでなく shared host から提供したい場合にのみ broker を使ってください。
