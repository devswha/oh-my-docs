**red-claw 编码智能体 harness (`gjc`)。** 清晰的访谈、稳健的计划、tmux 原生执行、持久的验证。

> 说明：Gajae-Code 是实验性的早期 beta 项目。可能存在粗糙之处，重要工作前请先核实输出。

## 它是什么

Gajae-Code 通过 npm 注册表以 `gajae-code` 发布，该包会安装 `gjc` 可执行文件。它刻意保持较小的公开智能体表面，同时让周边运行时更可靠——会话状态、worktree 隔离、tmux 编排、模型路由、工具执行和持久证据，都会在工作需要时就绪。

它不堆砌庞杂的默认 skill，而是通过把一个方法做得更好来改进 harness。

## 一个有用的循环

```text
deep-interview -> ralplan -> ultragoal
                         └─ 当并行 tmux worker 有帮助时可选 team 执行
```

- **`deep-interview`** — 在规划或改动代码前厘清意图。
- **`ralplan`** — 在变更前构建并评审方案。
- **`$ultragoal`** — 推进实现、修订、验证与证据汇总。
- **`$team`** — 仅当并行 worker 有收益时才加的可选执行模式，并非必需交接步骤。

该顺序描述操作流程，而非隐藏的自动化：智能体仍会报告改了什么、修订了什么、运行了哪些检查、有哪些证据支撑结果。

## 先记住这些

一开始只需要四步。

1. 准备 Bun、Git、tmux 与 provider credentials。
2. 用 `gjc --tmux` 启动。
3. 用 `deep-interview` 和 `ralplan` 澄清并规划。
4. 用 `ultragoal` 推进工作；只有 parallel tmux workers 有帮助时才加 `team`。

如果目标是 research 而不是 mutation，用 `gjc rlm` 开始。
