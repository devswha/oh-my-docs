HUD는 현재 세션이 무엇을 하고 있는지 간결하게 보여주는 모니터링 창구입니다. raw state 파일이나 로그를 직접 들여다볼 필요 없이 phase 변화와 진행 여부를 한눈에 확인할 수 있습니다.

HUD는 워크플로우를 시작하는 곳이 아닙니다. 보통 `$ralph`, `$team`, `$ultraqa` 같은 스킬이 돌아가는 동안 옆에서 지켜보는 용도입니다.

## 명령

```bash
omx hud
omx hud --watch
omx hud --json
omx hud --preset=focused
omx hud --tmux
```

사용 가능한 preset:

- `minimal`
- `focused`
- `full`

## 각 모드의 용도

| 명령 | 용도 |
|---|---|
| `omx hud` | 현재 HUD 상태를 한 번 출력 |
| `omx hud --watch` | 현재 터미널에서 계속 갱신 |
| `omx hud --json` | 스크립트나 외부 도구를 위한 구조화된 출력 |
| `omx hud --preset=<name>` | 표시 정보량 조절 |
| `omx hud --tmux` | 전용 tmux split 열기 |

## 사용 예

### 오래 도는 루프 지켜보기

```bash
omx hud --watch --preset=focused
```

`$ralph`나 `$ultraqa`가 반복 중일 때 로그를 열지 않고 phase 변화를 보고 싶을 때 씁니다.

### 외부 스크립트에 연결하기

```bash
omx hud --json
```

쉘 대시보드, 자동화, 외부 상태 통합에 씁니다.

### tmux pane으로 계속 띄워두기

```bash
omx hud --tmux --preset=minimal
```

tmux 안에서 OMX를 돌리는 중에 옆에 상태 pane을 두고 싶을 때 씁니다.

## 다른 모니터링 surface와의 차이

| surface | 보여주는 것 |
|---|---|
| **HUD** | 지금 무슨 일이 일어나고 있나 |
| **State** | 런타임 기록에 무엇이 적혀 있나 |
| **Trace** | 시간이 지나며 무슨 일이 있었나 |
| **Logs** | raw 상세 내용 |

가장 빠른 운영 뷰가 필요할 때 HUD부터 확인하면 됩니다.
