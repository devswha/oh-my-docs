OMX는 `~/.codex/`(사용자 전역)와 `.codex/`(프로젝트 범위) 두 곳에서 설정을 읽습니다. 같은 키가 겹치면 프로젝트 범위가 우선합니다.

## `.codex/config.toml`

| 필드 | 타입 | 설명 |
|---|---|---|
| `model` | string | 기본 Codex 모델. 현재 OMX routing에서는 frontier/default lane에 참여합니다. |
| `[features].hooks` | bool | setup-owned fallback hook을 위한 안정된 Codex native hook flag. legacy `codex_hooks`는 migration용입니다. |
| `[features].plugin_hooks` | bool | 지원되는 Codex plugin install의 plugin-scoped hook flag. |
| `[features].goals` | bool | goal-oriented workflow가 쓰는 Codex goal integration 활성화. |
| `custom-prompts` | string | OMX 기본 프롬프트를 확장하거나 교체할 디렉터리 경로 |
| `approval_policy` | string | Codex approval policy. autonomous launch는 의도적으로 위험/no-prompt 실행 경계를 선택할 수 있습니다. |

```toml
model = "gpt-5.5"
custom-prompts = "~/.codex/prompts"
approval_policy = "never"

[features]
hooks = true
goals = true
```

## `.omx-config.json`

Codex root TOML key가 아닌 OMX-owned routing/feature config는 `.omx-config.json`에 둡니다.

| Key | 목적 |
|---|---|
| `agentModels` | `{ "planner": "gpt-5.5" }` 같은 per-agent model override. built-in role pin보다 먼저 적용됩니다. |
| `agentReasoning` | `low`, `medium`, `high`, `xhigh` per-agent reasoning override. |
| `env` | `OMX_DEFAULT_FRONTIER_MODEL`, `OMX_DEFAULT_STANDARD_MODEL`, `OMX_DEFAULT_SPARK_MODEL` 같은 fallback env 값. 실제 shell env가 우선합니다. |
| `models` | `default`, `team`, `autopilot`, `team_low_complexity` 같은 mode default. |

`agentModels.planner`는 Autopilot/Ralplan planning에서 중요합니다. main/default lane이 cheap/mini면 OMX가 initial planning draft를 dedicated planner role로 보낼 수 있습니다. Team model routing은 `omx team status <team> --model-inspect`로 확인하세요.

## `.codex/hooks.json`

라이프사이클 시점마다 실행할 훅 핸들러를 등록합니다. 각 항목은 훅 이름을 command 또는 MCP tool에 연결합니다.

```json
{
  "hooks": {
    "session-end": {
      "command": "~/.codex/scripts/on-session-end.sh {{sessionId}}"
    }
  }
}
```

전체 훅 카탈로그는 [업스트림 저장소](https://github.com/Yeachan-Heo/oh-my-codex)를 참고하세요.

## CLI 플래그

| 플래그 | 설명 |
|---|---|
| `--madmax` | high-autonomy 실행 경계를 위한 dangerous bypass shorthand입니다. full Team pipeline 시작과 같은 의미가 아닙니다. |
| `--high` | high reasoning-effort shorthand입니다. model tier selector가 아닙니다. |
| `--xhigh` | 지원되는 경우 extra-high reasoning shorthand입니다. |
| `--skip-git-repo-check` | git 저장소 밖에서 실행할 때 발생하는 오류를 무시합니다. |
| `--spark` | team worker를 가벼운 spark tier로 라우팅합니다. |
| `--madmax-spark` | `--madmax`와 spark worker 라우팅을 함께 적용합니다. |
