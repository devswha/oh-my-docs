## 最新リリース — v0.14.4

デフォルトの frontier lane を `gpt-5.5` に引き上げつつ、standard/mini lane と spark lane は既存の厳密なモデル固定を維持するパッチリリースです。

**変更内容:**
- runtime defaults、Codex agent defaults、`omx explore` fallback behavior の frontier default が `gpt-5.5` になりました。
- setup/config guidance と regression coverage が `gpt-5.5` default を説明するよう更新され、既存の context size 推奨値は維持されています。
- 生成される setup config と executor worker launch defaults は、デフォルトで medium reasoning を使うようになりました。
- mini と spark lane は厳密に維持されます。`gpt-5.4-mini` と `gpt-5.3-codex-spark` の動作は変更されていません。

**修正内容:**
- Node/Cargo metadata、lockfile、changelog、release body、release notes、release-readiness collateral のリリースメタデータ drift を整合させました。

## v0.14.3

`0.14.2` 以降の信頼性を強化するリリースです。`omx question`/deep-interview return-pane behavior、project-local explore launch context、setup TOML repair safety、HUD window targeting、ultrawork protocol alignment、BusyBox cleanup compatibility、stale Stop/autopilot state handling、supervisor runtime events、Docker-host tmux question rendering、native Windows psmux worker pane bootstrap hardening を扱います。

## v0.14.2 と v0.14.1

interactive orchestration の信頼性に集中したパッチリリースです。より安全な `omx question` renderer behavior、shared tmux answer-submit semantics、より厳密な deep-interview state handling、Korean `ulw` keyboard drift handling、setup/update refresh resilience、lifecycle contract cleanup、より明確な deep-interview background-question guidance を含みます。

## v0.14.0

interactive orchestration の変更を中心とした minor release です。新しい `omx question` blocking-question entrypoint、structured deep-interview question obligations、advisory triage routing、validator-gated autoresearch、明示的な runtime run outcomes、より明確な specialist routing、出荷パッケージの hardening が含まれます。

## v0.13.x シリーズ

`omx adapt` foundations、Ralph/runtime authority fixes、hook と HUD の reliability、worker identity preservation、security hardening、detached tmux startup repairs を扱ったシリーズです。
