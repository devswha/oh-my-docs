## 最新リリース — v0.17.0

`0.16.4` 以降の minor release です。bounded Hermes MCP coordination、canonical `$design` workflow guidance、plugin-mode skill marketplace exposure、より強い UltraQA adversarial contract、Windows native-hook reliability、tmux ownership safety、startup shell isolation、committed project-memory loading、Ultragoal task-scoped goal reconciliation に焦点を当てています。

**追加内容:**
- Hermes MCP bridge: tmux scrollback や raw private state を公開せず、session/status read、audited follow-up dispatch、safe artifact read、log tail、session start、coordination report を bounded tool として提供します。
- Canonical design workflow: `DESIGN.md` と mirror された `$design` skill guidance が design workflow を定義し、`frontend-ui-ux` は deprecated になりました。
- Plugin-mode skill discovery: setup は local Codex plugin marketplace/cache と、Hermes を含む plugin-scoped MCP metadata を登録・検証します。

**変更内容:**
- `$ultraqa` は adversarial by contract となり、hostile scenario modeling、prompt-injection attempt、interrupt/cancel/resume、stale state check、必要に応じた temporary harness、cleanup evidence を要求します。
- CLI-first runtime authority guidance が、CLI runtime の所有する動作と MCP/default setup path をより明確に区別します。

**修正内容:**
- Windows native hook launch は PowerShell `ProcessStartInfo` shim により stdio と exit status を保持します。
- Tmux continuation は follow-up prompt を注入する前に mode/session/window ownership を検証します。
- Startup tmux launch は Codex 起動前の shell rc fan-out を避けます。
- Ultragoal は strict evidence gate のもと、完了済み task-scoped aggregate Codex goal を active OMX story に reconcile できます。
- Release-review MCP/Hermes test は inherited runtime env を isolate し、macOS temp root を canonicalize します。

## v0.16.4

`0.16.3` 以降の approved execution handoff、context-pack metadata visibility、Codex hook feature-flag migration、setup/notify ownership、HUD/runtime state-root visibility、Ralph completion audit evidence、Ultragoal completion proof requirements を強化する信頼性パッチです。

**追加内容:**
- Ralph、Team、planning handoff が approved context references、context-pack refs、private entry metadata、canonical PRD aliases、ready role refs、multiline launch hints、visible-hint lineage を実行ライフサイクル全体で保持します。
- Ralph completion は done-state claim を受け入れる前に audit evidence を記録・確認します。

**変更内容:**
- Ultragoal completion は final cleanup/review proof を必須にし、docs/skills でもその stop condition を明示します。
- skill catalog guidance は active OMX runtime surfaces を中心により厳密になりました。

**修正内容:**
- Codex hook setup はサポートされる場合 `[features].hooks = true` を優先し、古い CLI 向けに `[features].codex_hooks = true` fallback を維持し、stale aliases を dedupe し、clear reset 後も hooks を保持します。
- approved handoff は Team scale-up、multiline launch hints、visible hint lineage fallbacks、context-pack parsing edge cases、stale PostCompact/notify recursion hazards の中でも維持されます。
- HUD、boxed Team state-root precedence、plugin-mode skill discovery、plugin MCP cleanup は runtime authority により安全に根づきます。

## v0.14.4

デフォルトの frontier lane を `gpt-5.5` に引き上げつつ、standard/mini lane と spark lane は既存の厳密なモデル固定を維持するパッチリリースです。

**変更内容:**
- runtime defaults、Codex agent defaults、`omx explore` fallback behavior の frontier default が `gpt-5.5` になりました。
- setup/config guidance と regression coverage が `gpt-5.5` default を説明するよう更新され、既存の context size 推奨値は維持されています。
- 生成される setup config と executor worker launch defaults は、デフォルトで medium reasoning を使うようになりました。
- mini と spark lane は厳密に維持されます。`gpt-5.4-mini` と `gpt-5.3-codex-spark` の動作は変更されていません。

**修正内容:**
- Node/Cargo metadata、lockfile、changelog、release body、release notes、release-readiness collateral のリリースメタデータ drift を整合させました。
