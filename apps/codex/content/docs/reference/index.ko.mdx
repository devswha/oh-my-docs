## 참조란

참조(Reference)는 OMX를 설정 · 디버깅 · 운영할 때 **반복해서 찾아보게 되는 안정적 사실**을 모은 섹션입니다.

가이드가 "어떤 흐름을 고를까"를 다룬다면, 참조는 **정확한 이름 · 위치 · 우선순위 · 규칙**을 꺼내오는 곳입니다.

## 이 섹션에 있는 것

### 설정

[설정](/ko/docs/reference/configuration) — `.codex/config.toml`, `.codex/hooks.json`, CLI 플래그 등 설정의 중심.

- 설정 파일 위치
- 사용자 범위 vs 프로젝트 범위 우선순위
- launch flag 의미

### 퀵 커맨드

[퀵 커맨드](/ko/docs/reference/quick-commands) — 자주 쓰는 OMX 명령을 빠르게 찾는 페이지. 설명이 필요한 게 아니라 **명령 자체가 필요할 때** 봅니다.

### 환경

[환경](/ko/docs/reference/environment) — 환경 변수와 런타임 플래그.

예: `OPENAI_API_KEY`, `OMX_OPENCLAW`, `OMX_STATE_DIR`

### 마이그레이션

[마이그레이션](/ko/docs/reference/migration) — 버전 업그레이드나 저장소 이동 시 주의 사항.

### 변경 로그

[변경 로그](/ko/docs/reference/changelog) — 릴리스 기록과 실배포 변경 내역.

### 계약

[계약](/ko/docs/reference/contracts) — OMX가 유지하려는 동작 계약과 가이던스 표면.

### `.omx/` 디렉터리

[`.omx/` 디렉터리](/ko/docs/reference/omx-directory) — 상태 · 계획 · 노트 · wiki가 저장되는 구조.

### CJK IME

[CJK IME](/ko/docs/reference/cjk-ime) — 한국어 · 일본어 · 중국어 입력기 이슈와 우회 방법.

## 언제 여기를 먼저 보나

- 설정 파일 이름이나 위치가 헷갈릴 때
- 플래그 · 환경 변수의 정확한 이름이 필요할 때
- `.omx/` 하위 파일의 의미를 확인할 때
- 설명보다 **정확한 사실**이 필요할 때

"이걸 어떻게 이해할까"보다 **"정확히 뭐였지?"** 쪽이면 여기입니다.
