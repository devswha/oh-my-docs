## 最新版本 — v0.16.4

这是 `0.16.3` 之后的可靠性补丁，强化 approved execution handoff、context-pack metadata visibility、Codex hook feature-flag migration、setup/notify ownership、HUD/runtime state-root visibility、Ralph completion audit evidence 与 Ultragoal completion proof requirements。

**新增内容：**
- Ralph、Team 与 planning handoff 现在会在执行生命周期中保留 approved context references、context-pack refs、private entry metadata、canonical PRD aliases、ready role refs、multiline launch hints 与 visible-hint lineage。
- Ralph completion 在接受 done-state claim 前会记录并检查 audit evidence。

**变更内容：**
- Ultragoal completion 现在要求 final cleanup/review proof，docs/skills 也明确该 stop condition。
- skill catalog guidance 围绕 active OMX runtime surfaces 进一步收紧。

**修复内容：**
- Codex hook setup 在支持时优先写入 `[features].hooks = true`，同时保留旧 CLI 的 `[features].codex_hooks = true` fallback，dedupe stale aliases，并在 clear reset 后保持 hooks 生效。
- approved handoff 在 Team scale-up、multiline launch hints、visible hint lineage fallbacks、context-pack parsing edge cases，以及 stale PostCompact/notify recursion hazards 中都能保留。
- HUD、boxed Team state-root precedence、plugin-mode skill discovery 与 plugin MCP cleanup 更安全地扎根在 runtime authority 上。

## v0.14.4

这是一个补丁版本，将默认 frontier lane 提升到 `gpt-5.5`，同时保持 standard/mini lane 与 spark lane 的精确模型固定不变。

**变更内容：**
- runtime defaults、Codex agent defaults 与 `omx explore` fallback behavior 的 frontier default 现在使用 `gpt-5.5`。
- setup/config guidance 与 regression coverage 已更新为描述 `gpt-5.5` default，同时保留现有 context size 建议。
- 生成的 setup config 与 executor worker launch defaults 现在默认使用 medium reasoning。
- mini 与 spark lane 保持精确不变：`gpt-5.4-mini` 和 `gpt-5.3-codex-spark` 行为未改变。

**修复内容：**
- 对齐 Node/Cargo metadata、lockfile、changelog、release body、release notes 与 release-readiness collateral 中的发布元数据 drift。

## v0.14.3

这是 `0.14.2` 之后的可靠性强化版本，覆盖 `omx question`/deep-interview return-pane behavior、project-local explore launch context、setup TOML repair safety、HUD window targeting、ultrawork protocol alignment、BusyBox cleanup compatibility、stale Stop/autopilot state handling、supervisor runtime events、Docker-host tmux question rendering，以及 native Windows psmux worker pane bootstrap hardening。

## v0.14.2 与 v0.14.1

这些补丁版本聚焦 interactive orchestration 可靠性：更安全的 `omx question` renderer behavior、共享 tmux answer-submit semantics、更严格的 deep-interview state handling、Korean `ulw` keyboard drift handling、setup/update refresh resilience、lifecycle contract cleanup，以及更清晰的 deep-interview background-question guidance。

## v0.14.0

这是围绕 interactive orchestration 变化的 minor release，包括新的 `omx question` blocking-question entrypoint、structured deep-interview question obligations、advisory triage routing、validator-gated autoresearch、明确的 runtime run outcomes、更清晰的 specialist routing，以及对发布包的 hardening。
