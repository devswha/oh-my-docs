## 최신 릴리스 — v0.17.0

`0.16.4` 이후의 minor release입니다. bounded Hermes MCP coordination, canonical `$design` workflow guidance, plugin-mode skill marketplace exposure, 더 강한 UltraQA adversarial contract, Windows native-hook reliability, tmux ownership safety, startup shell isolation, committed project-memory loading, Ultragoal task-scoped goal reconciliation에 초점을 맞췄습니다.

**추가 사항:**
- Hermes MCP bridge: tmux scrollback이나 raw private state를 노출하지 않고 session/status read, audited follow-up dispatch, safe artifact read, log tail, session start, coordination report를 bounded tool로 제공합니다.
- Canonical design workflow: `DESIGN.md`와 mirror된 `$design` skill guidance가 design workflow를 정의하며, `frontend-ui-ux`는 deprecated되었습니다.
- Plugin-mode skill discovery: setup이 local Codex plugin marketplace/cache와 Hermes를 포함한 plugin-scoped MCP metadata를 등록·검증합니다.

**변경 사항:**
- `$ultraqa`는 adversarial by contract가 되어 hostile scenario modeling, prompt-injection attempt, interrupt/cancel/resume, stale state check, 필요 시 temporary harness, cleanup evidence를 요구합니다.
- CLI-first runtime authority guidance가 CLI runtime이 소유하는 동작과 MCP/default setup path를 더 명확히 구분합니다.

**수정 사항:**
- Windows native hook launch가 PowerShell `ProcessStartInfo` shim을 통해 stdio와 exit status를 보존합니다.
- Tmux continuation은 follow-up prompt 주입 전에 mode/session/window ownership을 검증합니다.
- Startup tmux launch는 Codex 시작 전에 shell rc fan-out을 피합니다.
- Ultragoal은 완료된 task-scoped aggregate Codex goal을 strict evidence gate와 함께 active OMX story로 reconcile할 수 있습니다.
- Release-review MCP/Hermes test는 inherited runtime env를 isolate하고 macOS temp root를 canonicalize합니다.

## v0.16.4

`0.16.3` 이후 approved execution handoff, context-pack metadata 가시성, Codex hook feature flag migration, setup/notify ownership, HUD/runtime state-root 가시성, Ralph completion audit evidence, Ultragoal completion proof requirement를 강화한 안정화 패치입니다.

**추가 사항:**
- Ralph, Team, planning handoff가 approved context reference, context-pack ref, private entry metadata, canonical PRD alias, ready role ref, multiline launch hint, visible-hint lineage를 끝까지 보존합니다.
- Ralph completion은 done-state claim을 받아들이기 전에 audit evidence를 기록하고 확인합니다.

**변경 사항:**
- Ultragoal completion은 final cleanup/review proof를 요구하며, docs/skills도 이 stop condition을 명시합니다.
- skill catalog guidance가 active OMX runtime surface 중심으로 더 엄격해졌습니다.

**수정 사항:**
- Codex hook setup은 지원되는 경우 `[features].hooks = true`를 우선 사용하고, 오래된 CLI를 위해 `[features].codex_hooks = true` fallback을 유지하며, stale alias를 dedupe하고 clear reset 이후에도 hook을 유지합니다.
- approved handoff가 Team scale-up, multiline launch hint, visible hint lineage fallback, context-pack parsing edge case, stale PostCompact/notify recursion 위험에서도 유지됩니다.
- HUD, boxed Team state-root precedence, plugin-mode skill discovery, plugin MCP cleanup이 runtime authority에 더 안전하게 고정됩니다.

## v0.14.4

기본 frontier lane을 `gpt-5.5`로 올리면서 standard/mini lane과 spark lane은 기존의 정확한 모델 고정값으로 유지하는 패치 릴리스입니다.

**변경 사항:**
- runtime 기본값, Codex agent 기본값, `omx explore` fallback 동작에서 frontier 기본값이 `gpt-5.5`를 사용합니다.
- setup/config 안내와 regression coverage가 `gpt-5.5` 기본값을 설명하도록 갱신되었고, 기존 context size 권장값은 유지됩니다.
- 생성되는 setup config와 executor worker launch 기본값이 이제 medium reasoning을 기본으로 사용합니다.
- mini와 spark lane은 그대로입니다. `gpt-5.4-mini`와 `gpt-5.3-codex-spark` 동작은 변경되지 않았습니다.

**수정 사항:**
- Node/Cargo metadata, lockfile, changelog, release body, release note, release-readiness 자료의 릴리스 메타데이터 drift를 정렬했습니다.

## v0.14.3

`0.14.2` 이후의 안정성 강화 릴리스입니다. `omx question`/deep-interview return-pane 동작, project-local explore launch context, setup TOML repair 안전성, HUD window targeting, ultrawork protocol 정렬, BusyBox cleanup 호환성, stale Stop/autopilot state 처리, supervisor runtime event, Docker-host tmux question rendering, native Windows psmux worker pane bootstrap hardening을 다룹니다.

## v0.14.2 및 v0.14.1
