title: "CJK IME"

---

Codex CLI はターミナルベースの入力モデルを使用しているため、CJK IME（Input Method Editor）と競合する場合があります。IME は文字を確定する前に複数のキー入力を組み合わせますが、この中間的な composition 状態がターミナル上で見えないため、文字の欠落、二重入力、意図より早い送信などの問題が生じることがあります。

## 既知の問題

**composition イベントの競合** — IME が文字を入力中のとき、一部のターミナルは完了していないキー入力を先に CLI へ送信します。Codex CLI はこれを別々の入力として処理するため、テキストが壊れたり予期しないコマンドが実行されたりすることがあります。

**composition 中の Enter キーの問題** — IME 候補ウィンドウが開いている状態で Enter を押すと、IME の選択ではなく未完成の入力が送信されます。macOS/Linux の韓国語・日本語・中国語ユーザーがよく経験する問題です。

**確定時の二重入力** — 一部の IME + ターミナルの組み合わせでは、確定された文字と raw keystroke の両方が送られ、文字が二重に入力されます。

## 回避策

**確定後に送信** — Enter を押す前に IME 候補ウィンドウが閉じ、composition が完全に終わっていることを確認してください。カーソルが通常状態（下線なし）に戻ってから Enter を押すのが安全です。

**スキル呼び出しは ASCII モードで** — `$ralph`、`$team` などの OMX スキル呼び出しは英語入力モードに切り替えてから入力してください。自然言語の引数は必要に応じて再度 CJK モードに変えて入力できます。

**コピー＆ペーストを活用** — CJK 文章をエディタで先に作成してから CLI に貼り付ける方法が安定しています。貼り付けは IME の composition 段階を経ないため、バッファに正確に入力されます。

**ターミナルを変更** — ターミナルによっては IME composition イベントをより適切に処理するものがあります。macOS では iTerm2、Windows では Windows Terminal が基本的な xterm 系よりも競合が少ない場合があります。

## 環境設定

ターミナルと Codex CLI の文字エンコーディングを合わせるため、`LANG`、`LC_ALL` を UTF-8 ロケールに設定してください。

```bash
export LANG=ja_JP.UTF-8
export LC_ALL=ja_JP.UTF-8
```
