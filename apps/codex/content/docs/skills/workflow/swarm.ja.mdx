## 一行要約

`$swarm` は作業を複数のエージェントに同時分散させます。現在は `$team` が同じ役割を担うため、新しい作業には `$team` を使うことが推奨されます。

## 呼び出し

```
$swarm "フロントエンド・バックエンド・テストを並列実装して"
```

マジックキーワード `swarm`、`team`、`coordinated team` でも起動します。

## `$swarm` vs `$team`

| 項目 | `$swarm` | `$team` |
|---|---|---|
| 状態 | レガシー/変形 | 現在の標準 |
| 機能 | 並列エージェント実行 | 並列 lane + stage ルーティング |
| 推奨 | 既存ワークフロー互換 | 新規作業推奨 |

`$team` は stage-aware ルーティング（plan → exec → verify → fix）をサポートします。新しい作業なら `$team` を使います。

## いつ使うか

| 合う | 避ける |
|---|---|
| 独立して分かれる並列作業 | 要件が曖昧なとき |
| 既存の `$swarm` ワークフローを維持する必要があるとき | lane 同士で衝突可能性があるとき |

並列化より単一オーナーが必要な作業なら `$ralph` がより安全です。
