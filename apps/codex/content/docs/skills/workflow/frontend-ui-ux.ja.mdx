## 一行要約

`$frontend-ui-ux` は画面設計、コンポーネント構造、インタラクションフローのようなフロントエンド特化作業を扱います。designer エージェントを中心に実行されます。

## 呼び出し

```
$frontend-ui-ux "ダッシュボードレイアウトとフィルターコンポーネントを設計して"
```

## いつ使うか

| 合う | 避ける |
|---|---|
| UI 構造・インタラクション設計 | バックエンドロジックのみの作業 |
| コンポーネント階層と状態フローの定義 | データモデル・API 設計 |
| アクセシビリティ・レスポンシブレイアウト要件 | |

## 動作方法

1. 要件から画面単位とコンポーネントを導出します。
2. インタラクションフローと状態管理の方向を決めます。
3. コードまたは設計ドキュメントとして結果を出力します。

## `$team` と組み合わせる

フロントエンドとバックエンドを並列に進められる場合、`$team` と組み合わせられます。

```
$team 2:executor "frontend-ui-ux の仕様通りにコンポーネントを実装、バックエンド API を並列で作成"
```
