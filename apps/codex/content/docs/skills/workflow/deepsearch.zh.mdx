## 一句话概述

`$deepsearch` 用于需要比 Grep 更深层探索的场景。多个 agent 并行分析结构、模式、依赖关系，然后汇总结果。

## 调用方式

```
$deepsearch "全面了解认证中间件在哪里、如何被使用"
```

## 何时使用

| 适合 | 避免 |
|---|---|
| 变更前需要了解影响范围时 | 单文件或单函数级别的探索 |
| 需要在遗留代码中找到模式时 | 范围已经明确的工作 |
| 跨多个模块的依赖关系分析 | |

## 工作方式

1. explore agent 构建文件和符号地图。
2. 并行探索相关区域。
3. 汇总结果，输出摘要和文件列表。

结果可以直接作为上下文传给 `$ralplan` 或 `$ralph`。

## 相关 skill

- [`$ralplan`](/zh/docs/skills/workflow/ralplan) — 将探索结果转化为计划
- [`$ralph`](/zh/docs/skills/workflow/ralph) — 在已知范围内执行
