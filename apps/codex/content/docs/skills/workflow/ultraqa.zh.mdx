title: $ultraqa
description: 自动循环诊断-修复-重跑，直到测试、构建、lint 全部通过的 QA 循环
---

`$ultraqa` 是不会在第一次失败时停下的自主 QA 循环。运行质量检查，失败时由 Architect agent 诊断根本原因，Executor agent 应用修复后再次检查。最多循环 5 次，同一失败连续出现 3 次则判断为结构性问题并提前终止。

## 何时使用

- 想不手动调试就让构建、测试、lint、类型检查通过时
- 接手损坏的代码库需要系统性第一轮修复时
- 功能完成后想在 review 前自动化验证时
- 想边看每轮进度日志边跟进时

触发关键词: "ultraqa", "fix until tests pass", "qa cycle", "make the build pass"

## 调用方式

```
codex
> ultraqa --tests
```

```
codex
> $ultraqa --build
```

```
codex
> $ultraqa --custom "no TypeScript errors in packages/api"
```

可用目标标志: `--tests`、`--build`、`--lint`、`--typecheck`、`--custom "模式"`、`--interactive`

## 工作方式

解析目标标志并开始第 1 轮。运行对应命令，通过退出码或输出模式确认结果。已经是通过状态则立即输出成功消息并结束。

失败时，Architect agent 分析输出并给出根本原因和具体修复方向。Executor agent 精确应用该修复，计数器递增后重新运行检查。直到达成目标、消耗完 5 轮，或同一失败连续 3 次为止。

每轮输出 `[ULTRAQA Cycle N/5]` 进度行。状态记录在 `.omx/state/ultraqa-state.json`，完成或取消时删除。

## 输出

- 经 Architect-Executor 修复循环修改的源文件
- 包含每轮失败摘要和修复内容的进度日志
- 成功时附带循环次数的结束消息；未达成时输出根本原因诊断消息
