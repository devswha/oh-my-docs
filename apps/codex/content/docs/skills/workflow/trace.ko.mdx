`$trace`는 현재 세션의 실행 기록을 꺼냅니다. `trace_timeline`과 `trace_summary` MCP 도구를 호출해 시간순 이벤트 로그와 집계 통계를 출력합니다. 키워드가 어떤 스킬을 트리거했는지, 에이전트 위임이 어떻게 이뤄졌는지, 어느 도구가 병목이었는지를 파악할 수 있습니다.

## 언제 쓰나

- 복잡한 멀티에이전트 세션에서 무슨 일이 일어났는지 파악할 때
- 예상치 못한 스킬 활성화나 모드 전환을 디버깅할 때
- 느린 도구나 에이전트를 찾아 병목을 제거할 때
- 훅이 어떤 순서로 실행됐는지 감사할 때

트리거 키워드: "trace", "show trace", "what happened", "flow trace", "session trace"

## 호출 방법

```
codex
> trace
```

```
codex
> $trace
```

## 동작 방식

`trace_timeline`을 인수 없이 호출해 최신 세션의 시간순 이벤트 로그를 가져옵니다. 타임라인에는 훅 실행, 키워드 감지, 스킬 활성화, 에이전트 위임, 모드 전환, 도구 호출이 발생 순서대로 나타납니다.

이어서 `trace_summary`로 집계 통계를 생성합니다. 각 훅의 실행 횟수, 감지된 키워드, 활성화된 스킬, 모드 전환 횟수, 가장 느린 도구 목록이 포함됩니다.

출력은 타임라인을 먼저 보여주고 요약이 뒤따릅니다. 모드 전환은 강조 표시되고, 시간을 많이 쓴 도구와 에이전트는 병목으로 명시됩니다. 키워드 → 스킬 → 에이전트로 이어지는 인과 구조도 함께 정리됩니다.

## 출력

- 현재 세션의 시간순 이벤트 타임라인
- 훅 실행 횟수, 키워드 감지, 스킬 활성화, 모드 전환 집계
- 가장 느린 도구와 에이전트를 명시한 병목 보고서
- 키워드 → 스킬 → 에이전트 인과 흐름 요약
