## 一行要約

要件が曖昧なときはコードより質問が先です。`$deep-interview` は Socratic な方法でユーザーと対話しながら、実装前にスコープを確定します。

## 呼び出し

```
$deep-interview "認証変更事項を明確にして"
```

マジックキーワード `deep interview`、`interview me`、`gather requirements`、`don't assume`、`ouroboros` でも起動します。

## いつ使うか

| 合う | 避ける |
|---|---|
| 要件がまだ整理されていない | スコープが明確ですぐ実装できるとき |
| ステークホルダーが複数いて方向が違うとき | プロトタイプのように探索が目的のとき |
| 「何を作るべきか」から確定する必要があるとき | |

## 標準フローでの位置

```
$deep-interview "認証変更スコープを整理して"   ← ここ
$ralplan "最も安全な実装経路を承認して"
$ralph "承認された計画を最後まで完了して"
```

`$deep-interview` は常に最初です。スコープが確定した後に `$ralplan` で計画を立ててください。

## 動作方法

- ユーザーが答えるたびに追加質問を生成します。
- 核心的な前提が明確になると要件サマリーを出力します。
- サマリーを確認した後に `$ralplan` に渡すか、そのまま実装に進めます。

## 関連スキル

- [`$ralph`](/ja/docs/skills/workflow/ralph) — 計画を実行
