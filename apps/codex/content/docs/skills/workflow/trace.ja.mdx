`$trace` は現在のセッションの実行記録を取り出します。`trace_timeline` と `trace_summary` MCP ツールを呼び出して時系列イベントログと集計統計を出力します。キーワードがどのスキルをトリガーしたか、エージェント委任がどのように行われたか、どのツールがボトルネックだったかを把握できます。

## いつ使うか

- 複雑なマルチエージェントセッションで何が起きたか把握するとき
- 想定外のスキル起動やモード切り替えをデバッグするとき
- 遅いツールやエージェントを見つけてボトルネックを除去するとき
- フックがどの順番で実行されたか監査するとき

トリガーキーワード: "trace", "show trace", "what happened", "flow trace", "session trace"

## 呼び出し

```
codex
> trace
```

```
codex
> $trace
```

## 動作方法

`trace_timeline` を引数なしで呼び出して最新セッションの時系列イベントログを取得します。タイムラインにはフック実行、キーワード検出、スキル起動、エージェント委任、モード切り替え、ツール呼び出しが発生順に表示されます。

続いて `trace_summary` で集計統計を生成します。各フックの実行回数、検出されたキーワード、起動されたスキル、モード切り替え回数、最も遅いツール一覧が含まれます。

出力はタイムラインを先に表示してサマリーが続きます。モード切り替えは強調表示され、時間を多く使ったツールとエージェントはボトルネックとして明示されます。キーワード → スキル → エージェントと続く因果構造も合わせて整理されます。

## 出力

- 現在のセッションの時系列イベントタイムライン
- フック実行回数、キーワード検出、スキル起動、モード切り替えの集計
- 最も遅いツールとエージェントを明示したボトルネックレポート
- キーワード → スキル → エージェントの因果フローサマリー
