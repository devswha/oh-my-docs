## 한 줄 요약

`$swarm`은 작업을 여러 에이전트에게 동시에 분산시킵니다. 현재는 `$team`이 같은 역할을 하므로, 새 작업에는 `$team`을 쓰는 것이 권장됩니다.

## 호출

```
$swarm "프론트엔드·백엔드·테스트를 병렬로 구현해줘"
```

매직 키워드 `swarm`, `team`, `coordinated team`으로도 활성화됩니다.

## `$swarm` vs `$team`

| 항목 | `$swarm` | `$team` |
|---|---|---|
| 상태 | 레거시/변형 | 현재 표준 |
| 기능 | 병렬 에이전트 실행 | 병렬 lane + stage 라우팅 |
| 권장 여부 | 기존 워크플로우 호환 | 신규 작업 권장 |

`$team`은 stage-aware 라우팅(plan → exec → verify → fix)을 지원합니다. 새 작업이라면 `$team`을 씁니다.

## 언제 쓰나

| 잘 맞는 경우 | 피할 경우 |
|---|---|
| 독립적으로 나뉘는 병렬 작업 | 요구가 흐릴 때 |
| 기존 `$swarm` 워크플로우를 유지해야 할 때 | lane끼리 충돌 가능성이 있을 때 |

병렬화보다 단일 소유자가 필요한 작업이라면 `$ralph`가 더 안전합니다.

## 관련 스킬

- [`$team`](/ko/docs/skills/workflow/pipeline) — 현재 표준 병렬 실행
- [`$ralph`](/ko/docs/skills/workflow/ralph) — 단일 소유자 반복 실행
