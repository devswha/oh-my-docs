## 一行要約

`$ralph` は作業を完了証拠が確認されるまで繰り返し実行します。一度で終わらなかったり途中で詰まる可能性がある作業に使います。

## 呼び出し

```
$ralph "承認された計画を最後まで完了して"
```

マジックキーワード `ralph`、`don't stop`、`must complete`、`keep going` でも起動します。

## いつ使うか

| 合う | 避ける |
|---|---|
| 計画があって最後まで押し通す必要があるとき | 要件がまだ曖昧なとき |
| ビルド・テスト失敗を自動修正する必要があるとき | 方向合意が先の作業 |
| 繰り返し試行が必要な作業 | lane に分けられる並列作業 |

## 動作方法

1. 作業状態を `.omx/state/` に記録します。
2. 実行 → 検証 → 失敗時修正 → 再実行を繰り返します。
3. verifier が完了証拠を確認するとループを終了します。

完了証拠は通常、ビルド通過、テスト通過、または指定した成果物の存在確認です。

## 標準フローでの位置

```
$deep-interview "..."      ← 要件確定
$ralplan "..."             ← 計画合意
$ralph "..."               ← ここで実行
```

## `$ralph` vs `$team`

どちらも実行スキルですが、方向が異なります。

| 項目 | `$ralph` | `$team` |
|---|---|---|
| 実行方式 | 単一オーナー、繰り返しループ | 複数 lane 並列 |
| 合う場合 | コンテキストを最後まで保持する必要があるとき | 作業が独立単位に分かれるとき |
| 衝突リスク | 低い | lane 設計が間違えると高くなる |

要件が曖昧だったり lane 同士で衝突しそうなら `$ralph` がより安全です。
