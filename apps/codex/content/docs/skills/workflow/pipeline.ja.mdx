## 一行要約

`$pipeline` は複数のステップを順番に実行するときに使います。各ステップが完了してから次に進み、中間結果が次のステップの入力になります。

## 呼び出し

```
$pipeline "lint → test → build → deploy の順で実行して"
```

## いつ使うか

| 合う | 避ける |
|---|---|
| CI/CD ステップのように順序が固定された作業 | 並列化可能な独立した作業 |
| 各ステップの出力が次のステップの入力になる場合 | 単一作業の実行 |
| 繰り返し再現可能なリリースフロー | |

## `$ralph` との違い

`$ralph` は完了証拠が出るまで繰り返し実行するループです。`$pipeline` はステップの順序が決まっていて、各ステップを一度ずつ通過します。ステップごとの結果を確認しながら進む場合は `$pipeline` がより適しています。
