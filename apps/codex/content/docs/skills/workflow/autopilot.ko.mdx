## 한 줄 요약

아이디어 하나를 넣으면 구현 → QA → 검증까지 OMX가 순서대로 처리합니다. 중간에 방향을 조율하고 싶다면 `$deep-interview → $ralplan → $ralph` 흐름을 쓰세요.

## 호출

```
$autopilot "build me a todo app in next.js"
```

매직 키워드 `autopilot`, `build me`, `I want a`로도 활성화됩니다.

## 언제 쓰나

| 잘 맞는 경우 | 피할 경우 |
|---|---|
| 프로토타입, 작은 기능 | 요구사항이 아직 흐릴 때 |
| end-to-end 흐름 빠르게 확인 | 아키텍처 합의가 먼저인 작업 |
| 제어보다 속도가 중요할 때 | 잘못된 방향이 비싼 작업 |

## 내부 흐름

1. explore — 현재 코드베이스 구조 파악
2. plan — 구현 경로 결정
3. execute — 코드 작성
4. QA (`$ultraqa`) — 테스트·검증·수정 사이클
5. verify — 완료 증거 수집

## `$autopilot` vs 계획 기반 흐름

`$autopilot`은 OMX가 흐름 대부분을 결정합니다. 단계를 손으로 제어하고 싶거나 요구사항을 먼저 확정해야 한다면 아래 순서가 더 안전합니다.

```
$deep-interview "무엇을 만들어야 하는지 정리해줘"
$ralplan "가장 안전한 계획을 세워줘"
$ralph "그 계획대로 끝까지 진행해줘"
```
