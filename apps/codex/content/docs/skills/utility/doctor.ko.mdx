`$doctor`로 호출합니다.

OMX 설치, 의존성, 환경 변수, 에이전트 연결 상태를 점검합니다. 뭔가 이상한데 어디가 문제인지 모를 때 첫 번째로 실행하는 스킬입니다.

## 언제 쓰나

- OMX 처음 설치 후 정상 동작 여부를 확인할 때
- 에이전트가 반응하지 않거나 예상과 다르게 동작할 때
- 업데이트 후 환경이 깨진 것 같을 때
- 지원 요청 전에 현재 상태 스냅샷을 뽑아야 할 때

## 명령

```bash
$doctor
$doctor --verbose
$doctor --fix
```

`--fix`는 자동으로 수정 가능한 문제를 수정합니다. 수정이 불가능하면 해결 방법을 안내합니다.

## 점검 항목

| 항목 | 설명 |
|---|---|
| OMX 버전 | 현재 설치된 버전과 최신 버전 비교 |
| Node.js 버전 | 요구 버전 충족 여부 |
| 환경 변수 | API 키, 설정 파일 유무 |
| 에이전트 연결 | Claude, Gemini 브리지 응답 여부 |
| MCP 서버 | 활성 MCP 서버 상태 |
| 훅 설정 | settings.json 훅 등록 여부 |

## 출력 읽는 법

- `OK` — 정상
- `WARN` — 동작은 하지만 권장 설정이 아님
- `ERROR` — 즉시 수정 필요

`ERROR`가 있으면 `$doctor --fix`를 먼저 시도하고, 해결이 안 되면 출력 결과를 첨부해 이슈를 제출하세요.
