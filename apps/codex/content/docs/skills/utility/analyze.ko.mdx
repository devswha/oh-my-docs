`$analyze`로 호출합니다.

코드베이스를 건드리지 않고 이해하는 데 집중하는 스킬입니다. 버그 원인 파악, 의존성 추적, 아키텍처 파악 등 "먼저 읽고 파악해야 할 때" 씁니다.

## 언제 쓰나

- 버그가 어디서 오는지 원인을 먼저 파악해야 할 때
- 코드 변경 전에 영향 범위를 확인해야 할 때
- 팀원에게 설명하거나 리뷰하기 전에 맥락을 정리해야 할 때
- 의심스러운 동작이 있는데 어디를 봐야 할지 모를 때

## 언제 피하나

- 수정이 목적이면 `$build-fix` 또는 executor 에이전트를 바로 쓰는 편이 빠름
- 계획 수립이 필요하면 `$ralplan`

## 명령

```bash
$analyze "왜 로그인 후 리다이렉트가 실패하는지 찾아줘"
$analyze "auth 모듈 의존성 전체 파악해줘"
$analyze --scope src/api "이 디렉터리에서 에러 처리 방식 정리해줘"
```

## 동작 방식

`$analyze`는 debugger 에이전트를 부릅니다. 코드를 직접 수정하지 않고, 파일 탐색 · 심볼 검색 · 참조 추적으로 원인과 구조를 정리한 뒤 결과를 텍스트로 돌려줍니다.

수정까지 한 번에 처리하고 싶으면 `$ralph`를 쓰세요.
