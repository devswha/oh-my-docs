## 概要

`$omx-setup` はOMX環境を初期化または更新します。スキルファイル、エージェントプロンプト、フック設定をインストールパスに合わせて配置します。初回インストール時とアップグレード後に設定を更新するときに使います。

CLIでは `omx setup` で同様に実行できます。

## 使用例

```text
$omx-setup
$omx-setup --scope user
$omx-setup --scope project
```

| オプション | 説明 |
|---|---|
| （なし） | デフォルトスコープにインストール |
| `--scope user` | `~/.codex/` にグローバルインストール |
| `--scope project` | `.codex/` にプロジェクトスコープでインストール |

## いつ使うか

- OMXを初めてインストールする場合
- `oh-my-codex` パッケージをアップグレードした後に設定を同期するとき
- 新しいマシンで既存環境と同じにそろえるとき
- スキルやフックが正しく動作しないときに再インストール

## いつ避けるか

- 環境異常の診断 — インストール問題が疑われる場合は `$omx-setup` の前に `$doctor` で確認してください。
