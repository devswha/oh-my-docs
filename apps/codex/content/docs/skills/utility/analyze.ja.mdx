`$analyze`で呼び出します。

コードベースを触らずに理解することに集中するスキルです。バグ原因の特定、依存関係の追跡、アーキテクチャの把握など「まず読んで把握すべきとき」に使います。

## いつ使うか

- バグがどこから来るのか原因を先に把握しなければならないとき
- コード変更前に影響範囲を確認する必要があるとき
- チームメンバーに説明したりレビューする前にコンテキストを整理したいとき
- 疑わしい動作があるがどこを見ればよいかわからないとき

## いつ避けるか

- 修正が目的なら `$build-fix` またはexecutorエージェントを直接使う方が早い
- 計画立案が必要なら `$ralplan`

## コマンド

```bash
$analyze "ログイン後リダイレクトが失敗する原因を探して"
$analyze "authモジュールの依存関係を全部把握して"
$analyze --scope src/api "このディレクトリのエラー処理方式を整理して"
```

## 動作方式

`$analyze`はdebuggerエージェントを呼び出します。コードを直接修正せず、ファイル探索・シンボル検索・参照追跡で原因と構造を整理し、結果をテキストで返します。

修正まで一度に処理したいなら `$ralph` を使ってください。
