## 개요

`$git-master`는 git 관련 작업을 `git-master` 에이전트에 위임합니다. 커밋 메시지 작성, 브랜치 관리, 히스토리 정리, PR 준비 같은 작업을 직접 하지 않고 에이전트가 처리하게 합니다.

## 언제 쓰는가

- 구현이 끝나고 커밋·PR을 만들어야 할 때
- 커밋 히스토리가 지저분해서 정리가 필요할 때
- 브랜치 전략이나 merge 방식을 결정해야 할 때
- 여러 변경을 논리적 단위로 나눠 커밋해야 할 때

## 언제 피하는가

- 단순한 `git add && git commit` 한 줄 — 직접 하는 게 빠릅니다.
- 진행 중인 작업을 잠시 저장하는 용도 — `git stash`를 직접 쓰세요.

## 사용 예시

```text
$git-master "지금까지 변경 사항을 기능 단위로 나눠 커밋해줘"
$git-master "PR 준비해줘. 커밋 메시지도 정리해줘"
$git-master "main에서 브랜치 따서 feature/auth로 시작해줘"
```
