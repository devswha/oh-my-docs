`$doctor`で呼び出します。

OMXのインストール、依存関係、環境変数、エージェント接続状態を点検します。何かおかしいがどこが問題かわからないとき、最初に実行するスキルです。

## いつ使うか

- OMX初回インストール後に正常動作を確認するとき
- エージェントが反応しないか期待と異なる動作をするとき
- アップデート後に環境が壊れているようなとき
- サポート依頼前に現在の状態スナップショットを取りたいとき

## コマンド

```bash
$doctor
$doctor --verbose
$doctor --fix
```

`--fix` は自動修正可能な問題を修正します。修正できない場合は解決方法を案内します。

## チェック項目

| 項目 | 説明 |
|---|---|
| OMXバージョン | 現在インストールされているバージョンと最新バージョンの比較 |
| Node.jsバージョン | 要求バージョンを満たしているか |
| 環境変数 | APIキー、設定ファイルの有無 |
| エージェント接続 | Claude、Geminiブリッジの応答有無 |
| MCPサーバー | アクティブなMCPサーバーの状態 |
| フック設定 | settings.jsonのフック登録有無 |

## 出力の読み方

- `OK` — 正常
- `WARN` — 動作はするが推奨設定ではない
- `ERROR` — 即時修正が必要

`ERROR` があれば `$doctor --fix` を先に試し、解決しなければ出力結果を添付してIssueを提出してください。
