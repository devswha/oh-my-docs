## 개요

`$ecomode`는 실행 중인 OMX 세션을 저비용 모드로 전환합니다. 무거운 멀티에이전트 파이프라인 대신 단일 에이전트로 작업하고, 가능한 곳에서 더 가벼운 모델을 씁니다. 탐색·분석보다 실행 비용이 신경 쓰일 때 켜는 스위치입니다.

## 언제 쓰는가

- 큰 작업이 이미 설계되었고 단순 실행만 남았을 때
- 병렬 에이전트가 필요 없는 반복적인 소규모 수정
- API 비용을 아껴야 하는 환경

## 언제 피하는가

- 설계나 아키텍처 판단이 필요한 작업 — ecomode는 분석 깊이를 줄이므로 적합하지 않습니다.
- 여러 시스템을 동시에 건드리는 복잡한 변경

## 사용 예시

```text
$ecomode
```

호출 후 이후 실행은 경량 에이전트와 단일 실행 경로를 기본으로 씁니다.

## 참고

ecomode는 현재 세션에만 적용됩니다. 세션을 재시작하면 기본 모드로 돌아옵니다.
