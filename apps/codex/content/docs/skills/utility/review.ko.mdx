## 개요

`$review`는 코드 리뷰를 `code-reviewer` 에이전트에 위임합니다. 로직 결함, 유지보수성, API 계약, 하위 호환성 같은 항목을 포괄적으로 검토합니다. 구현이 끝난 뒤 독립적인 리뷰 단계가 필요할 때 씁니다.

## 사용 예시

```text
$review
$review "auth 모듈 변경 사항 리뷰해줘"
$review "PR #42 전체 검토해줘"
```

인자 없이 호출하면 현재 변경 사항(git diff 기준)을 리뷰합니다.

## 언제 쓰는가

- `$executor`나 `$ralph`로 구현을 마친 뒤 독립 리뷰가 필요할 때
- PR을 열기 전 품질 체크 단계
- API 변경이 있어 하위 호환성을 확인해야 할 때
- 보안과 무관한 일반 코드 품질 검토

## 언제 피하는가

- 보안 취약점 중심 검토 — `$security-review`를 쓰세요.
- 빌드/타입 오류 수정 — `$build-fix`를 쓰세요.
- 구현 전 설계 검토 — `$ralplan`의 critic 단계가 더 적합합니다.
