`$code-review`로 호출합니다.

code-reviewer 에이전트를 호출하는 wrapper입니다. 구현이 끝난 코드에 대한 포괄적인 리뷰가 필요할 때 씁니다. API 계약, 하위 호환성, 로직 결함, 보안 취약점을 포함해 검토합니다.

## 언제 쓰나

- 구현이 끝난 코드를 PR 전에 검토할 때
- `$ralph`나 `$autopilot`이 생성한 코드를 확인할 때
- 특정 파일이나 변경 사항에 대한 품질 피드백이 필요할 때
- 하위 호환성이나 API 계약 준수를 확인해야 할 때

## 언제 피하나

- 보안 취약점만 집중적으로 보려면 `$security-review`가 더 적합
- 빌드 에러 수정이 목적이면 `$build-fix`
- 아직 구현 중인 코드에는 쓰지 않는 것이 좋음 — 완성된 이후에 돌리는 편이 효율적

## 명령

```bash
$code-review
$code-review "auth 모듈 변경 사항 리뷰해줘"
$code-review --scope src/api
$code-review --pr 123
```

## 검토 범위

| 항목 | 내용 |
|---|---|
| 로직 결함 | 잘못된 조건, 누락된 엣지 케이스 |
| API 계약 | 입출력 타입, 에러 처리 방식 |
| 하위 호환성 | 기존 호출자에 영향 있는 변경 |
| 유지보수성 | 중복, 불필요한 복잡도 |
| 보안 | 명백한 취약점 (심층 보안은 `$security-review`) |
