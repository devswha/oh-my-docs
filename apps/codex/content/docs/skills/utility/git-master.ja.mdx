## 概要

`$git-master` はgit関連作業を `git-master` エージェントに委任します。コミットメッセージの作成、ブランチ管理、ヒストリー整理、PR準備などの作業を自分でやらずエージェントに処理させます。

## いつ使うか

- 実装が終わってコミット・PRを作らなければならないとき
- コミットヒストリーが乱雑で整理が必要なとき
- ブランチ戦略やマージ方式を決めなければならないとき
- 複数の変更を論理的な単位に分けてコミットしなければならないとき

## いつ避けるか

- 単純な `git add && git commit` 一行 — 直接やる方が早いです。
- 進行中の作業を一時保存する用途 — `git stash` を直接使ってください。

## 使用例

```text
$git-master "今までの変更を機能単位に分けてコミットして"
$git-master "PR準備して。コミットメッセージも整理して"
$git-master "mainからブランチを切ってfeature/authで始めて"
```
