Model Context Protocol (MCP) is an open standard that lets AI clients call external tools through a common interface. OMX acts as an MCP client, invoking registered MCP servers from within skills, hooks, and agent workflows to reach file systems, APIs, and specialized services.

## What it is

MCP is a client-server protocol where the MCP client (OMX/Codex) sends structured tool-call requests and receives structured responses. Each MCP server exposes a named set of tools. OMX discovers configured servers at startup and calls their tools directly from within skills and hooks.

## How OMX uses it

- **Skill-level tool calls** — Skills like `$wiki`, `$trace`, and `$team` invoke MCP tools (`wiki_add`, `state_read`, `TeamCreate`, etc.) during normal execution.
- **Hook-level integration** — Hook scripts call MCP tools to read or write state before and after OMX operations.
- **Supported servers** — OMX recognizes the following servers out of the box:
  - `filesystem` — read and write local files
  - `github` — issues, PRs, code search
  - `context7` — library documentation lookup
  - `oh-my-claudecode_t` — the OMX plugin server, which exposes state, notepad, wiki, LSP, AST, and team tools
- **Configuration** — MCP servers are registered under the `mcpServers` key in the Claude Code settings file (`~/.claude/settings.json`). Each entry specifies a `command`, optional `args`, and optional `env` overrides.

## Example

Registering the filesystem and GitHub MCP servers:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/home/user/projects"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}" }
    },
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp"]
    }
  }
}
```

Once registered, tools from those servers are callable directly inside skills:

```
codex
> $wiki add "the DB schema lives in src/db/schema.ts"
```
