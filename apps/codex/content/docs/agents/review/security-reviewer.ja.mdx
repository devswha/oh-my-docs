`security-reviewer` は OWASP Top 10 を基準としてコードを評価し、ハードコードされたシークレットをスキャンし、依存関係を監査し、変更箇所のすべての trust boundary を点検します。発見事項は深刻度 × 悪用可能性 × 被害範囲の積で優先順位を付けます。脆弱なコードと同じ言語で安全なコード例を提供し、修正が抽象的なアドバイスにとどまらないようにします。

## 役割

- レビュー対象コードに適用可能な OWASP Top 10 項目を全件評価
- シークレットスキャン（API キー、パスワード、トークン）および依存関係監査（`npm audit`、`pip-audit`、`cargo audit`）を実行
- 認証、認可、入力検証、インジェクションベクター、機密データの取り扱い方式を評価
- 深刻度 × 悪用可能性 × 被害範囲の基準で発見事項を優先順位付けし、言語に合った修正例を提供

## いつ呼ばれるか

- `$security-review` で専任セキュリティパスが要求されるとき
- 認証、認可、API エンドポイント、データベースクエリを追加または変更する PR
- `code-reviewer` パスで潜在的な脆弱性が発見され、深層分析が必要なとき
- 決済データ、PII、ファイルアップロード、セッション管理を扱う変更をマージする前

## 入力

- Read・Grep でアクセスしたレビュー対象ソースファイル
- 過去のコミットでシークレットが誤って含まれていないか確認するための git ヒストリー
- 自動監査ツール実行のための依存関係マニフェスト

## 出力

- リスクレベル（HIGH / MEDIUM / LOW）、深刻度別発見件数、発見事項ごとの詳細（場所 `file:line`、OWASP カテゴリ、悪用可能性、被害範囲、安全なコード例）を含むセキュリティレビューレポート
- シークレット、インジェクション、認証、認可、依存関係の各項目を点検済みと確認するセキュリティチェックリスト

## 範囲の限界

- 読み取り専用 — 修正は実装しません。発見事項は `executor` に渡して修正します
- コード品質やスタイルの問題をセキュリティ発見事項と混在させません
- 未解決の CRITICAL・HIGH セキュリティ発見事項があるコードは承認しません
