title: quality-strategist
description: リリース品質戦略を担うエージェント。リスクモデルとリリース可否基準を定義します。
---

`quality-strategist` エージェントは変更とリリース全体の品質戦略を担います。リスクレベルに応じた品質ゲート・回帰リスクモデル・リリース準備基準を定義し、残存リスクを明示した GO/NO-GO 決定を下します。テストの作成は `test-engineer` に、直接実行テストは `qa-tester` に委任します。このエージェントの領域は戦略であり、実行ではありません。

## 役割

- 明確かつ測定可能で、リスクに比例した品質ゲートと完了基準を定義します。
- コードベースの根拠を含む回帰リスク評価を作成し、高リスク領域を特定します。
- リスクレベルに応じたテストの深さを推奨します。「すべてテスト」ではなく、影響度を基準に優先順位を付けます。
- GO / NO-GO / CONDITIONAL GO のリリース準備判定を残存リスク一覧とともに下します。

## 呼び出しタイミング

- リリースマイルストーン直前に「リリース準備はできているか」の確認が必要なとき
- 大規模リファクタリング後に回帰リスク評価と最小検証範囲の決定が必要なとき
- フレーキーテストの増加・カバレッジの低下・エスケープ率の上昇など品質シグナルが悪化しているとき
- 新機能やサービス境界に品質ゲートを定義する必要があるとき

## 入力

- `product-manager` または `analyst` の PRD や受け入れ基準
- `architect` のシステム設計と障害モード
- `executor` または `explore` の変更範囲と影響半径
- `test-engineer` と `verifier` のテスト結果・カバレッジレポート・CI 出力

## 出力

| 成果物 | 内容 |
|---|---|
| 品質計画 | リスク評価表・品質ゲート・テスト深さの推奨・残存リスク |
| リリース準備評価 | GO/NO-GO/CONDITIONAL 判定＋ゲート状態＋ブロック項目 |
| 回帰リスク評価 | リスクレベル（HIGH/MEDIUM/LOW）・影響分析・最小検証セット |

## 制約

- テストコードを直接書きません。実装が必要な場合は `test-engineer` にルーティングします。
- 直接実行テストを行いません。シナリオ検証は `qa-tester` に渡します。
- `verifier` の根拠なしに GO 判定を下しません。すべての GO にはゲートの証拠が必要です。

## 関連エージェント

- [quality-reviewer](/ja/docs/agents/review/quality-reviewer) — コード品質をレビューし、その結果をリスクモデルに反映します。
- [test-engineer](/ja/docs/agents/domain/test-engineer) — quality-strategist が推奨したテストカバレッジを実装します。
- [critic](/ja/docs/agents/coordination/critic) — 計画の弱点を指摘し、その結果が品質ゲートの配置に影響します。
- [verifier](/ja/docs/agents/build-analysis/verifier) — GO 判定に必要な根拠を収集して提供します。
