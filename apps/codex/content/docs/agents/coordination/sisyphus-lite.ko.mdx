`sisyphus-lite` 에이전트는 작업 대기열에서 다음 항목을 꺼내 최소한의 오버헤드로 처리하는 경량 실행기입니다. `$ralph`와 같은 지속성을 가지되 검증 절차를 간소화한 버전입니다. 범위가 명확하고 잘 정의된 작업에서 무거운 검증 게이트가 오히려 속도를 늦출 때 사용합니다.

## 역할

- 대기열에서 작업을 하나씩 가져와 검증된 결과를 낸 뒤 다음 작업을 가져옵니다.
- 기본적으로 낮은 추론 노력을 적용합니다. 위험하거나 여러 파일에 걸친 작업에서만 높입니다.
- 과도한 계획, 불필요한 에스컬레이션, 긴 설명 없이 바로 실행합니다.
- 진단과 간략한 결과 요약으로 최소한의 증거를 확인합니다.

## 호출 시점

- `$worker` 모드에서 범위가 한정된 작업 대기열을 지속적으로 처리할 때
- `$ralph` lite 모드에서 전체 ralph 검증 루프가 작업에 비해 과도한 절차일 때
- `$team` 파이프라인 내에서 단순하고 병렬화 가능한 lane을 담당하는 경량 워커로
- 작업 목록을 단계마다 수동으로 재호출하지 않고 지속적으로 처리하고 싶을 때

## 입력

- 작업 대기열 또는 한정된 작업 목록 (파일 경로, 설명, 수락 기준 포함)
- 작업별로 필요한 저장소 소스 (Glob/Grep/Read로 접근)
- 선택 사항: 미완료 작업 위치를 나타내는 `.omx/state/` 세션 컨텍스트

## 출력

- 각 작업별 완료된 변경 사항 (진단 또는 테스트 출력으로 검증)
- 파일 경로와 근거가 포함된 작업별 완료 요약
- 완료 항목이 반영된 업데이트된 대기열 상태

## 제약

- 아키텍처 판단, 깊은 분석, 시스템 간 설계 결정이 필요한 작업에는 적합하지 않습니다.
- lite 모드에서도 검증은 생략하지 않습니다. "증거 없음 = 미완료"입니다.
- 전문가 도움이 명확하게 필요한 경우에만 에스컬레이션합니다. 한정된 작업은 직접 처리합니다.

## 관련 에이전트

- [executor](/ko/docs/agents/build-analysis/executor) — 복잡하거나 여러 파일에 걸친 작업을 위한 전체 기능 구현 에이전트
- [team-executor](/ko/docs/agents/coordination/team-executor) — 오케스트레이터 메일박스와 연동하는 팀 인식 실행 워커
- [verifier](/ko/docs/agents/build-analysis/verifier) — 더 무거운 작업 이후 독립적으로 증거를 검증하는 에이전트
