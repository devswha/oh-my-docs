---

`code-simplifier` 负责在功能开发完成后整理代码。移除死路径，将单一用途的抽象展开为直接逻辑，将名称改为符合实际行为的形式，将复杂的条件语句重组为易读的顺序。代码的行为不会发生变化。

## 职责

- 移除死代码 — 不可达分支、未使用变量、过时的 feature flag、被注释掉的代码块
- 将单一用途的抽象转换为直接且易读的逻辑
- 将偏离原始意图的符号名称改为反映实际行为的名称
- 将复杂的条件语句和深度嵌套的逻辑重组为扁平的顺序

## 何时被调用

- 功能上线后，实现中积累了不再有必要存在的复杂性时
- 代码审查中有功能正确但难以理解的部分被指出时
- 团队在 sprint 后进行整理 pass 以管理技术债积累时
- 新成员入职时发现核心模块需要过多解释时

## 输入

提供需要整理的文件或模块、具体的关注点（死代码、过度抽象、命名、嵌套）、约束条件（不能变更的公开 API、需要保持的性能特性）。说明是哪种认知负担问题触发了这次整理，有助于引导重点所在。

## 输出

整理后的代码，附带说明每个变换为何能减少复杂性的变更摘要。现有行为契约保持不变 — 现有测试无需修改即可通过。还包括以前后对比形式呈现最大改进点的说明。

## 范围限制

- 不改变行为。若发生行为变更，那是 bug，不是简化。
- 不添加未被请求的新抽象或工具函数。简化是减少代码，而不是用其他代码替换。
- 不做安全或性能优化。转交给 `security-reviewer` 和 `quality-reviewer`。
