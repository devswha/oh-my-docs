---

`code-simplifier` は機能作業が終わった後にコードを整理する役割です。デッドパスを除去し、単一用途の抽象化を直接ロジックに展開し、名前を実際の動作に合わせて変更し、複雑な条件文を読みやすい順序に再構成します。コードが行うことは変わりません。

## 役割

- デッドコード除去 — 到達不可能な分岐、未使用変数、古くなったフィーチャーフラグ、コメントアウトされたブロック
- 単一用途の抽象化を直接的で読みやすいロジックに変換
- 本来の意図からずれたシンボル名を実際の動作を反映するように変更
- 複雑な条件文と深くネストしたロジックを平坦な順序に再構成

## いつ呼ばれるか

- 機能がデプロイされた後、もはや存在する必要のない複雑さが実装に蓄積されたとき
- コードレビューで機能的には正しいが追いにくいと指摘されたセクションがあるとき
- チームが技術的負債の蓄積を管理するためにスプリント後の整理パスを実施するとき
- 新規開発者オンボーディング中にコアモジュールに過剰な説明が必要だと判明したとき

## 入力

整理対象のファイルまたはモジュール、具体的な懸念事項（デッドコード、過剰な抽象化、命名、ネスト）、制約条件（変更してはいけない公開 API、維持すべきパフォーマンス特性）を提供します。どの認知負荷の問題がきっかけになったか説明すると、どこに集中するかのガイドになります。

## 出力

各変換が複雑さをなぜ減らすかを説明する変更サマリーとともに整理されたコード。既存の動作コントラクトはそのまま — 既存テストが修正なしでパスします。最大の改善箇所を前後で示すノートも含みます。

## 範囲の限界

- 動作は変えません。動作変更が生じた場合、それは単純化ではなくバグです。
- 要求されていない新しい抽象化やユーティリティは追加しません。単純化はコードを減らすことであり、別のコードに置き換えることではありません。
- セキュリティやパフォーマンスの最適化は行いません。`security-reviewer` と `quality-reviewer` に渡します。
