## 概要

`researcher` エージェントは外部ドキュメントとリファレンスを調査します。SDK の使い方、ライブラリ API 仕様、サービスの制約条件のようにコードベース内にない情報が必要なときに呼び出します。実装前に正確な情報を確保する役割のため、`executor` が推測でコードを書く状況を減らします。

## いつ呼ばれるか

- 初めて使う SDK やライブラリの正確な使い方を確認しなければならないとき
- 外部サービス API の制約条件、rate limit、認証方式を把握しなければならないとき
- 最新ドキュメントとトレーニングデータ基準の情報が異なる可能性があり、公式リファレンスを確認しなければならないとき
- `$external-context` や `$document-specialist` を通じて並列ドキュメント調査が必要なとき

## 使用例

```text
"Stripe Webhooks の検証方法を公式ドキュメント基準で調査してほしい"
"Next.js App Router での middleware 設定方法を調べてほしい"
"このライブラリ v3 から v4 へのマイグレーション方法を調査してほしい"
```

## 調査範囲

| 項目 | 内容 |
|---|---|
| SDK / ライブラリ | 公式リファレンス、関数シグネチャ、設定オプション |
| 外部サービス API | エンドポイント仕様、認証方式、制約条件 |
| マイグレーションガイド | バージョン変更点、breaking change、アップグレード手順 |
| ベストプラクティス | 公式推奨パターン、既知の注意事項 |

## 作業プロセス

1. 何を知る必要があるか調査対象を明確にします。
2. 公式ドキュメントやリファレンスを優先して確認します。
3. 必要に応じて context7 などのドキュメントツールを活用します。
4. 調査結果を要約し、実装にすぐ使える形で整理します。

## 入力

- 調査するライブラリ、SDK、サービス名
- 具体的に確認すべき API や機能
- 実装で詰まっている部分や不確かな部分

## 出力

- 公式ドキュメントに基づく正確な使い方の要約
- 関数シグネチャ、設定オプション、サンプルコード
- 既知の制約条件や注意事項
- 参考ソースリンク

## 範囲の限界

- 直接コードを書きません。情報を提供し `executor` が実装します。
- 公式ドキュメントがない場合は不確実性を明示します。
- 内部コードベースの探索は `explore` が担当します。

## 関連エージェント
