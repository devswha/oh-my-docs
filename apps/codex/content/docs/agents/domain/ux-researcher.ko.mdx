## 개요

`ux-researcher` 에이전트는 사용자 관점에서 인터페이스와 상호작용을 분석합니다. 사용성 문제를 찾고, 사용자 시나리오를 정리하고, 개선 방향의 근거를 만드는 역할입니다. `designer`가 시각적 설계를 담당한다면, `ux-researcher`는 그 설계 결정의 배경이 되는 사용자 데이터와 시나리오를 제공합니다.

## 언제 쓰는가

- UI 변경 전에 현재 인터페이스의 사용성 문제를 파악해야 할 때
- 사용자 시나리오와 페르소나를 정리해서 설계 방향을 잡아야 할 때
- 기능 변경이 실제 사용 흐름에 미치는 영향을 분석해야 할 때
- `designer`에게 넘기기 전에 UX 요구사항을 정리해야 할 때

## 사용 예시

```text
"현재 온보딩 플로우에서 사용성 문제를 찾아줘"
"이 기능의 주요 사용자 시나리오를 정리해줘"
"변경된 설정 화면이 기존 사용 흐름에 어떤 영향을 주는지 분석해줘"
```

## 분석 범위

| 항목 | 내용 |
|---|---|
| 사용성 분석 | 인터페이스의 마찰 지점, 혼란 요소 파악 |
| 사용자 시나리오 | 주요 사용 흐름과 엣지 케이스 정리 |
| 페르소나 | 대표 사용자 유형과 목표 정의 |
| 영향 분석 | 변경이 기존 사용 패턴에 미치는 영향 |

## 작업 프로세스

1. 분석 대상 화면이나 흐름을 파악합니다.
2. 사용자 관점에서 주요 시나리오와 문제 지점을 정리합니다.
3. 개선 방향의 근거를 만들어 `designer`나 `product-manager`에 넘깁니다.
4. 변경 후 사용 흐름에 미친 영향을 다시 검토합니다.

## 입력

- 분석 대상 UI나 기능 설명
- 기존 사용자 피드백이나 지표 (있는 경우)
- 제품 목표와 대상 사용자 정보

## 출력

- 사용성 문제 목록과 우선순위
- 주요 사용자 시나리오 및 페르소나 문서
- 개선 방향 근거 요약
- `designer`나 `product-manager`가 쓸 수 있는 UX 요구사항

## 제한 사항

- 시각 설계와 컴포넌트 레이아웃은 `designer`가 담당합니다.
- 실제 사용자 데이터 수집은 ux-researcher 범위 밖입니다.
- 구현은 `executor`가 합니다.

## 관련 에이전트

- [designer](/ko/docs/agents/domain/designer) — ux-researcher가 정리한 UX 요구사항을 시각 설계로 옮깁니다.
- [product-manager](/ko/docs/agents/domain/product-manager) — UX 조사 결과를 제품 의사결정에 반영합니다.
- [analyst](/ko/docs/agents/build-analysis/analyst) — UX 요구사항을 구체적인 수락 기준으로 변환합니다.
