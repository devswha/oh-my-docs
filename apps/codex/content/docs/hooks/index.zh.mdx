## 什么是钩子

钩子是响应 Codex CLI 生命周期事件自动执行的运行时代码。OMX 运行在钩子之上 — 用户不需要每次都记住内部命令，skill 就能自然地激活，状态也能在会话之间延续。

这不是用户直接调用的功能，而是**在背后驱动 OMX 的事件层**。

## 没有钩子会怎样

没有钩子，OMX 实质上就接近一个提示词集合。有了钩子才能：

- 从用户输入中检测魔法关键词
- 自动激活 skill
- 在工具执行前后设置守卫
- 恢复长期运行模式的状态并继续
- 将运行时状态保存到 `.omx/`

## 钩子介入的时机

| 时机 | OMX 做的事 |
|---|---|
| 会话开始 | 状态恢复、上下文注入、初始检查 |
| 提交提示词 | 魔法关键词检测、skill 激活 |
| 工具执行前 | 危险命令守卫、预验证 |
| 工具执行后 | 显示失败提示、后处理 |
| 模型停止 | 判断是否继续未完成的执行 |
