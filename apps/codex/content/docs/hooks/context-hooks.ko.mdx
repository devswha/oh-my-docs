컨텍스트 훅은 다음 프롬프트에 구조화된 지식을 끼워 넣는 OMX 훅의 하위 집합입니다. 사용자 메시지를 직접 수정하는 대신 Codex가 읽는 `<system-reminder>` 태그를 발행합니다. 덕분에 사용자 메시지는 그대로 유지되면서, 모델은 항상 최신 프로젝트 메모리와 활성 지시사항을 갖고 시작합니다.

## 주입되는 소스

**프로젝트 메모리**

`project_memory_write` 또는 `project_memory_add_directive`로 작성된 핵심 사실과 지시사항. `SessionStart`에 주입되고, 지시사항이 활성 상태면 이후 턴에서도 선택적으로 주입됩니다.

**세션 리마인더**

`notepad_write_working` 또는 `notepad_write_priority`로 작성된 단기 메모. 만료되거나 정리될 때까지 계속 주입됩니다.

**위키 요약**

위키 페이지가 있으면 `SessionStart`에 `.omx/wiki/`의 간략한 발췌를 표시합니다. 수집·재빌드 같은 쓰기 집약적 작업은 훅 경로를 막지 않도록 지연 처리합니다.

**모드 상태**

ralph, team, ultrawork 같은 활성 워크플로우 모드가 감지되면 현재 단계와 계속 지시사항을 함께 주입해, 모델이 중단 지점부터 올바르게 재개할 수 있습니다.

## 사용 예시

모든 커밋에 트레일러를 포함하라는 영구 지시사항 등록:

```bash
omx memory add-directive "All git commits must include Co-authored-by: OmX <omx@oh-my-codex.dev>"
```

다음 세션 시작 시 이 지시사항이 `<system-reminder>` 블록으로 자동 주입됩니다.

현재 notepad 내용 확인:

```bash
cat .omx/notepad.md
```
