Codex CLI 在会话流程的固定时间点发布事件。OMX 钩子绑定在这些事件上运行。了解哪些事件在何时发生，有助于追踪 OMX 为何在该时间点介入，并将 `.omx/hooks/*.mjs` 插件挂载到正确的位置。

## 事件列表

| 事件 | 发生时机 | OMX 所做的事 |
|---|---|---|
| `SessionStart` | Codex 会话初始化 | 恢复状态、填充 wiki 摘要、确认 `.omx/` 已加入 gitignore |
| `UserPromptSubmit` | 用户发送提示时 | 关键词检测、技能路由、deep-interview 输入锁定 |
| `PreToolUse` | 工具调用执行前 | Bash 安全检查、内联 `git commit` 格式守卫 |
| `PostToolUse` | 工具调用完成后 | 命令未找到指导、异常退出输出审查 |
| `Stop` | 模型完成一轮响应时 | ralph / autopilot / ultrawork / ultraqa 继续运行、停滞状态自动推进 |
| `PreCompact` | 上下文窗口接近限制时 | 保留中 — wiki 压缩延至 v2 |
| `Notification` | 系统通知发出时 | 通过 notify-hook 将通知路由到 Discord / Slack / Telegram |

`PreToolUse` 和 `PostToolUse` 目前只拦截 Bash 调用。非 Bash 工具通过运行时回退路径处理。完整的覆盖矩阵请参阅 [Codex 原生钩子](/zh/docs/hooks/codex-native-hooks)。

## 在插件中订阅事件

接收 `post-tool-use` 事件的插件示例：

```js
// .omx/hooks/my-plugin.mjs
export async function onHookEvent(event, sdk) {
  if (event.event !== 'post-tool-use') return;
  sdk.log.info('tool finished', event.context);
}
```

事件信封始终包含 `schema_version`、`event`、`timestamp`、`source`（`native` 或 `derived`）和 `context`。
