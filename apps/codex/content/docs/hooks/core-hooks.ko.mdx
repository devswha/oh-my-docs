코어 훅은 `omx setup` 실행 시 자동으로 설치되는 훅 핸들러입니다. 별도 설정 없이 OMX 워크플로우 스킬을 Codex 라이프사이클 이벤트에 연결합니다. 각 훅은 단일 책임만 담당하고, 해당 관심사가 비활성 상태면 아무것도 하지 않습니다.

## 구조

4개 훅 모두 `.codex/hooks.json`에 등록되고, `dist/scripts/codex-native-hook.js` 하나를 진입점으로 씁니다. 이 진입점이 Codex 이벤트 유형을 보고 올바른 핸들러로 디스패치합니다. 활성화 조건은 `.codex/config.toml`의 `[features] codex_hooks = true`입니다.

`.omx/hooks/*.mjs` 플러그인 훅은 이 인터페이스를 확장하지만 대체하지는 않습니다.

## 내장 훅 4개

**keyword-detector** (`UserPromptSubmit`)

모든 사용자 프롬프트에서 매직 키워드 구문을 스캔합니다. 매치되면 `.omx/state/`에 스킬 활성화 상태를 기록하고, 필요하면 개발자 컨텍스트를 앞에 붙입니다. `$skill` 명시적 호출과 "team", "parallel" 같은 암묵적 구문 매칭을 모두 지원하며, 우발적 산문에서 발동하지 않도록 의도 가드가 붙어 있습니다. 자세한 내용은 [매직 키워드](/ko/docs/hooks/magic-keywords)를 참고하세요.

**ralph stop-gate** (`Stop`)

모델이 멈출 때 ralph, autopilot, ultrawork, ultraqa 실행이 아직 활성 상태인지 확인합니다. 활성 상태이고 종료 조건이 아니면 `decision: "block"`을 반환해 계속 실행을 지시합니다. 상태는 `.omx/state/{mode}-state.json`에서 읽습니다.

**pre-tool-use guard** (`PreToolUse`, Bash 전용)

파괴적인 명령(`rm -rf dist` 등)에 경고하고, 필수 트레일러(`Co-authored-by: OmX <omx@oh-my-codex.dev>`)가 없는 인라인 `git commit` 호출을 차단합니다.

**post-tool-use advisor** (`PostToolUse`, Bash 전용)

비정상 종료 코드, 권한 거부 오류, 경로 미발견 메시지를 해석하고 다음 프롬프트 컨텍스트에 수정 안내를 주입합니다.

## 상태 확인

훅이 제대로 설치됐는지 확인:

```bash
omx doctor
```

키워드 감지 후 훅 상태 직접 확인:

```bash
cat .omx/state/skill-active-state.json
```
