## フックとは

フックは Codex CLI のライフサイクルイベントに反応して自動実行されるランタイムコードです。OMX はフックの上で動いています — ユーザーが毎回内部コマンドを覚えなくても、スキルが自然にアクティブになり、状態がセッション間で引き継がれます。

ユーザーが直接呼び出す機能ではなく、**OMX を裏で動かすイベントレイヤー**です。

## フックがなければ

フックなしでは OMX は事実上プロンプト集に近いものになります。フックがあるから:

- ユーザー入力からマジックキーワードを検出し
- スキルを自動アクティブ化し
- ツール実行の前後にガードをかけ
- 長期実行モードの状態を復元して続行し
- `.omx/` にランタイム状態を保存

できます。

## フックが介入するタイミング

| タイミング | OMX がすること |
|---|---|
| セッション開始 | 状態復元、コンテキスト注入、初期チェック |
| プロンプト送信 | マジックキーワード検出、スキルアクティブ化 |
| ツール実行前 | 危険コマンドのガード、事前検証 |
| ツール実行後 | 失敗ヒントの表面化、後処理 |
| モデル停止 | 未完了の実行を続けるか判断 |

## いつこのセクションを先に見るか

- なぜ特定のスキルが自動的にオンになったか気になるとき
- 実行がなぜ止まったか、またはなぜ続いたか追跡するとき

- `.omx/` の状態とランタイム動作の繋がりを理解したいとき
