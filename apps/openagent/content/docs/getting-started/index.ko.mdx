OmO는 같은 upstream 제품에서 두 edition으로 배포됩니다. OpenCode 전체 하네스가 필요하면 **Ultimate**, Codex CLI plugin bundle이 필요하면 LazyCodex가 설치하는 **Light**를 쓰세요.

## 사전 요구사항

| 항목 | 요구사항 |
|---|---|
| Harness | Ultimate는 OpenCode, Light는 OpenAI Codex CLI, 필요하면 둘 다. |
| Runtime | Ultimate는 `bunx`, Codex Light는 `npx lazycodex-ai install`. |
| Provider | Ultimate는 가진 provider 일부면 됩니다: Claude, OpenAI/ChatGPT, Gemini, GitHub Copilot, OpenCode Go/Zen, Kimi for Coding, Z.ai, Vercel AI Gateway. Light는 provider 구독 질문을 하지 않습니다. |
| OS | macOS ARM64/x64, Linux x64/ARM64(musl/Alpine 포함), Windows x64. |

## Edition 선택

| 원하는 것 | 실행 | 디스크에 생기는 것 |
|---|---|---|
| OpenCode용 Ultimate | `bunx oh-my-openagent install` | OpenCode plugin 등록, agent/model config, provider auth 안내. |
| Codex CLI용 Light | `npx lazycodex-ai install` | 관리되는 `~/.codex/plugins/cache/sisyphuslabs/omo/`, Codex marketplace/plugin 설정, 선택적 autonomous 권한, component CLI, plugin-scoped MCP. |
| 둘 다 | `bunx oh-my-openagent install --platform=both` | 두 설치 경로 모두. |

`lazycodex-ai`는 Codex Light edition용 공식 npm installer입니다. Codex marketplace 이름은 아닙니다. Codex 안에서는 marketplace `sisyphuslabs`, plugin `omo@sisyphuslabs`로 보입니다.

## 설치

<Steps>

<Step>

### Ultimate: OpenCode

```bash
bunx oh-my-openagent install
```

installer는 모델 구독을 묻고, OpenCode plugin을 등록하고, config 파일을 쓰고, 선택한 provider의 인증 안내를 띄웁니다. rename 전환기라 legacy package/CLI 이름 `oh-my-opencode`도 동작하지만, 새 문서와 설정은 `oh-my-openagent`를 기준으로 쓰는 편이 안전합니다.

</Step>

<Step>

### Ultimate: 비대화형 설치

에이전트에게 설치를 맡길 때는 flag를 직접 넘기세요.

```bash
bunx oh-my-openagent install --no-tui \
  --claude=yes \
  --openai=yes \
  --gemini=no \
  --copilot=no
```

| Flag | 의미 |
|---|---|
| `--claude=no\|yes\|max20` | Claude 구독. `max20`은 Claude Max 20x. |
| `--openai=no\|yes` | Oracle 같은 GPT 기반 에이전트용 ChatGPT/OpenAI. |
| `--gemini=no\|yes` | Gemini. |
| `--copilot=no\|yes` | GitHub Copilot. |
| `--opencode-go=no\|yes` | OpenCode Go (GLM/Kimi/MiniMax). |
| `--opencode-zen=no\|yes` | `opencode/` model catalog. |
| `--zai-coding-plan=no\|yes` | Z.ai Coding Plan. |
| `--kimi-for-coding=no\|yes` | Kimi for Coding. |
| `--vercel-ai-gateway=no\|yes` | Vercel AI Gateway. |
| `--skip-auth` | 인증 안내 건너뛰기. |

Provider 선택은 에이전트별입니다. OmO는 전역 priority 하나로 움직이지 않습니다.

</Step>

<Step>

### Light: Codex CLI

```bash
npx lazycodex-ai install
# non-interactive recommended mode:
npx lazycodex-ai install --no-tui --codex-autonomous
```

`--codex-autonomous`는 Codex를 agent식 자율 실행에 맞춰 설정합니다: `approval_policy = "never"`, `sandbox_mode = "danger-full-access"`, `network_access = "enabled"`. 권한 설정을 건드리지 않으려면 `--no-codex-autonomous`를 쓰세요.

</Step>

<Step>

### Codex marketplace 경로

marketplace 경로는 실험적이고 추가적인 설치 방법입니다. 기본 지원 경로는 여전히 `npx lazycodex-ai install`입니다.

```bash
codex plugin marketplace add https://github.com/code-yeongyu/lazycodex
codex plugin add omo@sisyphuslabs
```

Codex를 다시 시작해 새 hook을 검토/승인하고, background bootstrap notice가 끝난 뒤 한 번 더 재시작하세요.

</Step>

<Step>

### Provider 인증

```bash
opencode auth login
```

이 단계는 Ultimate/OpenCode용입니다. Gemini Antigravity OAuth는 upstream `opencode-antigravity-auth` plugin을 설치해서 씁니다. GitHub Copilot, Z.ai, OpenCode Go, OpenCode Zen, Kimi는 installer에서 넘긴 provider flag에 맞춰 기본값이 잡힙니다.

</Step>

<Step>

### 검증

```bash
opencode --version
bunx oh-my-openagent doctor
npx lazycodex-ai doctor
```

앞의 두 명령은 Ultimate, `npx lazycodex-ai doctor`는 Light 확인용입니다. `doctor`는 네 그룹을 봅니다.

| 그룹 | 확인 내용 |
|---|---|
| System | OpenCode binary, version, plugin 등록, legacy 이름 경고. |
| Config | JSONC parsing, schema validation, 파일 감지. |
| Tools | AST-Grep, LSP server, GitHub CLI, MCP server. |
| Models | capability cache, normalization, agent/category override, fallback availability. |

뭔가 어긋나 보이면 `doctor --verbose`로 한 번 더 보세요.

</Step>

<Step>

### 첫 작업

OpenCode 세션에서 입력하세요.

```bash
ulw fix the failing login flow and verify it in the browser
```

정확도가 먼저라면 실행 전에 계획부터 잠그세요.

```bash
@plan refactor the auth module without behavior changes
/start-work
```

GPT-native deep work가 필요하면 agent selector에서 Hephaestus를 골라 절차가 아니라 목표를 주세요.

Codex Light를 설치한 뒤에는 Codex 안에서 `ultrawork`, `$ulw-loop`, 또는 `omo@sisyphuslabs` marketplace bundle이 설치한 Codex plugin skill을 쓰면 됩니다.

</Step>

</Steps>

## Telemetry

익명 active-install telemetry는 기본 ON입니다. UTC 하루에 한 번, hashed installation id만 보내고 PostHog person profile은 만들지 않습니다. 끄려면 둘 중 하나입니다.

```bash
OMO_SEND_ANONYMOUS_TELEMETRY=0
OMO_DISABLE_POSTHOG=1
OMO_CODEX_SEND_ANONYMOUS_TELEMETRY=0
OMO_CODEX_DISABLE_POSTHOG=1
```
