## 变更日志

Oh My ClaudeCode 各个版本的主要变更摘要。

### v4.12.0（当前版本）

**新特性**
- Team：按角色的供应商和模型路由，支持已解析的路由快照 — 在 `.claude/omc.jsonc` 中按规范角色配置 `roleRouting`（如 `codex critic`、`gemini reviewer`）(#2614)
- HUD：显示额外的使用花费数据（关闭 #2570）
- HUD：添加 MiniMax 编码计划使用供应商
- HUD：按供应商拆分使用缓存以消除跨会话抖动
- Release：将 release 技能重写为通用仓库感知的发布助手 — 分析仓库 + CI，缓存规则到 `.omc/RELEASE_RULE.md`，然后指导你完成发布过程（#2501）
- CI：添加升级测试以捕获警告和错误

**缺陷修复（精选）**
- 持久 stop 钩子：对齐合约并收紧智能体输出合约（#2653）
- Context 膨胀：消除重复规则/技能注入的三个源（#2577）
- Pre-tool-enforcer：通过集中式状态根解析器尊重 `OMC_STATE_DIR`（#2621）
- Ralplan：在自动启动前需要调用意图（#2620）
- Team：缩短触发消息以适应 `sendToWorker` 200 字符限制（#2617）
- Installer：在更新期间使用 OMC 风格的 frontmatter 保留用户技能（#2573）
- Permission-handler：允许只读 `gh issue`/`pr` 命令；添加 installer lib 断言
- Deep-interview：替换五个剩余的硬编码 20%/0.2 阈值信号（#2545）
- State-root：跨钩子入口点集中化 `OMC_STATE_DIR` 解析（#2532）
- Keyword-detector：忽略粘贴的成绩单块；抑制 review-seed 回显（#2541）
- Session-start：将陈旧的 `CLAUDE_PLUGIN_ROOT` 符号链接到最新版本
- Doctor：针对插件缓存检测 `CLAUDE.md` 版本漂移
- Tmux：通过包装函数集中化所有 tmux 执行

**重构**
- HUD：按单位而不是重置时间分类 z.ai `TOKENS_LIMIT`
- MCP：清理 standalone-server 和 `listtools` 测试

**统计：** 合并 121 个 PR · 6 个新特性 · 67 个缺陷修复 · 3 个其他变更

### v4.8.0

**智能体系统**
- 19 个专业智能体（4 个泳道：构建/分析、审查、领域、协调）
- 智能体层级系统：LOW（haiku）、MEDIUM（sonnet）、HIGH（opus）
- 委派类别：visual-engineering、ultrabrain、artistry、quick、writing

**技能系统**
- 28 个技能（初始发布）
- 技能流水线元数据（预览）
- learner 技能用于从会话自动提取技能

**钩子系统**
- 11 个生命周期事件
- keyword-detector 用于自动魔法关键词检测
- persistent-mode 用于 ralph/ultrawork 持久性
- code-simplifier 钩子（可选）

**配置系统**
- `config.jsonc` 配置文件（JSONC 格式）
- 2 级用户/项目配置
- 所有设置都可通过环境变量覆盖
- 自动 `forceInherit` 检测（CC Switch、Bedrock、Vertex AI）

**状态管理**
- Notepad：在上下文压缩后仍然存在的笔记
- Project Memory：跨会话维护项目知识
- Plan Notepad：捕获每个计划的学习
- `OMC_STATE_DIR` 用于集中式状态管理

**Team 编排**
- `omc team` CLI 命令
- 5 阶段流水线：plan → prd → exec → verify → fix
- team ralph：结合 team + 迭代验证

**autopilot**
- 5 阶段自主执行：expansion → planning → execution → qa → validation
- 3 类型验证：functional、security、quality
- Cancel/resume 支持

**外部模型集成**
- Codex/Gemini CLI worker 支持
- CCG（Claude-Codex-Gemini）三模型编排
- 委派路由（可选）

**代码智能**
- 12 个 LSP 工具（hover、goto definition、find references、diagnostics 等）
- 2 个 AST 工具（ast-grep search/replace）
- 目录级诊断

**平台支持**
- macOS、Linux：Bash 钩子
- Windows：推荐 WSL2，Node.js 钩子（实验性）
