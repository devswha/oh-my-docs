## 核心环境变量

### DISABLE_OMC

禁用所有 OMC 钩子。

```bash
export DISABLE_OMC=1
```

设置任何值时，OMC 被完全禁用。
Claude Code 运行时使用其默认行为，不使用 OMC。

### OMC_SKIP_HOOKS

仅跳过特定钩子。用逗号分隔。

```bash
export OMC_SKIP_HOOKS="keyword-detector,persistent-mode"
```

当你想禁用特定钩子而不是完全禁用（`DISABLE_OMC`）时使用此选项。

## 状态管理

### OMC_STATE_DIR

在集中式位置管理状态文件。

```bash
export OMC_STATE_DIR="$HOME/.claude/omc"
```

| 项目 | 默认值 | 设置后 |
|------|--------|--------|
| 状态存储位置 | `{worktree}/.omc/` | `$OMC_STATE_DIR/{project-id}/` |
| 删除 worktree 时 | 状态与之一起删除 | 状态被保留 |

项目标识符使用 Git 远程 URL 的哈希。
状态跨同一仓库的不同 worktree 共享。

## 特性控制

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `OMC_PARALLEL_EXECUTION` | `true` | 允许并行智能体执行 |
| `OMC_LSP_TOOLS` | `true` | 启用 LSP 工具 |
| `OMC_MAX_BACKGROUND_TASKS` | `5` | 最大后台任务数 |
| `OMC_LSP_TIMEOUT_MS` | `15000` | LSP 请求超时（毫秒）|

## 模型路由

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `OMC_ROUTING_ENABLED` | `true` | 启用智能模型路由 |
| `OMC_ROUTING_DEFAULT_TIER` | `MEDIUM` | 默认模型层级（LOW/MEDIUM/HIGH）|
| `OMC_ROUTING_FORCE_INHERIT` | `false` | 强制父模型继承 |
| `OMC_ESCALATION_ENABLED` | `true` | 允许升级 |

### 层级模型覆盖

```bash
export OMC_MODEL_HIGH="claude-opus-4-6"    # HIGH 层级默认模型
export OMC_MODEL_MEDIUM="claude-sonnet-4-6" # MEDIUM 层级默认模型
export OMC_MODEL_LOW="claude-haiku-4-5"    # LOW 层级默认模型
```

覆盖每个层级的默认模型 ID。在 Bedrock/Vertex AI 环境中特别有用。

### 模型别名覆盖

```bash
export OMC_MODEL_ALIAS_HAIKU="custom-haiku-model"
export OMC_MODEL_ALIAS_SONNET="custom-sonnet-model"
export OMC_MODEL_ALIAS_OPUS="custom-opus-model"
```

在使用非标准供应商时映射模型名称。

## 外部模型

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `OMC_EXTERNAL_MODELS_DEFAULT_CODEX_MODEL` | `gpt-5.3-codex` | 默认 Codex CLI 模型 |
| `OMC_EXTERNAL_MODELS_DEFAULT_GEMINI_MODEL` | `gemini-3.1-pro-preview` | 默认 Gemini CLI 模型 |
| `OMC_EXTERNAL_MODELS_DEFAULT_PROVIDER` | - | 默认外部供应商（codex/gemini）|
| `OMC_EXTERNAL_MODELS_FALLBACK_POLICY` | `provider_chain` | 模型失败时的回退策略 |
| `OMC_CODEX_DEFAULT_MODEL` | - | （已弃用）改用 `OMC_EXTERNAL_MODELS_DEFAULT_CODEX_MODEL` |
| `OMC_GEMINI_DEFAULT_MODEL` | - | （已弃用）改用 `OMC_EXTERNAL_MODELS_DEFAULT_GEMINI_MODEL` |

## 委派路由

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `OMC_DELEGATION_ROUTING_ENABLED` | `false` | 启用委派路由至外部供应商 |
| `OMC_DELEGATION_ROUTING_DEFAULT_PROVIDER` | `claude` | 默认委派供应商 |

## MCP 服务器

| 变量 | 说明 |
|------|------|
| `EXA_API_KEY` | Exa MCP 服务器 API 密钥 |

## 高级设置

| 变量 | 说明 |
|------|------|
| `OMC_BRIDGE_SCRIPT` | Python 桥接脚本路径（自动检测）|
| `OMC_USE_NODE_HOOKS` | 设置为 `1` 时，在 macOS/Linux 上使用 Node.js 钩子 |

## 使用示例

### 成本节省模式

```bash
# 禁用路由，所有智能体继承父模型
export OMC_ROUTING_FORCE_INHERIT=true
```

### 调试模式

```bash
# 禁用特定钩子以隔离问题
export OMC_SKIP_HOOKS="persistent-mode,keyword-detector"
```

### 大型项目

```bash
# 增加 LSP 超时
export OMC_LSP_TIMEOUT_MS=30000

# 允许更多后台任务
export OMC_MAX_BACKGROUND_TASKS=10
```
