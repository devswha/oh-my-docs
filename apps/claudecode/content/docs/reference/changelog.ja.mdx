## 変更履歴

Oh My ClaudeCode のバージョン別の主要な変更内容です。

### v4.12.0 (current)

**新機能**
- Team: ロール別プロバイダーとモデルルーティングおよび resolved-routing スナップショット — `.claude/omc.jsonc` で canonical role（例：`codex critic`、`gemini reviewer`）ごとに `roleRouting` を設定できます (#2614)
- HUD: 追加の利用額支出データを表示 (closes #2570)
- HUD: MiniMax coding plan 利用プロバイダーを追加
- HUD: プロバイダー別に利用キャッシュを分割し、クロスセッション スラッシング を排除
- Release: リリーススキルを汎用リポジトリ対応リリースアシスタントとして書き直し — リポジトリと CI を分析し、ルールを `.omc/RELEASE_RULE.md` にキャッシュしてリリースをガイド (#2501)
- CI: アップグレードテストを追加して警告とエラーをキャッチ

**バグ修正（抜粋）**
- Persistent stop hook: コントラクトを整列し、エージェント出力コントラクトを厳密化 (#2653)
- Context-bloat: 繰り返されるルール/スキル注入の 3 つの根源を排除 (#2577)
- Pre-tool-enforcer: 集中化されたステート根解決器経由で `OMC_STATE_DIR` を尊重 (#2621)
- Ralplan: 自動開始前に呼び出し意図を要求 (#2620)
- Team: trigger メッセージを短縮して `sendToWorker` 200 文字制限に適合 (#2617)
- Installer: アップデート中にユーザースキルを OMC スタイルフロントマター付きで保持 (#2573)
- Permission-handler: 読み取り専用 `gh issue`/`pr` コマンドを許可、インストーラーライブラリアサーションを追加
- Deep-interview: 5 つの残りのハードコード化された 20%/0.2 しきい値シグナルを置換 (#2545)
- State-root: hook エントリポイント全体で `OMC_STATE_DIR` 解決を集中化 (#2532)
- Keyword-detector: 貼り付けられたトランスクリプトブロックを無視、レビューシード echo を抑制 (#2541)
- Session-start: stale `CLAUDE_PLUGIN_ROOT` を最新バージョンにシンボリックリンク
- Doctor: `CLAUDE.md` バージョンドリフトをプラグインキャッシュに対して検出
- Tmux: すべての tmux 実行をラッパー関数経由で集中化

**リファクタリング**
- HUD: z.ai `TOKENS_LIMIT` をリセット時間ではなくユニットで分類
- MCP: deslop standalone-server と `listtools` テスト

**統計:** 121 PR マージ · 6 個の新機能 · 67 個のバグ修正 · 3 個のその他の変更

### v4.8.0

**エージェントシステム**
- 19 個の専門エージェント（4 レーン: Build/Analysis、Review、Domain、Coordination）
- エージェント段階システム: LOW (haiku)、MEDIUM (sonnet)、HIGH (opus)
- 委譲カテゴリ: visual-engineering、ultrabrain、artistry、quick、writing

**スキルシステム**
- 28 個のスキル（初期リリース）
- スキルパイプラインメタデータ（プレビュー）
- セッションから自動スキル抽出のための learner スキル

**フックシステム**
- 11 個のライフサイクルイベント
- 自動マジックキーワード検出のための keyword-detector
- ralph/ultrawork persistence のための persistent-mode
- code-simplifier フック（オプトイン）

**設定システム**
- `config.jsonc` 設定ファイル（JSONC フォーマット）
- 2 レベルのユーザー/プロジェクト設定
- すべての設定を環境変数経由でオーバーライド可能
- 自動 `forceInherit` 検出（CC Switch、Bedrock、Vertex AI）

**ステート管理**
- Notepad: コンテキスト圧縮を生き延びるノート
- Project Memory: セッション全体でプロジェクトナレッジを維持
- Plan Notepad: プラン別の学習をキャプチャ
- `OMC_STATE_DIR` 集中化ステート管理用

**Team オーケストレーション**
- `omc team` CLI コマンド
- 5 段階パイプライン: plan → prd → exec → verify → fix
- team ralph: team + iterative verification を組み合わせ

**autopilot**
- 5 段階自律実行: expansion → planning → execution → qa → validation
- 3 種類の validation: functional、security、quality
- Cancel/resume サポート

**外部モデル統合**
- Codex/Gemini CLI ワーカーサポート
- CCG（Claude-Codex-Gemini）3 モデルオーケストレーション
- 委譲ルーティング（オプトイン）

**コード知能**
- 12 個の LSP ツール（hover、goto definition、find references、diagnostics など）
- 2 個の AST ツール（ast-grep search/replace）
- ディレクトリレベルの診断

**プラットフォームサポート**
- macOS、Linux: Bash フック
- Windows: WSL2 推奨、Node.js フック（実験的）
