## 개요

Plan은 인터뷰, 직접, 합의, 리뷰 4가지 모드를 지원하는 전략 계획 수립 스킬입니다. 요청이 얼마나 구체적인지에 따라 자동으로 모드를 선택하고, 코드베이스를 먼저 탐색한 뒤 사용자에게 질문합니다.

## 사용법

```bash
/oh-my-claudecode:omc-plan "implement user authentication"
/oh-my-claudecode:omc-plan --consensus "redesign the API layer"
/oh-my-claudecode:omc-plan --direct "add caching to the search endpoint"
/oh-my-claudecode:omc-plan --review
```

매직 키워드로도 활성화됩니다.

```
plan this feature
plan the migration
let's plan the refactoring
```

## 모드 선택

| 모드 | 트리거 | 동작 |
|------|--------|------|
| Interview | 광범위한 요청 (기본) | 인터랙티브 요구사항 수집 |
| Direct | `--direct` 또는 상세한 요청 | 인터뷰 건너뛰고 바로 계획 생성 |
| Consensus | `--consensus` 또는 `ralplan` | Planner-Architect-Critic 반복 합의 |
| Review | `--review` | 기존 계획에 대한 Critic 평가 |

## Interview 모드

광범위하거나 모호한 요청에 대해 인터랙티브로 요구사항을 수집합니다.

- 코드베이스 사실은 explore 에이전트로 먼저 파악합니다
- 사용자에게는 한 번에 하나의 질문만 합니다
- 각 질문은 이전 답변을 기반으로 합니다
- 사용자가 준비되면 계획을 생성합니다

## Consensus 모드

RALPLAN-DR 구조적 숙의를 쓰는 합의 모드입니다.

1. **Planner**가 초기 계획과 RALPLAN-DR 요약 작성
2. **Architect**가 아키텍처 건전성 검토 (순차 실행)
3. **Critic**이 품질 기준 평가 (Architect 완료 후 실행)
4. 거부 시 최대 5회 반복 수정
5. 승인 시 ADR 포함한 최종 계획 출력

### RALPLAN-DR 요약 구성

- 원칙 (3-5개)
- 의사결정 동인 (상위 3개)
- 실행 가능한 옵션 (2개 이상), 각각의 장단점
- `--deliberate` 모드: 사전 사망 분석 (3개 시나리오) + 확장 테스트 계획

## 옵션/플래그

| 플래그 | 설명 |
|--------|------|
| `--direct` | 인터뷰 없이 직접 계획 생성 |
| `--consensus` | 합의 모드 활성화 |
| `--review` | 기존 계획 리뷰 |
| `--interactive` | 합의 모드에서 사용자 프롬프트 활성화 |
| `--deliberate` | 고위험 작업용 심층 숙의 모드 |
| `--architect codex` | Architect 단계에서 Codex 사용 |
| `--critic codex` | Critic 단계에서 Codex 사용 |

## 계획 출력 형식

모든 계획에 포함되는 항목:

- 요구사항 요약
- 수락 기준 (테스트 가능한)
- 구현 단계 (파일 참조 포함)
- 리스크와 완화 방안
- 검증 단계
- Consensus 모드: RALPLAN-DR 요약 + ADR

계획은 `.omc/plans/`에 저장됩니다.
