## 概要

Deepinit はコードベース全体で階層的な AGENTS.md ドキュメントを生成します。AGENTS.md ファイルは、各ディレクトリの内容、コンポーネント関係、コード作業時の考慮事項を説明する AI 可読ドキュメントです。

目標は、エージェントがコードベースをより深く理解し、より正確に作業するのを支援することです。

## 使用方法

```bash
/oh-my-claudecode:deepinit
/oh-my-claudecode:deepinit [path]
```

## 階層的タグ付け

すべての AGENTS.md (root を除く) には親参照タグが含まれます。

```
/AGENTS.md                        (root, 親なし)
├── src/AGENTS.md                 <!-- Parent: ../AGENTS.md -->
│   ├── src/components/AGENTS.md  <!-- Parent: ../AGENTS.md -->
│   └── src/utils/AGENTS.md       <!-- Parent: ../AGENTS.md -->
└── docs/AGENTS.md                <!-- Parent: ../AGENTS.md -->
```

## AGENTS.md 構造

各ファイルに含まれるセクション:

- **Purpose**: ディレクトリの役割の説明
- **Key Files**: 説明付きの重要なファイルのリスト
- **Subdirectories**: サブディレクトリのリスト
- **For AI Agents**: 作業命令、テスト要件、コードパターン
- **Dependencies**: 内部および外部依存

## 実行ワークフロー

1. **ディレクトリ構造をマップ**: explore エージェントが全体構造をマップ
2. **タスク計画を生成**: 深さレベルで整理
3. **レベル別に生成**: 親から子の順 (親参照有効性を確保)
4. **既存ファイルを更新**: 既に存在する場合は比較と統合 (手動セクションを保持)
5. **階層を検証**: 親参照、孤立ファイル、完全性をチェック

## 並列化ルール

- 同じレベルのディレクトリ: 並列処理
- 異なるレベル: 順序処理 (親が最初)
- 大規模ディレクトリ: 専用エージェントを割り当て
- 小規模ディレクトリ: 複数を単一エージェントで処理

## 更新モード

既存の AGENTS.md ファイルを持つコードベースで実行する場合:

1. 既存のコンテンツを読み込みと解析
2. 現在のディレクトリ状態を分析
3. 既存と現在の差分を比較
4. 自動生成セクションを更新、`<!-- MANUAL -->` 下の手動セクションを保持
