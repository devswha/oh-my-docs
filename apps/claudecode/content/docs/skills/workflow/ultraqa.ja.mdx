## 概要

UltraQA は自律的 QA サイクリングワークフローです。目標（テスト合格、ビルド成功、lint クリア など）が達成されるまで、qa-tester 実行、architect 診断、executor 修正を繰り返します。

最大 5 サイクル実行し、同じエラーが 3 回繰り返された場合は根本的な問題として扱って早期終了します。

## 使用方法

```bash
/oh-my-claudecode:ultraqa --tests
/oh-my-claudecode:ultraqa --build
/oh-my-claudecode:ultraqa --lint
/oh-my-claudecode:ultraqa --typecheck
/oh-my-claudecode:ultraqa --custom "all endpoints return 200"
/oh-my-claudecode:ultraqa --interactive
```

## 目標タイプ

| フラグ | 目標 | チェック対象 |
|------|------|-----------------|
| `--tests` | テスト | すべてのテストスイートが合格 |
| `--build` | ビルド | ビルド成功（終了コード 0） |
| `--lint` | Lint | Lint エラーなし |
| `--typecheck` | 型チェック | TypeScript エラーなし |
| `--custom "pattern"` | カスタム | 出力内の特定パターンをチェック |
| `--interactive` | インタラクティブ | qa-tester 経由での CLI/サービステスト |

## サイクルワークフロー

各サイクルは以下のステップを通じて進みます。

1. **QA 実行**：目標タイプに基づいて検証を実行
2. **結果チェック**：合格の場合は成功メッセージで終了
3. **Architect 診断**：失敗時に opus モデルで根本原因を分析
4. **修正適用**：Executor が architect の推奨に基づいて修正を適用
5. **繰り返し**：ステップ 1 に戻る

## 終了条件

| 条件 | アクション |
|-----------|--------|
| 目標達成 | `ULTRAQA COMPLETE: Goal met after N cycles` |
| 5 サイクル到達 | `ULTRAQA STOPPED: Max cycles. Diagnosis: ...` |
| 同じエラー 3 回 | `ULTRAQA STOPPED: Same failure detected 3 times` |
| 環境エラー | `ULTRAQA ERROR: [issue description]` |

## 進捗出力

```
[ULTRAQA Cycle 1/5] Running tests...
[ULTRAQA Cycle 1/5] FAILED - 3 tests failing
[ULTRAQA Cycle 1/5] Architect diagnosing...

[ULTRAQA Cycle 2/5] Running tests...
[ULTRAQA Cycle 2/5] PASSED - All 47 tests pass
[ULTRAQA COMPLETE] Goal met after 2 cycles
```
