## 개요

OMC Teams는 tmux 페인에서 N개의 CLI 워커 프로세스를 생성해서 병렬로 작업을 실행합니다. Claude, Codex, Gemini 세 가지 에이전트 타입을 지원합니다.

Claude Code 네이티브 팀 에이전트를 쓰는 `/team`과 달리, 외부 CLI 프로세스를 직접 실행하는 방식입니다.

## 사용법

```bash
/oh-my-claudecode:omc-teams N:claude "task description"
/oh-my-claudecode:omc-teams N:codex "task description"
/oh-my-claudecode:omc-teams N:gemini "task description"
```

### 파라미터

- **N**: CLI 워커 수 (1-10)
- **agent-type**: `claude`, `codex`, 또는 `gemini`
- **task**: 모든 워커에 분배할 작업 설명

## 예시

```bash
/omc-teams 2:claude "implement auth module with tests"
/omc-teams 2:codex "review the auth module for security issues"
/omc-teams 3:gemini "redesign UI components for accessibility"
```

## 요구사항

- **tmux**: 설치되어 있어야 합니다 (`command -v tmux`)
- **활성 tmux 세션**: 워커 페인 생성에 필요합니다
- **claude CLI**: `npm install -g @anthropic-ai/claude-code`
- **codex CLI**: `npm install -g @openai/codex`
- **gemini CLI**: `npm install -g @google/gemini-cli`

## 워크플로우

1. **전제조건 확인**: tmux 설치 및 활성 세션 확인
2. **입력 파싱**: N, agent-type, task 추출 및 검증
3. **작업 분해**: 작업을 N개의 독립 하위 작업으로 분리
4. **CLI 팀 시작**: `omc team N:agent "task"` 실행
5. **모니터링**: `omc team status <team-name>`으로 진행 상황 확인
6. **종료**: 필요 시 `omc team shutdown <team-name>`
7. **보고**: 결과 요약 보고

## `/team`과의 차이점

| 항목 | `/team` | `/omc-teams` |
|------|---------|-------------|
| 워커 타입 | Claude Code 네이티브 팀 에이전트 | claude/codex/gemini CLI 프로세스 |
| 호출 방법 | `TeamCreate` / `Task` / `SendMessage` | `omc team` CLI + tmux |
| 조율 방식 | 네이티브 팀 메시징, 단계별 파이프라인 | tmux 워커 런타임 + CLI API 상태 파일 |
| 사용 시점 | Claude 네이티브 팀 오케스트레이션 | 외부 CLI 워커 실행 |
