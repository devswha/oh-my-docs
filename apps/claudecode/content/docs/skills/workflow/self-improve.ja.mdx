## 概要

`self-improve` は **自律的進化コード改善エンジン** です。ターゲットリポジトリに対して研究 → 計画 → 実行 → トーナメント選抜ループを実行します。完全なライフサイクルを管理します：セットアップ、研究、計画、実行、トーナメント選抜、履歴記録、可視化、停止条件評価。

これは **レベル 4 スキル** です：開始すると停止条件が満たされるまで完全に自律実行されます。イテレーション間での確認は一切行いません。

## 使用方法

```bash
/oh-my-claudecode:self-improve
```

## 自律実行ポリシー

**ループは改善中にユーザーに確認を求めることはありません。** ゲートチェックに合格してループが開始された後、停止条件が満たされるまで完全に自律実行されます。

- イテレーション間での確認なし
- 「要約して待機」なし
- エージェント失敗時：1 回リトライ、失敗したらスキップして続行
- すべての計画が却下時：ログして次のイテレーションに進む
- すべてのエグゼキューター失敗時：ログして続行
- ベンチマークエラー時：ログ、エグゼキューター失敗とマーク、続行
- **明示的な停止条件のみ** ループを停止

## トラストバウンダリ

ループはベンチマークコマンドをターゲットリポジトリ内で現状のまま実行します。ユーザーは **セットアップ中にリポジトリパスとベンチマークコマンドを明示的に確認** します。ループは：

- パッケージをインストールしません
- システム設定を変更しません
- ベンチマークコマンドが実行する以外のネットワークリソースにアクセスしません
- **シール済みファイル**：`validate.sh` がベンチマークコードの変更を禁止し、評価の自己変更を防止

## 状態レイアウト

すべての状態は `.omc/self-improve/` の下に置かれます：

```
.omc/self-improve/
├── config/                       # ユーザー設定
│   ├── settings.json             # agents, benchmark, thresholds, sealed_files
│   ├── goal.md                   # 改善目標 + ターゲットメトリクス
│   ├── harness.md                # ガードレールルール（H001/H002/H003）
│   └── idea.md                   # ユーザー実験アイデア
├── state/                        # ランタイム状態
│   ├── agent-settings.json       # iterations, best_score, status
│   ├── iteration_state.json      # イテレーション内進捗（再開可能）
│   ├── research_briefs/          # ラウンドあたりの研究出力
│   ├── iteration_history/        # ラウンドあたりの完全な履歴
│   ├── merge_reports/            # トーナメント結果
│   └── plan_archive/             # アーカイブ済みプラン（永続）
├── plans/                        # アクティブなプラン（現在のラウンド）
└── tracking/                     # 可視化データ
    ├── raw_data.json             # すべての候補スコア
    ├── baseline.json             # 初期ベンチマークスコア
    ├── events.json               # 設定変更
    └── progress.png              # 生成チャート
```

OMC モードライフサイクル：`.omc/state/sessions/{sessionId}/self-improve-state.json`

## ループステージ

1. **セットアップゲート** — 目標、ベンチマーク、シール済みファイル、ガードレールルールを確認
2. **研究** — 次のイテレーション向けのコンテキストを収集
3. **計画** — 候補改善計画を生成
4. **実行** — 計画を候補として並列適用
5. **トーナメント選抜** — 各候補をベンチマーク、勝者を保持
6. **履歴記録** — イテレーション結果をログ、追跡を更新
7. **停止チェック** — 停止条件を評価；満たされていなければステップ 2 に戻る

## 停止条件

ループは以下でのみ停止：

- ターゲットメトリクスに到達（`goal.md` から）
- 最大イテレーションに到達
- N 回連続イテレーション全体での繰り返し失敗
- ユーザーの明示的なキャンセル（`/oh-my-claudecode:cancel`）
