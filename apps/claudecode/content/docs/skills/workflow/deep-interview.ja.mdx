## 概要

Deep Interview は Ouroboros にインスパイアされた Socratic 質問法を数学的な曖昧さスコアリングと組み合わせます。ターゲットを絞った質問で隠れた仮定を浮き出させ、加重された複数の次元で明確性を測定し、曖昧さがしきい値 (デフォルト 20%) を下回るまで続行を拒否します。

3 段階パイプラインの最初のステップとして設計されています: **deep-interview → ralplan → autopilot**

## 使用方法

```bash
/oh-my-claudecode:deep-interview "vague idea or description"
```

マジックキーワードでも起動できます。

```
deep interview me about this project idea
ouroboros help me clarify what I want
I have a vague idea about a task management tool
```

## 曖昧さスコア

インタビュー中、明確性は 4 つの次元で測定されます。

| 次元 | Greenfield 重み | Brownfield 重み |
|-----------|-------------------|-------------------|
| 目標の明確さ | 40% | 35% |
| 制約の明確さ | 30% | 25% |
| 成功基準の明確さ | 30% | 25% |
| コンテキストの明確さ | - | 15% |

Brownfield (既存コードの変更) は既存システムの理解も測定します。

### スコア解釈

| 範囲 | 意味 |
|-------|---------|
| 0-10% | 完全に明確、すぐに進行 |
| 10-20% | 十分に明確 (デフォルトしきい値) |
| 20-40% | いくつかのギャップあり、インタビュー継続 |
| 40-60% | 重大なギャップあり、弱い次元に焦点 |
| 60-80% | 非常に不明確、リフレーミングが必要 |
| 80-100% | ほとんど何も不明 |

## チャレンジエージェント

質問の視点は特定のラウンドで変わります。

| モード | 起動 | 目的 |
|------|-----------|---------|
| Contrarian | ラウンド 4 以降 | 「反対が本当だったら?」 — 根本的な仮定に異議 |
| Simplifier | ラウンド 6 以降 | 「最もシンプルなバージョンは?」 — 複雑性を削除 |
| Ontologist | ラウンド 8 以降 (曖昧さ > 30%) | 「これは本当は何?」 — 本質を発見 |

各モードは 1 度だけ使用されます。

## 実行フェーズ

### Phase 1: 初期化

- ユーザーのアイデアを解析
- Brownfield と Greenfield を検出 (explore エージェント)
- 状態を初期化

### Phase 2: インタビューループ

- 最も弱い次元をターゲットとした質問を生成
- 1 度に 1 つの質問のみ
- 各回答後に曖昧さスコアを計算と表示
- コードベースの事実は explore エージェント経由で最初に収集

### Phase 3: チャレンジエージェント

視点はラウンド 4、6、8 で変わります。

### Phase 4: スペック結晶化

曖昧さしきい値に達したら、スペックファイルを生成します: `.omc/specs/deep-interview-{slug}.md`

### Phase 5: 実行ブリッジ

実行オプションを提示します:

1. **Ralplan → Autopilot** (推奨): コンセンサス精錬その後自律実行
2. **Autopilot** (高速): コンセンサス精錬なしの直接自律実行
3. **Ralph**: 継続ループ実行
4. **Team**: N 個の並列エージェント
5. **インタビュー継続**: より大きな明確さのためにインタビューを継続

## 3 段階パイプライン

```
deep-interview → ralplan → autopilot
  曖昧さ ≤ 20%   コンセンサス計画   実行
```

各ステージは異なる品質ゲートを提供します。

1. **Deep Interview**: 明確さゲート (ユーザーが何を望んでいるか知っているか?)
2. **Ralplan**: 実現可能性ゲート (アプローチは健全か?)
3. **Autopilot**: 正確性ゲート (コードは動作するか?)
