## 개요

Team은 N개의 Claude Code 네이티브 에이전트를 생성해서 공유 작업 목록에서 협업하게 하는 조율 스킬입니다. 팀 관리, 에이전트 간 메시지, 작업 의존성이 내장되어 있고 외부 의존성은 필요 없습니다.

5단계 파이프라인(team-plan, team-prd, team-exec, team-verify, team-fix)으로 작업을 진행합니다.

## 사용법

```bash
/oh-my-claudecode:team N:agent-type "task description"
/oh-my-claudecode:team "task description"
/oh-my-claudecode:team ralph "task description"
```

### 파라미터

- **N**: 팀원 에이전트 수 (1-20). 생략하면 작업 분해에 따라 자동 결정
- **agent-type**: team-exec 단계에서 쓸 에이전트 유형 (executor, debugger, designer, codex, gemini 등). 생략하면 단계별 자동 라우팅
- **task**: 분해하고 분배할 고수준 작업 설명
- **ralph**: 지정하면 Ralph의 지속 루프로 팀 파이프라인을 감싸서 실패 시 재시도와 architect 검증 수행

## 예시

```bash
# 5명의 executor로 TypeScript 에러 수정
/team 5:executor "fix all TypeScript errors across the project"

# 3명의 debugger로 빌드 에러 수정
/team 3:debugger "fix build errors in src/"

# 자동 크기 결정, 자동 라우팅
/team "refactor the auth module with security review"

# Ralph 지속 루프와 함께
/team ralph "build a complete REST API for user management"

# Codex CLI 워커 사용
/team 3:codex "review all API endpoints for security"
```

## 파이프라인 단계

### team-plan

explore와 planner 에이전트가 작업을 분석하고 실행 계획을 수립합니다.

### team-prd

analyst가 요구사항을 분석하고 수락 기준을 정의합니다.

### team-exec

팀원들이 병렬로 작업을 실행합니다. 단계별 에이전트 라우팅이 적용됩니다.

### team-verify

verifier와 필요한 리뷰어가 결과를 검증합니다.

### team-fix

검증에서 발견된 문제를 수정합니다. 결함 유형에 따라 executor 또는 debugger가 쓰입니다.
수정 루프는 최대 시도 횟수로 제한됩니다.

## Team + Ralph

`team ralph`은 팀 파이프라인을 Ralph의 지속 루프로 감쌉니다.

- 팀이 실패하면 Ralph가 재시도합니다
- 완료 시 architect 검증을 수행합니다
- 어느 한쪽을 취소하면 다른 쪽도 함께 취소됩니다

## 역할(Role)별 Provider & Model 라우팅

> **범위:** `/team`에만 적용됩니다. Task 기반 위임은 별도 시스템(`delegationRouting`)을 사용하며, 둘은 공존하도록 설계되어 있습니다.

canonical 역할별로 어떤 provider(`claude`, `codex`, `gemini`)와 어떤 모델 티어를 사용할지 선언합니다. 라우팅은 팀 생성 시점에 **한 번** 해석되어 `TeamConfig.resolved_routing`에 저장됩니다 — spawn, scale-up, 재시작 모두 이 스냅샷을 읽기 때문에, 한 팀의 수명 동안 역할별 워커 CLI와 모델이 안정적으로 유지됩니다.

### 예시 — `.claude/omc.jsonc`

```jsonc
{
  "team": {
    "roleRouting": {
      "orchestrator":  { "model": "inherit" },
      "planner":       { "provider": "claude", "model": "HIGH" },
      "analyst":       { "provider": "claude", "model": "HIGH" },
      "executor":      { "provider": "claude", "model": "MEDIUM" },
      "critic":        { "provider": "codex" },
      "code-reviewer": { "provider": "gemini" },
      "test-engineer": { "provider": "gemini", "model": "MEDIUM" }
    }
  }
}
```

### Canonical 역할

`orchestrator`, `planner`, `analyst`, `architect`, `executor`, `debugger`, `critic`, `code-reviewer`, `security-reviewer`, `test-engineer`, `designer`, `writer`, `code-simplifier`, `explore`, `document-specialist`.

별칭은 파싱 시점에 정규화됩니다 — `reviewer` → `code-reviewer`, `quality-reviewer` → `code-reviewer`, `harsh-critic` → `critic`, `build-fixer` → `debugger`. 알 수 없는 역할은 검증에서 실패합니다.

### 스펙 필드

- **`provider`** — `"claude" | "codex" | "gemini"`. 생략 시 `claude`.
- **`model`** — 티어(`"HIGH" | "MEDIUM" | "LOW"`) 또는 명시적 모델 ID. 티어는 `routing.tierModels`로 해석됩니다.
- **`agent`** — 선택. Claude 에이전트 이름(예: `"critic"`, `"executor"`). resolved provider가 `claude`일 때만 유효.

`orchestrator`는 `claude`로 고정되며, 해당 역할은 `model`만 설정 가능합니다.

### 환경변수 override

```bash
OMC_TEAM_ROLE_OVERRIDES='{"critic":{"provider":"codex"},"code-reviewer":{"provider":"gemini"}}'
```

우선순위: `OMC_TEAM_ROLE_OVERRIDES` > `.claude/omc.jsonc` (프로젝트) > `~/.config/claude-omc/config.jsonc` (사용자) > 내장 기본값.

### CLI 폴백

설정된 provider의 CLI가 spawn 시점에 `PATH`에 없으면, team lead가 경고 메시지를 보내고 동일 티어 + agent를 유지한 채 Claude로 결정적으로 폴백합니다. 폴백은 의도적으로 "크게 보이도록" 설계되어 있습니다. `omc doctor --team-routing`으로 사전 확인할 수 있습니다.

### Zero-config 동작

`team.roleRouting`이 비어 있으면 패치 이전 동작이 유지됩니다 — 모든 워커는 Claude, 모델 티어는 `routing.tierModels`를 따르고 `/team 3:executor ...`는 여전히 Claude Sonnet executor 3명을 spawn합니다.

## 설정

`.claude/omc.jsonc` (프로젝트) 또는 `~/.config/claude-omc/config.jsonc` (사용자)에서 선택적으로 설정합니다.

```jsonc
{
  "team": {
    "ops": {
      "maxAgents": 20,
      "defaultAgentType": "claude",
      "monitorIntervalMs": 30000,
      "shutdownTimeoutMs": 15000
    }
  }
}
```

- **`ops.maxAgents`** — 최대 팀원 수 (기본 20)
- **`ops.defaultAgentType`** — `/team`에서 provider를 지정하지 않을 때 쓸 CLI (`claude` | `codex` | `gemini`, 기본 `claude`)
- **`ops.monitorIntervalMs`** — `TaskList` 폴링 주기 (기본 30초)
- **`ops.shutdownTimeoutMs`** — 종료 응답 대기 시간 (기본 15초)
