## 개요

HUD는 Claude Code의 상태 라인에 OMC 실시간 상태를 표시하는 Heads-Up Display를 설정하고 관리하는 스킬입니다. 활성 모드, 컨텍스트 사용량, 에이전트 상태, TODO 진행 상황 등을 한눈에 볼 수 있습니다.

## 사용법

```bash
/oh-my-claudecode:hud              # 현재 상태 표시 (필요 시 자동 설정)
/oh-my-claudecode:hud setup        # HUD 상태 라인 설치/수리
/oh-my-claudecode:hud minimal      # 최소 디스플레이
/oh-my-claudecode:hud focused      # 포커스 디스플레이 (기본값)
/oh-my-claudecode:hud full         # 전체 디스플레이
/oh-my-claudecode:hud status       # 상세 상태
```

## 프리셋

### Minimal

핵심 정보만 표시합니다.

```
[OMC] ralph | ultrawork | todos:2/5
```

### Focused (기본값)

관련 요소를 모두 표시합니다.

```
[OMC] branch:main | ralph:3/10 | US-002 | ultrawork | ctx:67% | agents:2 | todos:2/5
```

### Full

에이전트 상세 정보를 포함한 멀티라인 디스플레입니다.

```
[OMC] repo:omc branch:main | ralph:3/10 | US-002 | ctx:[████░░]67% | agents:3 | todos:2/5
├─ O architect    2m   analyzing architecture patterns...
├─ e explore     45s   searching for test files
└─ s executor     1m   implementing validation logic
```

## 디스플레이 요소

| 요소 | 설명 |
|------|------|
| `[OMC]` | 모드 식별자 |
| `repo:name` | Git 저장소 이름 |
| `branch:name` | Git 브랜치 이름 |
| `ralph:3/10` | Ralph 반복 횟수/최대 |
| `US-002` | 현재 PRD 스토리 ID |
| `ctx:67%` | 컨텍스트 윈도우 사용량 |
| `agents:2` | 실행 중인 서브에이전트 수 |
| `todos:2/5` | TODO 완료 상태 |

## 색상 코드

- **초록**: 정상/건강
- **노랑**: 경고 (컨텍스트 > 70%, ralph > 7)
- **빨강**: 위험 (컨텍스트 > 85%, ralph 최대)

## 설정

`~/.claude/settings.json`의 `omcHud` 키에서 설정합니다.

```json
{
  "omcHud": {
    "preset": "focused",
    "elements": {
      "omcLabel": true,
      "contextBar": true,
      "agents": true,
      "todos": true,
      "ralph": true,
      "showTokens": false
    }
  }
}
```

사용 가능한 프리셋: `minimal`, `focused`, `full`, `dense`, `analytics`, `opencode`

## 사용량 Provider

`showTokens`가 활성화되면 HUD는 현재 세션을 지원하는 provider에서 가져온 사용량 및 지출 데이터를 표시합니다. 계정을 전환할 때 세션 간 캐시 충돌을 제거하기 위해 사용량 캐시는 provider별로 분리되어 있습니다.

지원 provider:

- **Claude** (claude.ai 구독, Anthropic API, Bedrock)
- **z.ai** — 주간 `TOKENS_LIMIT`을 단위(unit) 기준으로 분류 (pro+ 티어 지원)
- **MiniMax** — coding plan 사용량 표시, `minimaxi.com` / `minimax.com` 도메인 모두 지원. 라우팅은 자격 증명 유무와 독립적으로 동작
- **Codex / OpenAI** (`omc ask codex` 또는 team 워커로 라우팅된 경우)

provider가 노출할 경우 추가 지출 데이터도 표시됩니다.
