## 概要

`verify` はあいまいな「うまくいくはずだ」という主張を**具体的な証拠**に変える。機能、修正、またはリファクタが実際に機能することに自信を持ちたい場合、完了を宣言する前に使用します。

このスキルは `verifier` エージェントの軽量な対抗言 — サブエージェント生成なし直接呼び出すことができるフォーカスされたワークフロー。

## 使用方法

```bash
/oh-my-claudecode:verify
```

証明する必要のある正確な動作を述べます。スキルが優先度順に検証オプションをガイド。

## ワークフロー

1. **特定する** 証明する必要のある正確な動作
2. **既存テストを優先**
3. カバレッジが欠落している場合、利用可能な**最も狭い直接検証コマンド**を実行
4. 直接自動化で十分でない場合、**手動検証ステップ**を説明し、具体的で観察可能な証拠を集める
5. **実際に検証されたもののみレポート**

## 検証順序

1. 既存テスト（`npm test`、`pytest` など）
2. タイプチェック / ビルド（`tsc`、`npm run build`）
3. 狭い直接コマンドチェック（`curl`、`node -e ...` など）
4. 手動またはインタラクティブ検証

## ルール

- **証拠なしで変更が完了したと言わない**
- チェックが失敗した場合、**失敗を明確に含める**
- 現実的な検証パス存在しない場合、**明確にそれを述べる** 誤魔化すのではなく
- うるさいログより簡潔な証拠サマリーを優先

## 出力フォーマット

```
Verified:           <何が証明されたか>
Commands run:       <何が実行されたか>
Passed:             <何が合格したか>
Failed/unverified:  <何が合格しなかった、またはチェックされなかったか>
```
