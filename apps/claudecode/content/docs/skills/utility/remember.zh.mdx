## 概述

`remember` 将持久的、可重用的知识推升到**正确的记忆表面**，而不是让它埋在聊天历史中。当会话发现值得保存的东西时使用它 — 一个项目事实、一个约定、一个陷阱 — 并且你想将其路由到正确的目的地。

## 使用

```bash
/oh-my-claudecode:remember
```

描述你想保存的内容。该技能对每个项目进行分类并提议最佳存储位置。

## 记忆表面

| 表面 | 目的 | 生命周期 |
|---|---|---|
| **项目记忆**（`.omc/project-memory.json`） | 持久的团队/项目知识 | 会话间持久 |
| **Notepad 优先级**（`.omc/notepad.md`） | 短的高信号上下文用于下一轮 | 在压缩中存活 |
| **Notepad 工作中** | 临时活跃会话笔记 | 会话作用域 |
| **文档 / AGENTS / CLAUDE 文件** | 持久的指令和约定 | 永久、版本控制 |

## 工作流

1. 收集相关的会话发现
2. 对每个项目进行分类：
   - 持久项目事实
   - 临时工作笔记
   - 操作员偏好或指令
   - 重复 / 陈旧 / 冲突信息
3. 为每个项目提议最佳目的地
4. 仅写入或更新适当的记忆表面
5. 指出应清理的重复或冲突

## 规则

- **不要将所有内容倾倒到一个存储中**
- 对持久团队知识优先选择**项目记忆**
- 对短期工作上下文优先选择**notepad**
- 保持项目**简洁和可操作**
- 如果有不确定的内容，将其标记为不确定而不是作为事实存储

## 输出

- 存储了什么
- 存储在哪里（哪个表面）
- 找到的任何重复/冲突
