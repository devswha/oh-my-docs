## 개요

`skillify`는 성공한 멀티스텝 워크플로우를 나중에 재발견하는 대신 구체적인 스킬 드래프트로 잡아둡니다. 현재 세션에서 재사용 가능한 OMC 스킬이 될 만한 반복 패턴을 발견했을 때 사용하세요.

[learner](/ko/docs/skills/utility/learner)와 자연스러운 보완 관계입니다 — `learner`는 세션 경험에서 통찰을 추출하고, `skillify`는 설치 가능한 구체적 `SKILL.md`를 산출합니다.

## 사용법

```bash
/oh-my-claudecode:skillify
```

동작했던 워크플로우를 설명하면, 스킬이 그것을 재사용 가능한 SKILL.md로 구조화합니다.

## 워크플로우

1. **식별**: 세션에서 완수한 반복 가능한 작업
2. **추출**: 구조화된 조각들
   - 입력
   - 순서가 있는 단계
   - 성공 기준
   - 제약 / 함정
   - 스킬의 최선 목표 위치
3. **결정**: 워크플로우가 어디에 속하는가
   - 레포 빌트인 스킬
   - 사용자/프로젝트 학습 스킬 (`.omc/skills/`)
   - 문서 전용
4. **드래프트**: 명확한 트리거, 단계, 성공 기준이 있는 SKILL.md 작성
5. **표면화**: 아직 안전하게 인코딩하기 모호한 것 명시

## 규칙

- **실제로 반복 가능한** 워크플로우만 잡기
- 스킬을 **실용적이고 좁게** 유지
- 모호한 산문 대신 **명시적 성공 기준** 우선
- 워크플로우에 미해결 분기 결정이 있다면 드래프트 전에 명시

## 출력

- 제안 스킬 이름
- 목표 위치 (레포 / 사용자 / 프로젝트)
- 드래프트 워크플로우 구조
- 미해결 질문 (있다면)

## 관련 스킬

- [learner](/ko/docs/skills/utility/learner) — 세션에서 재사용 통찰 추출
- [skill](/ko/docs/skills/utility/skill) — 로컬 스킬 관리 (list / add / remove / search / edit)
- [remember](/ko/docs/skills/utility/remember) — 지식을 올바른 메모리 표면으로 라우팅
