## 概览

AI 冗余清理器是一个有界的清理工作流，从 AI 生成的代码中移除不必要的复杂性（冗余）同时保留行为。它优先删除而非添加，专注于简化而非新功能。

## 使用

```bash
/oh-my-claudecode:ai-slop-cleaner <target>
/oh-my-claudecode:ai-slop-cleaner <target> --review
```

也可以通过魔法关键词激活：

```
deslop this module: too many wrappers
anti-slop cleanup src/auth
```

## 冗余分类

清理前，对代码中的冗余类型进行分类。

| 类型 | 说明 |
|------|-------------|
| Duplication | 重复逻辑、复制粘贴分支、冗余助手 |
| Dead code | 未使用的代码、无法到达的分支、陈旧标志、调试残留 |
| Unnecessary abstraction | 直通包装器、推测性间接、单一用途的助手层 |
| Boundary violations | 隐藏耦合、职责错位、错误层导入 |
| Missing tests | 代码缺乏锁定行为、弱回归覆盖 |

## 工作流

### 1. 保护当前行为

在做出更改前添加或运行回归测试。如果测试不能先进行，明确记录验证计划。

### 2. 编写清理计划

限制在请求的文件/功能区域、列出要删除的具体冗余、按从最安全删除到风险较高的集成排序。

### 3. 按 Pass 执行

- **Pass 1**：删除死代码
- **Pass 2**：移除重复
- **Pass 3**：清理名称和错误处理
- **Pass 4**：加强测试

在每个 pass 后运行有针对性的验证。

### 4. 质量门

运行回归测试、lint、类型检查和单元/集成测试。如果门失败，修复问题或回滚风险清理。

## Review 模式（`--review`）

`--review` 是清理工作后的一个单独审查者 pass。

- 审查清理计划、更改的文件和回归覆盖，不编辑文件
- 识别剩余的死代码、重复、不必要的包装器和缺失的测试
- 呈现审查者判定和任何必要的后续操作
