## 概要

`skillify` は成功したマルチステップワークフローを具体的なスキルドラフトとしてキャプチャし、後で再発見するのではなく。現在のセッションが反復可能なパターンを発見し、それが再利用可能な OMC スキルになるべき場合に使用します。

これは [learner](/ja/docs/skills/utility/learner) の自然な補完です — `learner` はセッション経験から洞察を抽出し、`skillify` はインストール準備ができた具体的な `SKILL.md` を生成します。

## 使用方法

```bash
/oh-my-claudecode:skillify
```

機能したワークフローを説明します。スキルはそれを再利用可能な SKILL.md に構造化するプロセスをガイドします。

## ワークフロー

1. **特定する** セッションが達成した反復可能なタスク
2. **抽出する** 構造化された部分：
   - 入力
   - 順序付きステップ
   - 成功基準
   - 制約 / 落とし穴
   - スキルの最適な目標ロケーション
3. **決定する** ワークフローは以下のいずれかに属するかどうか：
   - リポ組み込みスキル
   - ユーザー/プロジェクト学習スキル（`.omc/skills/`）
   - ドキュメンテーションのみ
4. **ドラフト** SKILL.md コンテンツ（クリアなトリガー、ステップ、成功基準付き）
5. **サーフェース** 安全にエンコードするのはまだあまりにもファジーなもの

## ルール

- **実際に反復可能な**ワークフローのみをキャプチャ
- スキルを**実用的でスコープ内**に保つ
- **明示的な成功基準**をあいまいなプロージュより優先
- ワークフローがまだ未解決のブランチング決定を持っている場合、ドラフト作成前にそれらをノート

## 出力

- 提案されたスキル名
- ターゲットロケーション（リポ / ユーザー / プロジェクト）
- ドラフトワークフロー構造
- 開いている質問（ある場合）

## 関連スキル

- [skill](/ja/docs/skills/utility/skill) — ローカルスキル（リスト / 追加 / 削除 / 検索 / 編集）を管理
