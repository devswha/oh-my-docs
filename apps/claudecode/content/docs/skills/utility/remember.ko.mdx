## 개요

`remember`는 채팅 히스토리에 묻혀 있는 지식을 **올바른 메모리 표면**에 영구 보존합니다. 세션에서 보존할 가치가 있는 것 — 프로젝트 사실, 컨벤션, 함정 — 을 발견했을 때 이 스킬로 적절한 곳에 라우팅하세요.

## 사용법

```bash
/oh-my-claudecode:remember
```

보존하고 싶은 내용을 설명하면, 스킬이 각 항목을 분류하고 최선의 저장 위치를 제안합니다.

## 메모리 표면

| 표면 | 용도 | 수명 |
|---|---|---|
| **Project memory** (`.omc/project-memory.json`) | 영구 팀/프로젝트 지식 | 세션 간 유지 |
| **Notepad priority** (`.omc/notepad.md`) | 다음 턴을 위한 짧고 신호 강한 컨텍스트 | 압축에서 살아남음 |
| **Notepad working** | 임시 활성 세션 노트 | 세션 단위 |
| **Docs / AGENTS / CLAUDE 파일** | 영구 지시사항과 컨벤션 | 영구, 버전 관리됨 |

## 워크플로우

1. 관련 세션 발견 사항 수집
2. 각 항목 분류:
   - 영구 프로젝트 사실
   - 임시 작업 노트
   - 운영자 선호/지시사항
   - 중복 / 오래된 / 충돌하는 정보
3. 각 항목에 최선의 목적지 제안
4. 적절한 메모리 표면만 작성/갱신
5. 정리해야 할 중복/충돌 명시

## 규칙

- **모든 것을 한 저장소에 던지지 말 것**
- 영구 팀 지식은 **project memory** 우선
- 단명 작업 컨텍스트는 **notepad** 우선
- 항목은 **간결하고 실행 가능하게** 유지
- 불확실한 것은 사실로 저장하지 말고 불확실하다고 표시

## 출력

- 무엇이 저장됐는지
- 어디에 저장됐는지 (어느 표면)
- 발견된 중복/충돌
