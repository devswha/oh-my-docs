## 概要

`omc-reference` は **内部で自動ロードされるリファレンススキル**で、詳細な OMC カタログをインラインコンテキストとして配信します。これはユーザーが呼び出すワークフローではなく、オーケストレータが詳細なカタログ情報が必要とされるときに自動的にロードされます。それはすべての `CLAUDE.md` セッションに存在する必要がありません。

リファレンスが自動ロードされるユースケース：

- エージェントへの委譲
- OMC ツール（状態、notepad、プロジェクト記憶、LSP、AST）の使用
- チームのオーケストレーション
- コミットプロトコルの遵守
- スキルの呼び出し

## フロントマター

```yaml
---
name: omc-reference
description: OMC エージェントカタログ、利用可能なツール、チームパイプラインルーティング、コミットプロトコル、スキルレジストリ。エージェントへの委譲、OMC ツール使用、チーム オーケストレーション、コミット作成、スキル呼び出しの際に自動ロード。
user-invocable: false
---
```

`user-invocable: false` は **このスキルを直接呼び出すスラッシュコマンドがない** ことを意味します。これは `CLAUDE.md` をコンパクトに保ちながら、重要な瞬間 — 委譲、コミット、またはツール呼び出しを行う際 — オーケストレータに完全なカタログを提供するために存在します。

## 含まれる内容

- **エージェントカタログ** — 19 個すべての OMC エージェント（デフォルトモデル付き）
- **モデルルーティング** — `haiku` / `sonnet` / `opus` 選択ガイドライン
- **ツール参照** — 外部 AI オーケストレーション、OMC 状態、チームランタイム、notepad、プロジェクト記憶、コード知能（LSP、AST）
- **チームパイプライン** — `team-plan → team-prd → team-exec → team-verify → team-fix` ステージルーティング
- **コミットプロトコル** — Git トレーラーフォーマット（Constraint、Rejected、Directive、Confidence、Scope-risk、Not-tested）
- **スキルレジストリ** — ワークフロー対ユーティリティ分類

## 存在する理由

`CLAUDE.md` ファイルはすべての Claude セッションにロードされます。これらをスリムに保つことはコンテキスト予算にとって重要です。`omc-reference` は重いカタログの詳細を保持するため、実際に関連性がある瞬間 — オーケストレータが委譲、コミット、またはツール呼び出しを決定する際 — にのみコンテキストに入ります。
