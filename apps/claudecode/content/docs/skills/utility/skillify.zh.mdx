## 概述

`skillify` 捕获成功的多步工作流作为具体的技能草稿，而不是稍后重新发现它。当当前会话发现应该成为可重用 OMC 技能的可重复模式时使用它。

这是 [learner](/zh/docs/skills/utility/learner) 的自然补充 — `learner` 从会话经验中提取见解，而 `skillify` 生成准备安装的具体 `SKILL.md`。

## 使用

```bash
/oh-my-claudecode:skillify
```

描述有效的工作流。该技能引导你将其构建为可重用的 SKILL.md。

## 工作流

1. **识别**会话完成的可重复任务
2. **提取**结构化部分：
   - 输入
   - 有序步骤
   - 成功标准
   - 约束 / 陷阱
   - 技能的最佳目标位置
3. **决定**工作流是否属于：
   - 一个仓库内置技能
   - 一个用户/项目学习技能（`.omc/skills/`）
   - 仅文档
4. **草稿** SKILL.md 内容，包含清晰的触发器、步骤和成功标准
5. **指出**仍然太模糊而无法安全编码的任何内容

## 规则

- 仅捕获**真正可重复的**工作流
- 保持技能**实用且作用域明确**
- 优先选择**显式成功标准**而不是模糊的文字
- 如果工作流仍然存在未解决的分支决策，在草稿前注明

## 输出

- 提议的技能名称
- 目标位置（仓库 / 用户 / 项目）
- 草稿工作流结构
- 开放性问题（如有）

## 相关技能

- [learner](/zh/docs/skills/utility/learner) — 从会话中提取可重用见解
- [skill](/zh/docs/skills/utility/skill) — 管理本地技能（列出 / 添加 / 删除 / 搜索 / 编辑）
- [remember](/zh/docs/skills/utility/remember) — 将知识路由到正确的记忆表面
