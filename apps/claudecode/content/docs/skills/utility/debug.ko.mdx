## 개요

`debug`는 OMC/Claude-Code 세션 문제, 워크플로우 고장, 혼란스러운 런타임 동작을 진단하는 집중 스킬입니다. 진짜 실패 신호를 빠르게 찾고 다음 수정 단계를 설명합니다.

`debugger` 에이전트(애플리케이션 코드의 근본 원인 분석)와 달리, `debug` 스킬은 **OMC 런타임 자체** — 훅, 에이전트, 상태 파일, 오케스트레이션 배관 — 을 위해 최적화돼 있습니다.

## 사용법

```bash
/oh-my-claudecode:debug
```

증상을 자연어로 설명하면, 스킬이 가장 관련 있는 로컬 증거부터 검사합니다.

## 워크플로우

1. 이슈 설명을 주의 깊게 읽기
2. 가장 관련 있는 로컬 증거를 먼저 검사:
   - **trace 도구** — `trace_summary`, `trace_timeline`
   - **state 도구** — `state_read`, `state_list_active`, `state_get_status`
   - **notepad / project memory** — 실패와 관련 있을 때
   - **실패한 테스트나 명령** — 직접 재현
3. 가능하다면 이슈를 좁게 재현
4. 증상과 근본 원인을 구분
5. 가장 작은 다음 수정 또는 검증 단계 추천

## 출력 형식

```
관측된 실패:    <사용자가 보는 것>
근본 원인 가설: <현재 최선의 설명>
증거:           <가설을 뒷받침하는 것>
다음 액션:      <구체적인 다음 단계>
```

## 규칙

- **추측보다 실제 증거 우선**
- 이슈가 오케스트레이션, 훅, 에이전트 흐름과 관련 있을 때 trace/state 표면 사용
- 앱 코드가 아닌 제품/런타임 버그라면 **명확히 말할 것**
- 실패를 격리하기 전에 광범위한 재작성을 처방하지 말 것
