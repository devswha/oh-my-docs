## 概览

Learner 从经验中提取可复用的技能，如困难的错误修复、特定于代码库的变通方法和浪费时间的陷阱。它存储的不仅仅是简单的代码片段，而是原则和决策启发式，教会 Claude 如何思考某种问题。

## 使用

```bash
/oh-my-claudecode:learner
```

最适用于这些情况：

- 解决需要深入调查的棘手错误后
- 发现该代码库特有的非显而易见的变通方法后
- 发现会浪费时间的隐藏陷阱后
- 发现影响项目的未记录行为后

## 提取标准

不是每个解决方案都值得保存为技能。

| 标准 | 好例子 | 坏例子 |
|-----------|-------------|-------------|
| 不可搜索 | 该代码库中的 ESM 自定义路径解析 | 用 TypeScript 读取文件 |
| 上下文特定 | 服务器上的 aiohttp 代理崩溃。py:42 | 用 try/catch 处理错误 |
| 精确的操作指导 | 在 tsconfig.json 中检查 moduleResolution | 处理边界情况 |
| 来之不易的知识 | worker.ts:89 处的竞争条件 | 通用编程模式 |

## 技能存储位置

- **用户级**：`~/.claude/skills/omc-learned/` - 在所有项目间可用（罕见）
- **项目级**：`.omc/skills/` - 仅在此仓库中使用（默认）

## 技能格式

```markdown
# [技能名称]

## 洞察
发现的基本原则。

## 为什么这很重要
如果不知道这个会发生什么。

## 识别模式
你如何知道何时应用此技能。

## 方法
决策启发式。如何思考，而非代码。
```
