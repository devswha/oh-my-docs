## 概述

`verify` 将模糊的"应该有效"声明转换为**具体证据**。当你想在声称完成前有信心一个功能、修复或重构真的有效时使用它。

这个技能是 `verifier` 智能体的轻量级对应物 — 它是一个你可以直接调用而不生成子智能体的聚焦工作流。

## 使用

```bash
/oh-my-claudecode:verify
```

说明必须证明的确切行为。该技能按优先级顺序引导你通过验证选项。

## 工作流

1. **识别**必须证明的确切行为
2. 优先选择**现有测试**
3. 如果覆盖缺失，运行可用的**最狭隘的直接验证命令**
4. 如果直接自动化不足，描述**手动验证步骤**并收集具体可观察证据
5. **仅报告实际验证的内容**

## 验证顺序

1. 现有测试（`npm test`、`pytest` 等）
2. 类型检查 / 构建（`tsc`、`npm run build`）
3. 狭隘的直接命令检查（`curl`、`node -e ...` 等）
4. 手动或交互式验证

## 规则

- **不要在没有证据的情况下说一个更改是完整的**
- 如果检查失败，**清楚地包括失败**
- 如果不存在现实的验证路径，**明确说出这一点**而不是虚张声势
- 优先选择简洁的证据摘要而不是嘈杂的日志

## 输出格式

```
验证：           <证明了什么>
运行的命令：       <执行了什么>
通过：             <什么通过了>
失败/未验证：  <什么没有通过或没有被检查>
```
