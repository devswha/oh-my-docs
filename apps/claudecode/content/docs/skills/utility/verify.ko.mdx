## 개요

`verify`는 "동작할 것 같다"는 모호한 주장을 **구체적 증거**로 바꿉니다. 기능, 수정, 리팩토링이 실제로 동작한다는 확신이 필요할 때 사용하세요.

이 스킬은 `verifier` 에이전트의 가벼운 대응물입니다 — 서브에이전트를 띄우지 않고 직접 호출 가능한 집중 워크플로우입니다.

## 사용법

```bash
/oh-my-claudecode:verify
```

증명해야 할 동작을 명시하면, 스킬이 우선순위 순서로 검증 옵션을 안내합니다.

## 워크플로우

1. **식별**: 증명해야 할 정확한 동작
2. **기존 테스트 우선**
3. 커버리지가 없다면 가능한 한 **가장 좁은 직접 검증 명령** 실행
4. 직접 자동화로 부족하면 **수동 검증 단계**를 설명하고 관측 가능한 증거 수집
5. **실제로 검증된 것만 보고**

## 검증 순서

1. 기존 테스트 (`npm test`, `pytest` 등)
2. 타입체크 / 빌드 (`tsc`, `npm run build`)
3. 좁은 직접 명령 검사 (`curl`, `node -e ...` 등)
4. 수동 또는 대화형 검증

## 규칙

- **증거 없이 완료라고 말하지 말 것**
- 검사가 실패하면 **실패를 명확히 포함**
- 현실적 검증 경로가 없다면 허세 대신 **명시적으로 말할 것**
- 노이즈 로그보다 간결한 증거 요약 우선

## 출력 형식

```
검증됨:        <증명된 것>
실행 명령:     <무엇이 실행됐나>
통과:          <통과한 것>
실패/미검증:   <통과 못 한 것 또는 검사 안 한 것>
```
