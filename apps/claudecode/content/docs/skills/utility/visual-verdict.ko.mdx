## 개요

Visual Verdict는 생성된 UI 스크린샷을 레퍼런스 이미지와 비교하고, 다음 편집 반복을 구동하는 엄격한 JSON 판정을 반환합니다. 시각적 충실도 요구사항에 대한 결정론적 합격/불합격 가이드를 제공합니다.

## 사용법

```bash
/oh-my-claudecode:visual-verdict
```

## 사용 시기

- 시각적 충실도 요구사항이 있는 작업 (레이아웃, 간격, 타이포그래피, 컴포넌트 스타일링)
- 생성된 스크린샷과 하나 이상의 레퍼런스 이미지가 있을 때
- 편집을 계속하기 전에 결정론적 합격/불합격 가이드가 필요할 때

## 입력

- `reference_images[]` — 하나 이상의 이미지 경로
- `generated_screenshot` — 현재 출력 이미지
- 선택사항: `category_hint` — 예: `hackernews`, `sns-feed`, `dashboard`

## 출력 형식

다음 형태의 **JSON만** 반환합니다:

```json
{
  "score": 87,
  "verdict": "revise",
  "category_match": true,
  "differences": [
    "상단 네비게이션 간격이 레퍼런스보다 좁음",
    "기본 버튼의 폰트 웨이트가 더 작음"
  ],
  "suggestions": [
    "네비게이션 항목 가로 패딩을 4px 증가",
    "기본 버튼 font-weight를 600으로 설정"
  ],
  "reasoning": "핵심 레이아웃은 일치하지만 스타일 세부사항에서 차이가 있습니다."
}
```

### 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `score` | integer 0-100 | 시각적 충실도 점수 |
| `verdict` | string | `pass`, `revise`, 또는 `fail` |
| `category_match` | boolean | 생성된 출력이 의도된 UI 카테고리와 일치하는지 |
| `differences` | string[] | 구체적 시각적 불일치 (레이아웃, 간격, 타이포그래피, 색상) |
| `suggestions` | string[] | 차이점에 연결된 실행 가능한 다음 편집 |
| `reasoning` | string | 1-2문장 요약 |

## 합격 기준

- 목표 합격 임계값은 **90점 이상**
- `score < 90`이면 편집을 계속하고 추가 시각적 검토 전에 visual-verdict를 다시 실행
- 다음 스크린샷이 임계값을 통과할 때까지 시각적 작업을 완료로 처리하지 **않음**

## 디버그 시각화

불일치 진단이 어려울 때:

1. visual-verdict를 권위 있는 결정으로 유지
2. 픽셀 수준 diff 도구(pixelmatch 오버레이)를 **보조 디버그 도구**로 사용하여 핫스팟 위치 파악
3. 픽셀 diff 핫스팟을 구체적인 `differences[]`와 `suggestions[]` 업데이트로 변환
