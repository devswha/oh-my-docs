## 概要

キャンセルは、現在アクティブな OMC モードを自動検出してキャンセルする統一キャンセルスキルです。リンクされたモードを依存関係を尊重しながら正しい順序でクリーンアップします。

これは任意の OMC モードを終了する標準的な方法であり、作業完了後に状態をクリーンアップするためにも使用されます。

## 使用方法

```bash
/oh-my-claudecode:cancel
/oh-my-claudecode:cancel --force
```

マジックキーワードでも有効化できます：

```
cancelomc
stopomc
```

## 自動検出順序

キャンセルは次の順序でアクティブなモードを検出してキャンセルします：

1. Autopilot（リンクされた ralph/ultraqa を含む）
2. Ralph（リンクされた ultrawork を含む）
3. Ultrawork（スタンドアロン）
4. UltraQA（スタンドアロン）
5. Ultrapilot（スタンドアロン）
6. Pipeline（スタンドアロン）
7. Team（Claude Code ネイティブ）
8. OMC Teams（tmux CLI ワーカー）
9. Plan Consensus（スタンドアロン）

## フォースモード

`--force` または `--all` を使うとすべてのセッションとレガシー成果物を完全に削除します。

```bash
/oh-my-claudecode:cancel --force
```

ワークスペースをリセットしたい場合に使用してください。

## モード別キャンセル動作

### Autopilot
現在のフェーズを保存して非アクティブ化します。リンクされた ralph と ultraqa もクリーンアップします。再開可能です。

### Ralph
永続モードを非アクティブ化します。リンクされた ultrawork が存在する場合はクリーンアップします。

### Team
各チームメンバーに shutdown_request を送信し、応答を待ちます。確認後、TeamDelete でクリーンアップします。リンクされた ralph が存在する場合はクリーンアップします。

## 保持された状態

| モード | 状態保持 | 再開コマンド |
|------|---------|-----------|
| Autopilot | はい（フェーズ、ファイル、スペック、計画） | `/oh-my-claudecode:autopilot` |
| Ralph | いいえ | - |
| Ultrawork | いいえ | - |
| Team | いいえ | - |
| Plan Consensus | はい（計画ファイルパス） | - |

## 関連スキル

すべてのワークフロースキルはキャンセルで終了できます。
