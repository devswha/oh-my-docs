## 概要

AI スロップ クリーナーは、AI が生成したコードから不要な複雑さ（スロップ）を削除しながら動作を保持する制限されたクリーンアップ ワークフローです。削除を追加より優先し、新機能よりも単純化に焦点を当てます。

## 使用方法

```bash
/oh-my-claudecode:ai-slop-cleaner <target>
/oh-my-claudecode:ai-slop-cleaner <target> --review
```

マジックキーワードでも有効化できます：

```
deslop this module: too many wrappers
anti-slop cleanup src/auth
```

## スロップ分類

クリーニング前にコード内のスロップの種類を分類します。

| 種類 | 説明 |
|-----|------|
| 重複 | 繰り返されるロジック、コピペされたブランチ、冗長なヘルパー |
| デッドコード | 未使用コード、到達不可能なブランチ、古いフラグ、デバッグの残り |
| 不要な抽象化 | パススルーラッパー、推測的インダイレクション、単一使用のヘルパーレイヤー |
| 境界違反 | 隠れたカップリング、誤った責任配置、不正なレイヤーインポート |
| テストの欠落 | ロック解除されていないコード動作、弱い回帰カバレッジ |

## ワークフロー

### 1. 現在の動作を保護

変更前に回帰テストを追加または実行します。テストを先に実行できない場合は、検証計画を明示的に文書化します。

### 2. クリーンアップ計画を作成

要求されたファイル/機能領域にスコープを限定し、削除する具体的なスロップをリストアップし、最も安全な削除からより危険な統合へと順序付けします。

### 3. パスごとに実行

- **パス 1**: デッドコードを削除
- **パス 2**: 重複を削除
- **パス 3**: 名前とエラーハンドリングをクリーンアップ
- **パス 4**: テストを強化

各パス後にターゲットを絞った検証を実行します。

### 4. 品質ゲート

回帰テスト、lint、型チェック、ユニット/統合テストを実行します。ゲートが失敗した場合、問題を修正するか危険なクリーンアップをロールバックします。

## レビューモード（`--review`）

`--review` はクリーンアップ作業後の別のレビューパスです。

- ファイルを編集せずにクリーンアップ計画、変更ファイル、回帰カバレッジをレビュー
- 残りのデッドコード、重複、不要なラッパー、欠落テストを特定
- レビュアーの検証と必要なフォローアップアクションを提示
