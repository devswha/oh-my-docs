## 概要

ラーナーは、厳しいバグ修正、コードベース固有の回避策、時間を浪費するトラップなどの経験から再利用可能なスキルを抽出します。単純なコードスニペットではなく、Claude が一種の問題をどう考えるかを教える原則と意思決定ヒューリスティックスを保存します。

## 使用方法

```bash
/oh-my-claudecode:learner
```

次の状況で最適に使用されます：

- 深い調査が必要な厳しいバグを解決した後
- このコードベース固有の自明でない回避策を発見した後
- 忘れられるともったいない隠れたトラップを発見した後
- プロジェクトに影響する文書化されていない動作を発見した後

## 抽出基準

すべてのソリューションがスキルとして保存する価値があるわけではありません。

| 基準 | 良い例 | 悪い例 |
|-----|--------|--------|
| 検索不可能 | このコードベースの ESM カスタムパス解決 | TypeScript でのファイル読み取り |
| コンテキスト固有 | server.py:42 での aiohttp プロキシクラッシュ | try/catch でエラーを処理 |
| 正確なアクション ガイダンス | tsconfig.json の moduleResolution を確認 | エッジケースを処理 |
| 苦労して得た知識 | worker.ts:89 での競合状態 | 一般的なプログラミングパターン |

## スキル保存位置

- **ユーザーレベル**: `~/.claude/skills/omc-learned/` - すべてのプロジェクト全体で利用可能（稀）
- **プロジェクトレベル**: `.omc/skills/` - このリポジトリでのみ使用（デフォルト）

## スキルフォーマット

```markdown
# [スキル名]

## インサイト
発見された基本原則。

## なぜこれが重要か
これを知らないと何が起こるか。

## 認識パターン
このスキルがいつ適用されるかを知る方法。

## アプローチ
意思決定ヒューリスティックス。コードではなく、考え方。
```
