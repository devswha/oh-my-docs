## 概要

`debug` は OMC/Claude Code セッション問題、ワークフロー破損、または混乱したランタイム動作に対する焦点を絞った診断スキルです。実際の失敗シグナルを素早く見つけ、次の是正ステップを説明します。

`debugger` エージェント（アプリケーションコードで根本原因分析を実行）とは異なり、`debug` スキルは **OMC ランタイム自体** — フック、エージェント、状態ファイル、オーケストレーション基盤に最適化されています。

## 使用方法

```bash
/oh-my-claudecode:debug
```

症状を自然言語で説明します。スキルは最も関連性の高いローカルエビデンスを最初に検査します。

## ワークフロー

1. 問題説明を注意深く読む
2. 最も関連性の高いローカルエビデンスを最初に検査：
   - **トレースツール** — `trace_summary`、`trace_timeline`
   - **状態ツール** — `state_read`、`state_list_active`、`state_get_status`
   - **notepad / プロジェクト記憶** — 失敗に関連する場合
   - **失敗したテストまたはコマンド** — 直接的な再現
3. 可能であれば問題を狭い範囲で再現
4. 症状と根本原因を区別
5. 最小限の次の修正または検証ステップを推奨

## 出力フォーマット

```
観察された失敗:    <ユーザーが見るもの>
根本原因の仮説: <現在の最良説明>
エビデンス:            <仮説を支持するもの>
最小限の次のアクション: <具体的な次のステップ>
```

## ルール

- **推測より実在するエビデンスを優先**
- オーケストレーション、フック、またはエージェントフローに関わる問題には trace/state サーフェスを使用
- 問題がアプリコードではなく製品/ランタイムバグである場合、**明確に言及する**
- 失敗を分離する前に広い書き換えを処方しない
