## 概要

`remember` は耐久性のある再利用可能な知識を**正しいメモリサーフェース**に昇格させ、チャット履歴に埋もれたままにするのではなく。セッションがプロジェクトファクト、コンベンション、落とし穴など保存する価値のあるものを発見したときに使用し、正しい宛先にルーティングしたい場合。

## 使用方法

```bash
/oh-my-claudecode:remember
```

保存したいものを説明します。スキルが各アイテムを分類し、最適なストレージロケーションを提案します。

## メモリサーフェース

| サーフェース | 目的 | ライフタイム |
|---|---|---|
| **プロジェクトメモリ**（`.omc/project-memory.json`） | 耐久的なチーム/プロジェクト知識 | セッション全体で永続 |
| **ノートパッド優先度**（`.omc/notepad.md`） | 次のターンのための短い高信号コンテキスト | コンパクション後も保持 |
| **ノートパッド作業中** | 一時的な活動セッションノート | セッション範囲 |
| **ドキュメント / AGENTS / CLAUDE ファイル** | 耐久的な指示とコンベンション | 永続的、バージョン管理 |

## ワークフロー

1. 関連するセッション発見を集める
2. 各アイテムを分類する：
   - 耐久的なプロジェクトファクト
   - 一時的な作業中ノート
   - オペレーター好みまたは指示
   - 重複 / 陳腐化 / 矛盾する情報
3. 各アイテムの最適な宛先を提案する
4. 適切なメモリサーフェースのみを書き込むまたは更新する
5. クリーンアップすべき重複または矛盾を呼び出す

## ルール

- **すべてを 1 つのストアに捨てないでください**
- 耐久的なチーム知識には**プロジェクトメモリ**を優先
- 短命な作業コンテキストには**ノートパッド**を優先
- エントリを**簡潔で実行可能**に保つ
- 何か不確実な場合、それをファクトとして保存するのではなく、不確実とマークする

## 出力

- 何が保存されたか
- それが保存されたところ（どのサーフェース）
- 発見された重複/矛盾
