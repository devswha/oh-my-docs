## 概述

Project Session Manager（PSM）使用 git worktree 和 tmux 会话自动化管理格离化开发环境。它允许你并行独立地进行 PR 审查、问题修复和功能开发。

如果只需要快速创建 worktree，也可以使用 `omc teleport` 命令。

## 使用

```bash
/oh-my-claudecode:project-session-manager review <ref>
/oh-my-claudecode:project-session-manager fix <ref>
/oh-my-claudecode:project-session-manager feature <project> <name>
/oh-my-claudecode:project-session-manager list
/oh-my-claudecode:project-session-manager attach <session>
/oh-my-claudecode:project-session-manager kill <session>
/oh-my-claudecode:project-session-manager cleanup
/oh-my-claudecode:project-session-manager status
```

别名 `psm` 也可用：

```bash
/psm review omc#123
/psm fix omc#42
/psm feature omc add-webhooks
```

## 项目参考格式

| 格式 | 示例 |
|--------|---------|
| 别名 | `omc#123` |
| 完整 | `owner/repo#123` |
| URL | `https://github.com/owner/repo/pull/123` |
| 当前仓库 | `#123` |

## 主要命令

### review

创建 PR 审查会话。检出 PR 分支，创建 worktree，启动 tmux 会话，运行 Claude Code。

### fix

创建问题修复会话。获取问题信息并创建 fix 分支。

### feature

创建功能开发会话。创建 feature 分支并设置 worktree。

### cleanup

自动清理已合并 PR 或已关闭问题的会话。

## Teleport 命令

比 PSM 更轻量级的 worktree 创建工具。创建 worktree 而不进行 tmux 会话管理。

```bash
omc teleport #123           # 为问题/PR 创建 worktree
omc teleport my-feature     # 为功能创建 worktree
omc teleport list           # 列出 worktree
omc teleport remove <name>  # 删除 worktree
```

## 提供方

| 提供方 | CLI | 参考格式 |
|----------|-----|-----------------|
| GitHub（默认） | `gh` | `owner/repo#123`、别名、URL |
| Jira | `jira` | `PROJ-123`、别名 |

## 配置

在 `~/.psm/projects.json` 中配置项目别名。

```json
{
  "aliases": {
    "omc": {
      "repo": "Yeachan-Heo/oh-my-claudecode",
      "local": "~/Workspace/oh-my-claudecode",
      "default_base": "main"
    }
  }
}
```

## 需求

- `git`（v2.5+ 带 worktree 支持）
- `jq`
- `tmux`（可选，推荐）
- `gh`（GitHub）或 `jira`（Jira）
