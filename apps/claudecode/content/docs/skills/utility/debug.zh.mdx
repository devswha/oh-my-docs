## 概览

`debug` 是一个聚焦的诊断技能，用于 OMC/Claude Code 会话问题、工作流断裂或令人困惑的运行时行为。它快速找到真实的失败信号并解释下一个纠正步骤。

与 `debugger` 智能体（对应用代码进行根本原因分析）不同，`debug` 技能针对 **OMC 运行时本身**优化 — 钩子、智能体、状态文件和编排管道。

## 使用

```bash
/oh-my-claudecode:debug
```

用纯文本描述症状。该技能首先检查最相关的本地证据。

## 工作流

1. 仔细阅读问题描述
2. 首先检查最相关的本地证据：
   - **trace 工具** — `trace_summary`, `trace_timeline`
   - **状态工具** — `state_read`, `state_list_active`, `state_get_status`
   - **notepad / 项目记忆** — 当与失败相关时
   - **失败的测试或命令** — 直接重现
3. 如果可能，狭义地重现问题
4. 区分症状和根本原因
5. 推荐最小的下一步修复或验证步骤

## 输出格式

```
观察到的失败：    <用户看到什么>
根本原因假设：    <目前的最佳解释>
证据：            <支持假设的内容>
最小下一步操作：  <具体下一步>
```

## 规则

- 偏好 **真实证据而非猜测**
- 当问题涉及编排、钩子或智能体流时使用 trace/state 表面
- 如果问题是产品/运行时错误而非应用代码，**直言不讳**
- 在隔离失败前不要规定广泛的重写
