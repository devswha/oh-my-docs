## MCP 도구란?

OMC는 Claude Code 에이전트에게 MCP(Model Context Protocol) 도구를 줍니다. 에이전트가 상태 관리, 메모 저장, 코드 분석, 데이터 처리에 씁니다.

사용자가 직접 호출하는 [스킬](/ko/docs/skills)과 달리, 도구는 에이전트가 작업 중에 내부적으로 사용합니다.

## 도구 카테고리

### State

실행 모드의 상태를 관리하는 도구들입니다.

`state_read`, `state_write`, `state_clear`, `state_list_active`, `state_get_status`

Autopilot, Ralph, Ultrawork 등의 실행 모드가 현재 상태를 저장하고 읽는 데 사용합니다.

### Notepad

컨텍스트 컴팩션에서 살아남는 메모 시스템입니다.

`notepad_read`, `notepad_write_priority`, `notepad_write_working`, `notepad_write_manual`, `notepad_prune`, `notepad_stats`

긴 세션에서 중요한 정보가 컨텍스트 윈도우에서 밀려나도 유지됩니다.

### Project Memory

프로젝트별 장기 메모리를 관리합니다.

`project_memory_read`, `project_memory_write`, `project_memory_add_note`, `project_memory_add_directive`

프로젝트 구조, 규칙, 학습 내용을 세션 간에 유지합니다.

### LSP

Language Server Protocol을 통한 코드 인텔리전스 도구입니다.

`lsp_hover`, `lsp_goto_definition`, `lsp_find_references`, `lsp_diagnostics`, `lsp_rename` 등 12개 도구

타입 정보, 정의 이동, 참조 찾기, 에러 진단, 심볼 검색 등을 할 수 있습니다.

### AST Grep

AST 기반 구조적 코드 검색 및 치환 도구입니다.

`ast_grep_search`, `ast_grep_replace`

정규식이 아닌 코드 구조 패턴으로 검색하고 치환합니다.

### Python REPL

지속적 Python REPL로 데이터 분석을 수행합니다.

`python_repl`

세션 간 상태가 유지되는 Python 환경입니다.

### Session Search

이전 로컬 세션 기록과 트랜스크립트 아티팩트를 검색합니다.

`session_search`

세션 ID, 타임스탬프, 소스 경로, 매칭 발췌문을 구조화된 JSON으로 반환합니다.

### Trace

에이전트 플로우 트레이스 데이터를 분석합니다.

`trace_timeline`, `trace_summary`

`trace_timeline`은 훅, 키워드, 스킬, 에이전트, 도구를 시간 순으로 표시합니다. `trace_summary`는 훅 통계, 키워드 빈도, 스킬 활성화, 모드 전환, 도구 병목을 집계합니다.

### Shared Memory

팀 협업을 위한 에이전트 간 공유 메모리입니다.

`shared_memory_write`, `shared_memory_read`, `shared_memory_list`, `shared_memory_delete`, `shared_memory_cleanup`

협업 워크플로우 중 에이전트들이 팀 경계를 넘어 데이터를 공유할 수 있게 합니다.

### Skills

내부 스킬 관리 도구입니다.

`load_omc_skills_local`, `load_omc_skills_global`, `list_omc_skills`

로컬 프로젝트와 전역 사용자 디렉터리에서 사용 가능한 OMC 스킬을 로드하고 나열합니다.
