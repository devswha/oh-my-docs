## 概要

Explore は、コードベース内のファイル、コード パターン、シンボル間の関係を素早く見つける探索専門エージェントです。
デフォルト モデルは haiku で、高速で安価であり、「物がどこにあるか」を見つけることに特化しています。

Explore は読み取り専用エージェントです。
ファイルを作成または変更せず、検索結果をメッセージ テキストとして返します。

内部コードベース探索のみを処理します。
外部ドキュメント、学術論文、API リファレンスの検索は `document-specialist` が処理します。

## 使用時期

- 特定の機能を実装するファイルを見つける必要がある場合
- コード パターンまたはシンボルが使用されている場所を追跡する場合
- プロジェクト構造とファイル間の関係を理解する場合
- 他のエージェント（planner、executor など）が作業開始前にコードベース コンテキストを収集する場合

## 使用例

```
"認証関連コードがどこにあるかを見つけよ"
"この関数を呼び出すすべてのファイルを見つけよ"
"プロジェクトのルーティング構造をマッピングする"
```

Explore は単一の検索ではなく、複数の角度から並列検索を実行します。
例えば、「認証関連コード」を見つけるときは、コントローラー、ミドルウェア、トークン検証、セッション管理などを同時に検索します。

## 使用ツール

- **Glob**: ファイル名/パターンでファイルを検出
- **Grep**: テキスト パターンでコード コンテンツを検索
- **ast_grep_search**: 関数シェイプやクラス構造などの構造的パターンを検索
- **lsp_document_symbols**: ファイルのシンボル概要を表示（関数、クラス、変数）
- **lsp_workspace_symbols**: ワークスペース全体でシンボル名を検索
- **Bash**: git コマンドで履歴/変更レコードをクエリ

## コンテキスト保存戦略

Explore はコンテキスト ウィンドウ スペースを節約します。

- 200 行を超えるファイルの場合、まず `lsp_document_symbols` で概要をチェック
- 500 行を超えるファイルの場合、ファイル全体の読み取りよりも構造的ツールを優先
- 大きなファイルの場合、`offset`/`limit` パラメータを使用して必要なセクションのみを読み取り
- 並列読み取りは一度に 5 ファイルまでに制限

## 他のエージェントとの組み合わせ

- **planner**: 計画作成前にコードベース コンテキストを収集するために最初に実行
- **executor**: 実装前に既存パターンと依存関係を理解するために explore エージェントを起動
- **architect**: アーキテクチャ分析中の初期ファイル マッピングに使用
- **debugger**: バグに関連するコードを素早く見つけるために使用
