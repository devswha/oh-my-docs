## 概述

Analyst 是一个智能体，将已定的产品范围转换为可实现的验收标准。
其核心目的是在规划开始前发现需求中的缺口。

它关注"这能被构建吗？"，不对"这应该被构建吗？"进行市场/价值判断。
在规划前发现需求缺口比在实现后发现便宜 100 倍。

Analyst 是一个只读智能体。

## 何时使用

- 在实现特性前验证需求的完整性
- 在查找缺失的问题和未定义的边界条件
- 在确认验收标准是否可测试
- 在提前识别范围扩张风险

## 使用示例

```
"分析用户删除特性的需求"
"找出这份 PRD 中缺少什么"
"总结这个特性的边界情况"
```

Analyst 对每个需求检查以下问题：
- 它是完整的吗？它可测试吗？它明确吗？
- 有哪些未验证的假设？
- 什么被明确地包括或排除在范围外？
- 有哪些前置依赖项？

## 输出格式

Analyst 以结构化格式生成分析报告。

- **缺失的问题**：未被提出的问题及其重要性
- **未定义的护栏**：需要边界定义的区域
- **范围风险**：潜在的范围扩张区域和预防策略
- **未验证的假设**：假设及验证方式
- **缺失的验收标准**：可测量的成功标准
- **边界情况**：异常场景和处理方式

## 与其他智能体的结合

- **planner**：Planner 根据 Analyst 的分析创建计划。Planner 在生成计划前咨询 Analyst
- **architect**：需要代码分析时委托给 architect
- **critic**：如果计划已存在，critic 审查它
