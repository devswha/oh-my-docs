## 概述

Writer 是一个创建开发者想要阅读的清晰、准确的技术文档的智能体。
它处理 README、API 文档、架构文档、用户指南和代码注释。

其默认模型是 haiku，使其运行快速且廉价。

核心原则是准确性。
不准确的文档比没有文档更糟 — 它将开发者引向错误的方向。
所有代码示例必须经过测试，所有命令必须经过验证。

## 何时使用

- 当编写 README 或项目文档时
- 当创建 API 文档时
- 当编写迁移说明或变更日志时
- 当添加代码注释时

## 使用示例

```
"Write a README for this project"
"Document the auth API endpoints"
"Add JSDoc comments to this module"
```

## 编写流程

1. **请求分析**：识别确切的文档任务
2. **代码库探索**：使用 Glob/Grep/Read 理解要文档化的代码（并行运行）
3. **检查现有文档风格**：理解现有文档的结构、风格和约定
4. **使用验证的代码示例编写文档**：测试所有示例和命令
5. **报告结果**：包括验证结果

## 编写原则

- **主动语态、直接语言、无不必要的词语**
- **可扫描的结构**：使用标题、代码块、表格和项目符号
- **准确的范围**：仅文档化请求的内容，防止范围蔓延
- **验证**：在包括代码示例和命令前对其进行测试。如果无法测试，显式说明

Writer 仅执行编写阶段。
它不在同一上下文中自我审查或自我批准。
当需要审查时，它传递给单独的审查/验证阶段。

## 与其他智能体组合

- **explore**：用于定位和理解要文档化的代码的结构
- **executor**：当需要文档更新和代码更改时与 executor 并行运行
- **document-specialist**：外部 API/SDK 文档查找由 document-specialist 处理
