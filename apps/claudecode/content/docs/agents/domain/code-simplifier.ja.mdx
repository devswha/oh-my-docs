## 概要

Code Simplifier は、明確性、一貫性、保守性を向上させながらコードの機能を完全に保持するエージェントです。
最近変更されたコードにプロジェクトのベストプラクティスを適用し、コードをクリーンでシンプルに保ちます。

明示的で読みやすいコードを過度に圧縮されたコードより優先します。
「どのように行うか」を変更するだけで、動作は変えません。

## 使用時機

- 最近書いたコードの可読性を向上させたいとき
- 不要な複雑性を減らしたいとき
- コードをプロジェクト標準に合わせるとき
- Stop フック により自動実行されるとき（オプトイン）

## 使用例

```
"Simplify the code I just wrote"
"Improve the readability of this file"
```

## コア原則

### 機能保持

元の機能、出力、動作を変更しません。
エクスポートされたシンボルを変更したり、関数シグネチャを変更したり、制御フローを並べ替えたりしません。

### プロジェクト標準を適用

- ES モジュール、適切なインポート順序、`.js` 拡張子
- トップレベル宣言には `function` キーワードを優先（アロー関数より）
- トップレベル関数に明示的な戻り値型アノテーション
- camelCase（変数）、PascalCase（型）命名
- TypeScript strict モード パターン

### 明確性の向上

- 不要な複雑性とネストを削減
- 重複するコードと抽象化を削除
- 明確な変数名・関数名で可読性を向上
- 関連するロジックを統合
- コードが明確に示すものを単に説明するコメントを削除
- **ネストされた三項演算子なし** — 代わりに `switch` または `if/else` チェーンを使用

### バランスの維持

以下の過度な簡素化を避けます：
- コード明確性や保守性を害する簡素化
- 理解しにくい過度に巧妙なソリューション
- 多くの責務を1つの関数に組み込むこと
- コード整理を改善する有用な抽象化の削除
- 可読性より「行数を減らす」ことを優先

## Stop フック 自動実行

Code Simplifier は Stop フック として自動実行できます（デフォルトは無効）。

`~/.omc/config.json` で有効化：

```json
{
  "codeSimplifier": {
    "enabled": true,
    "extensions": [".ts", ".tsx", ".js", ".jsx", ".py", ".go", ".rs"],
    "maxFiles": 10
  }
}
```

有効化すると、Claude が停止するたびに `git diff HEAD --name-only` で変更ファイルを検出し、
code-simplifier エージェントに委譲します。
ターンごとに 1 回だけ実行されます。

## 他のエージェントとの組み合わせ

- **executor**: Executor が実装を完了した後、Code Simplifier が可読性を向上
- **code-reviewer**: コードレビュー結果に基づくリファクタリングに使用
