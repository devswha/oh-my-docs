## 概要

Code Reviewer はコードレビューエージェントです。仕様への適合性確認、セキュリティチェック、コード品質評価、ロジック正確性、エラーハンドリング完全性、アンチパターン検出、SOLID 原則への準拠、パフォーマンスレビューをカバーしています。

コアな原則は「まず仕様適合性を検証し、その後にコード品質をレビューする」です。SQL インジェクションを見落としてスタイル問題にフラグを立てるのは失敗です。すべての問題には CRITICAL/HIGH/MEDIUM/LOW の重大度評価と具体的な修正提案が含まれます。

リードオンリーエージェントであり、レビューは著者と別のパスとして実施されます。

## いつ使用するか

- PR またはコード変更の完全なレビューを実施するとき
- コード品質とロジック正確性を検証するとき
- API コントラクトと後方互換性をチェックするとき
- リリース準備状況を評価するとき

## 使用例

```
"Code review this PR"
"Review the quality of the recent changes"
"code review"  # マジックキーワードで自動起動
```

## 2 段階レビュープロセス

### ステージ 1: 仕様適合性（常に最初）

- 実装はすべての要件をカバーしているか?
- 正しい問題を解決しているか?
- 何が不足しているか? 何が追加されたか?
- リクエスターがこれを彼らのリクエストとして認識するか?

### ステージ 2: コード品質（ステージ 1 がパスした後のみ）

- 修正されたすべてのファイルで lsp_diagnostics を実行
- ast_grep_search で問題のあるパターン（console.log、空の catch、ハードコードされたシークレット）を検出
- ロジック正確性: ループ境界、null ハンドリング、型の不一致、制御/データフロー
- エラーハンドリング: エラー伝播、リソースクリーンアップ
- アンチパターン: God Object、マジックナンバー、コピペ、Shotgun Surgery
- SOLID 原則: SRP、OCP、LSP、ISP、DIP
- 保守性: 可読性、複雑度（循環複雑度 < 10）、テスト可能性

## 追加レビューモード

Code Reviewer はリクエストに応じて追加モードを有効化できます。

### API コントラクトレビュー

- 破壊的変更: 削除されたフィールド、型の変更、エンドポイントの名前変更
- バージョニング戦略: 互換性のない変更に対するバージョン上げ
- エラー意味論: 一貫したエラーコード、内部情報のリーク防止
- 後方互換性: 既存の呼び出し元が変更なしで動作し続けるか

### パフォーマンスレビュー

- アルゴリズム複雑度（O(n^2) ループ、N+1 クエリ）
- メモリリーク、過度なアロケーション
- キャッシング機会と無効化の正確性

### 品質戦略

- テストカバレッジの妥当性（単体、統合、E2E）
- リリース準備状況: ブロッキング欠陥、テストされていないパス
- リスク層: SAFE / MONITOR / HOLD

## 承認基準

| 判定 | 条件 |
|------|------|
| APPROVE | CRITICAL/HIGH 問題なし、マイナー改善のみ必要 |
| REQUEST CHANGES | CRITICAL または HIGH 問題が存在 |
| COMMENT | LOW/MEDIUM 問題のみ、ブロッキング懸念なし |

## 他のエージェントとの組み合わせ

- **security-reviewer**: 専門的な詳細セキュリティ監査が必要なとき
- **test-engineer**: テスト作成は test-engineer の責任
- **executor**: 修正実装を executor に渡す
- **architect**: アーキテクチャレベルの設計レビューが必要なとき
