## フックとは

フックは Claude Code のライフサイクルイベントに応答して自動的に実行されるスクリプトです。
oh-my-claudecode は複数のフックを使用して Claude Code のデフォルト動作を拡張します。

ユーザーがプロンプトを送信するとき、ツールが実行されるとき、またはセッションが開始/終了するとき、
フックは自動的に実行され、追加コンテキストを注入するか、モードを有効化するか、状態を管理します。

## どのように動作するか

フックは `hooks.json` ファイルで定義され、各フックは次の構造に従います：

```json
{
  "eventName": [
    {
      "matcher": "*",
      "hooks": [
        {
          "type": "command",
          "command": "node scripts/hook-script.mjs",
          "timeout": 5
        }
      ]
    }
  ]
}
```

- **eventName**: フックが応答するライフサイクルイベント
- **matcher**: フックを実行するための条件（`*` はすべてのケースを意味します）
- **command**: 実行する Node.js スクリプト
- **timeout**: 最大実行時間（秒単位）

フック出力は `<system-reminder>` タグを通じて Claude に注入されます。
追加コンテキストは `hookSpecificOutput.additionalContext` を通じて渡されます。

## フックカテゴリー

OMC フックは 4 つの主要カテゴリに分類されます。

### 実行モードフック

自律実行、並列処理、持続化などの実行モードを管理します。

| フック | 説明 |
|----|------|
| autopilot | アイデアから動作するコードまで完全に自律実行 |
| ralph | 検証完了まで持続的に実行 |
| ultrawork | 最大限の並列エージェント実行 |
| ultraqa | 目標が達成されるまで繰り返し QA サイクル実行 |
| persistent-mode | セッション間のモード状態を保持 |

### コアフック

オーケストレーションとキーワード検出を含むコア機能を処理します。

| フック | 説明 |
|----|------|
| keyword-detector | マジックキーワード（ultrawork、ralph など）を検出 |
| skill-injector | スキルプロンプトを注入 |
| pre-tool-enforcer | ツール使用前にルールを検証 |
| post-tool-verifier | ツール使用結果を検証 |

### コンテキスト管理フック

メモリ、プロジェクト状態、コンパクションを管理します。

| フック | 説明 |
|----|------|
| notepad | コンパクション耐性メモリシステム |
| project-memory | プロジェクトメモリ管理 |
| pre-compact | コンパクション前の処理 |

### 品質/検証フック

コード品質、パーミッション管理、サブエージェント追跡を処理します。

| フック | 説明 |
|----|------|
| permission-handler | パーミッションリクエストと検証 |
| subagent-tracker | サブエージェント起動/完了を追跡 |
| code-simplifier | Stop で最近変更したファイルを自動簡略化（オプトイン） |

## フックの無効化

### すべてを無効化

```bash
export DISABLE_OMC=1
```

### 選択的な無効化

```bash
export OMC_SKIP_HOOKS="keyword-detector,notepad"
```

特定のフックをスキップするにはカンマで区切ります。
