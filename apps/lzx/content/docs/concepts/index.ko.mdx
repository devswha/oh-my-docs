LazyCodex는 [OmO(oh-my-openagent)](https://github.com/code-yeongyu/oh-my-openagent)를 Codex에서 쓰기 좋게 묶은 배포판입니다. 설치 명령은 짧지만, 안쪽에는 Codex용 OmO 플러그인, 스킬, 훅, MCP 서버, Commands 기반 워크플로우가 들어 있습니다.

> _"LazyVim이 Neovim을 우리 모두가 쓸 수 있게 만들었듯, LazyCodex는 Codex를 그렇게 만듭니다."_

[lazy.nvim](https://github.com/folke/lazy.nvim)에 대한 [LazyVim](https://github.com/LazyVim/LazyVim)을, Codex 버전으로 생각하면 됩니다.

## Thin Distribution

LazyCodex 자체는 작은 설치 alias에 가깝습니다. `lazycodex-ai`가 Codex 플랫폼을 대상으로 OmO 설치를 실행하고, 실제 기능은 플러그인 `omo`가 제공합니다.

```text
lazycodex/
├── package.json       → lazycodex-ai 설치 alias
└── plugins/omo/       → Codex 플러그인, 스킬, 훅, MCP 서버
```

설치 절차는 [시작하기](/ko/docs/getting-started)에 두고, 이 섹션에서는 설치 뒤 Codex에 어떤 층이 더해지는지만 설명합니다.

## What Gets Installed

| Layer | Codex에서의 의미 |
| --- | --- |
| Commands | `$init-deep`, `$ulw-plan`, `$start-work`, `$ulw-loop` 같은 워크플로우 진입점 |
| Skills | 리뷰, 디버깅, 리팩터링, 프런트엔드, LSP, 규칙 주입 같은 전문 플레이북 |
| Hooks | 세션 시작, 프롬프트 제출, 편집 후, compact 후, stop 시점에 동작하는 자동 보조 장치 |
| MCP Servers | `grep_app`, `context7`, `codegraph`, `git_bash`, `lsp` 같은 도구 연결 |
