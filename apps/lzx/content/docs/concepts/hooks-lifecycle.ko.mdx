LazyCodex(LZX)는 Codex 세션의 정해진 지점에서 발동하는 라이프사이클 훅과 함께 OmO 플러그인을 제공합니다. 이 훅들은 프로젝트 규칙을 계속 로드해 두고, 설치 상태를 점검하며, 편집 후 피드백과 진단을 실행합니다. 아래 내용은 최신 `4.12.1` 플러그인 manifest와 훅 파일 기준입니다.

## Trigger Matrix

| 이벤트 | 발동하는 것 |
|---|---|
| `SessionStart` | 프로젝트 규칙 로딩, 텔레메트리, auto update 확인, bootstrap provisioning, CodeGraph bootstrap을 실행합니다. |
| `UserPromptSubmit` | 프로젝트 규칙을 다시 주입하고, ultrawork 트리거와 `$ulw-loop` steering을 확인합니다. |
| `PreToolUse` | Bash 호출에는 Git Bash MCP 안내를, goal 생성에는 `$ulw-loop` 예산 보호를 적용합니다. |
| `PostToolUse` | 주석 검사, LSP 진단, CodeGraph init 안내, `apply_patch` 규칙 매칭, thread title hygiene를 실행합니다. |
| `Stop` / `SubagentStop` | `$start-work` continuation과 LazyCodex executor evidence 검증을 실행합니다. |
| `PostCompact` | Git Bash 알림, 프로젝트 규칙, LSP 진단 캐시를 초기화합니다. |

## SessionStart

Codex 세션이 시작되면 기본 상태를 맞추는 훅들이 실행됩니다.

- **Rules** — `rules hook session-start`가 OMO 프로젝트 규칙을 로드해, 에이전트가 지원되는 rules와 instruction 파일을 이미 컨텍스트에 갖고 시작합니다.
- **Telemetry** — `telemetry hook session-start`가 세션 시작을 기록합니다.
- **Auto update** — 플러그인 업데이트 상태를 확인합니다.
- **Bootstrap provisioning** — 설치된 플러그인 캐시, 설정 블록, bin 링크 같은 LazyCodex bootstrap 상태를 확인합니다.
- **CodeGraph bootstrap** — CodeGraph 컴포넌트 초기화 상태를 확인합니다.

## UserPromptSubmit

프롬프트를 제출할 때마다 세 개의 훅이 실행됩니다.

- **Rules** — `rules hook user-prompt-submit`가 OMO 프로젝트 규칙을 다시 주입해, 대화가 길어져도 규칙이 계속 남아 있습니다.
- **Ultrawork 트리거** — 프롬프트에서 `ultrawork` / `ulw` 흐름을 감지합니다.
- **ulw-loop steering** — `$ulw-loop`가 이어지는 동안 필요한 steering 상태를 확인합니다.

## PreToolUse

도구 호출 직전에는 두 가지 보조 훅이 있습니다.

- **Git Bash MCP 안내** — Bash 도구가 호출될 때 Windows/Git Bash 경로에서 필요한 MCP 안내를 제공합니다.
- **Goal budget 보호** — goal 생성 도구가 호출될 때 `$ulw-loop`가 필요한 예산 제약을 유지합니다.

## PostToolUse

편집류 도구가 실행된 뒤 OmO가 그 결과를 검사합니다. 매처는 편집 계열을 대상으로 합니다.

```text
^(apply_patch|write|Write|edit|Edit|multi_edit|multiedit|MultiEdit)$
```

매칭되면 두 개의 훅이 실행됩니다.

- **comment-checker** — `comment-checker hook post-tool-use`가 방금 작성하거나 변경한 주석에 대해 피드백을 줍니다. [comment-checker](/ko/docs/skills/comment-checker)를 참고하세요.
- **LSP** — `lsp hook post-tool-use`가 편집된 코드에 language-server 진단을 실행합니다. [lsp](/ko/docs/skills/lsp)를 참고하세요.

더 좁은 두 번째 매처(`^apply_patch$`)는 추가로 `rules hook post-tool-use`를 실행해 패치에 대해 프로젝트 규칙을 매칭합니다.

편집류 도구가 아니어도 일부 후처리가 있습니다. CodeGraph 관련 도구 호출 뒤에는 CodeGraph init 안내가 실행되고, thread 생성 뒤에는 team mode 컴포넌트가 thread title hygiene을 확인합니다.

## Stop / SubagentStop

응답이나 하위 에이전트가 멈출 때도 continuation 훅이 실행됩니다.

- **start-work continuation** — `$start-work`가 계획을 끝까지 이어 가야 할 때 다음 진행 상태를 확인합니다.
- **executor evidence 검증** — `lazycodex-executor` 하위 에이전트가 끝날 때 증거가 충분한지 확인합니다.

## PostCompact

Codex가 컨텍스트를 컴팩션할 때 — `manual` 또는 `auto` — OmO는 Git Bash MCP 알림, 프로젝트 규칙, LSP 진단 캐시를 초기화합니다. 덕분에 오래된 컨텍스트가 요약되어도 다음 턴에서 필요한 안내와 진단 상태가 다시 정리됩니다.

## Installed Components

위 훅들은 설치된 컴포넌트로 들어가는 얇은 진입점입니다.

| 컴포넌트 | 담당 |
|---|---|
| `rules` | 세션 시작, 프롬프트 제출, `apply_patch`, 컴팩션 후 프로젝트 규칙 관리. |
| `bootstrap` | LazyCodex 설치 상태와 provisioning 확인. |
| `telemetry` | 세션 시작 기록. |
| `comment-checker` | 편집류 도구 이후의 주석 피드백. |
| `lsp` | 편집류 도구 이후의 language-server 진단과 compact 후 캐시 초기화. |
| `ultrawork` | 프롬프트 시점의 ultrawork 트리거 감지. |
| `ulw-loop` | loop steering과 goal budget 보호. |
| `start-work-continuation` | `$start-work` 실행 흐름 continuation. |
| `git-bash` | Bash 호출과 compact 후 Git Bash MCP 안내. |
| `codegraph` | CodeGraph bootstrap과 init guidance. |
| `teammode` | thread title hygiene 확인. |
| `lazycodex-executor-verify` | 하위 에이전트 evidence 검증. |
