---
title: Model Routing
description: How LazyCodex aligns model profiles with Codex roles.
---

LazyCodex (LZX) 会在安装过程中根据 Codex 配置和 agent roles 对齐模型 profile。实际应用的值来自插件内置的 `model-catalog.json`，以及当前 Codex build 支持的模型。

## Current Baseline

`4.12.1` 的 bundled `model-catalog.json` 将当前默认 profile 放在 `gpt-5.5` 上。

| Profile | Model | Reasoning |
|---|---|---|
| default | `gpt-5.5` | `high` |
| plan mode | `gpt-5.5` | `xhigh` |
| worker | `gpt-5.5` | `high` |
| verifier | `gpt-5.5` | `high` |

如果 Codex 或 OpenAI 的模型配置变化，实际看到的模型名可能不同。因此本文重点不是比较模型性能，而是说明 LazyCodex 在哪里使用模型 profile。

## Why Role Profiles Exist

角色 profile 用来分离工作性质。

- 普通工作使用默认模型设置。
- 计划模式可能需要更强 reasoning 设置。
- worker 与 verifier 分离执行和判断，让同一个结果能从不同角度检查。

这与 [Agent Roles](/zh/docs/concepts/discipline-agents) 配套。即使多个角色同时移动，也能在 Codex 配置中追踪每个角色使用的模型 profile。

## User Notes

- 安装后看到的模型名可能和文档不同。此时以 Codex 支持的最新模型名和 LazyCodex 的 `model-catalog.json` 为准。
- 模型设置同时影响质量和速度。随意降低它们，可能让规划、评审、QA 质量一起变弱。
- 如果怀疑有问题，先检查安装状态和 Codex 配置。
