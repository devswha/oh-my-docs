title: Model Routing
description: How LazyCodex aligns model profiles with Codex roles.

LazyCodex (LZX) は install 時に、Codex 設定と agent roles に合う model profiles を合わせます。この値は plugin に含まれる `model-catalog.json` と、利用中の Codex build がサポートする models によって適用されます。

## Current Baseline

`4.12.1` の bundled `model-catalog.json` は、現在の default profiles を `gpt-5.5` 中心に置いています。

| Profile | Model | Reasoning |
|---|---|---|
| default | `gpt-5.5` | `high` |
| plan mode | `gpt-5.5` | `xhigh` |
| worker | `gpt-5.5` | `high` |
| verifier | `gpt-5.5` | `high` |

Codex や OpenAI 側の model 構成が変わると、実際に見える model name は変わることがあります。そのため、このページは model performance の比較ではなく、LazyCodex が model profiles をどこで使うかに集中します。

## Why Role Profiles Exist

Role profiles は作業の性質を分けるための仕組みです。

- 一般作業は default model setting に従います。
- planning mode はより強い reasoning setting を要求することがあります。
- worker と verifier は実行と検証を分け、同じ結果を別の視点から確認します。

これは [Agent Roles](/ja/docs/concepts/discipline-agents) と組み合わさります。複数の roles が同時に動いても、各 role がどの model profile で動くかを Codex 設定で追跡できます。

## User Notes

- install 後に表示される model name は文書と異なることがあります。その場合、Codex がサポートする最新 model name と LazyCodex の `model-catalog.json` が優先です。
- model setting は品質と速度を同時に扱います。むやみに下げると、planning、review、QA の品質が一緒に揺らぎます。
- 問題が疑われる場合は、まず install state と Codex 設定を確認するのが安全です。
