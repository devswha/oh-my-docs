---
title: Hooks & Lifecycle
description: Lifecycle hooks for rules, bootstrap, diagnostics, and continuation.
---

LazyCodex (LZX) 随 OmO 插件提供 Codex 会话固定节点触发的 lifecycle hooks。这些 hooks 会持续加载项目规则、检查安装状态，并在编辑后运行反馈和诊断。以下内容基于最新 `4.12.1` plugin manifest 和 hook 文件。

## Trigger Matrix

| 事件 | 会触发什么 |
|---|---|
| `SessionStart` | 加载项目规则、记录 telemetry、检查 auto update、bootstrap provisioning、CodeGraph bootstrap。 |
| `UserPromptSubmit` | 重新注入项目规则，检查 ultrawork trigger 和 `$ulw-loop` steering。 |
| `PreToolUse` | 对 Bash 调用提供 Git Bash MCP 指引，并保护 `$ulw-loop` 的 goal budget 约束。 |
| `PostToolUse` | 运行 comment check、LSP diagnostics、CodeGraph init guidance、`apply_patch` rule matching、thread title hygiene。 |
| `Stop` / `SubagentStop` | 运行 `$start-work` continuation 和 LazyCodex executor evidence verification。 |
| `PostCompact` | 重置 Git Bash 提醒、项目规则、LSP diagnostic cache。 |

## SessionStart

Codex 会话开始时，hooks 会对齐基础状态。

- **Rules**：`rules hook session-start` 加载 OMO 项目规则，让智能体在开始时就拥有受支持的 rules 和 instruction 文件上下文。
- **Telemetry**：`telemetry hook session-start` 记录会话开始。
- **Auto update**：检查插件更新状态。
- **Bootstrap provisioning**：检查插件缓存、配置块、bin links 等 LazyCodex bootstrap 状态。
- **CodeGraph bootstrap**：检查 CodeGraph 组件初始化状态。

## UserPromptSubmit

每次提交 prompt 时会运行三个 hooks。

- **Rules**：`rules hook user-prompt-submit` 重新注入 OMO 项目规则，让长对话中规则也能保留。
- **Ultrawork trigger**：检测 prompt 中的 `ultrawork` / `ulw` 流程。
- **ulw-loop steering**：在 `$ulw-loop` 持续运行时检查 steering 状态。

## PreToolUse

工具调用前有两个辅助 hooks。

- **Git Bash MCP guidance**：Bash 工具被调用时提供 Windows/Git Bash MCP 指引。
- **Goal budget protection**：goal 创建工具被调用时，保留 `$ulw-loop` 所需预算约束。

## PostToolUse

编辑类工具运行后，OmO 会检查结果。matcher 针对编辑工具族：

```text
^(apply_patch|write|Write|edit|Edit|multi_edit|multiedit|MultiEdit)$
```

匹配时运行两个 hooks。

- **comment-checker**：`comment-checker hook post-tool-use` 对刚写入或修改的注释给出反馈。参见 [comment-checker](/zh/docs/skills/comment-checker)。
- **LSP**：`lsp hook post-tool-use` 对编辑后的代码运行 language-server diagnostics。参见 [lsp](/zh/docs/skills/lsp)。

更窄的第二个 matcher（`^apply_patch$`）还会运行 `rules hook post-tool-use`，用项目规则匹配 patch。

非编辑工具也可能有后处理。CodeGraph 相关工具调用后可运行 CodeGraph init guidance，thread 创建后 team mode 组件可检查 thread title hygiene。

## Stop / SubagentStop

响应或子智能体停止时也会运行 continuation hooks。

- **start-work continuation**：当 `$start-work` 必须执行到计划完成时，检查下一步状态。
- **executor evidence verification**：检查完成的 `lazycodex-executor` 子智能体是否提供了足够证据。

## PostCompact

Codex 手动或自动 compact 上下文时，OmO 会重置 Git Bash MCP 提醒、项目规则和 LSP diagnostic cache。旧上下文被摘要后，下一轮会重新获得所需指引和诊断状态。

## Installed Components

这些 hooks 是进入已安装组件的薄入口。

| 组件 | 负责内容 |
|---|---|
| `rules` | session start、prompt submit、`apply_patch`、compact 后的项目规则管理。 |
| `bootstrap` | LazyCodex 安装状态和 provisioning 检查。 |
| `telemetry` | 记录 session start。 |
| `comment-checker` | 编辑类工具后的注释反馈。 |
| `lsp` | 编辑类工具后的 language-server diagnostics，以及 compact 后 cache reset。 |
| `ultrawork` | prompt 时的 ultrawork trigger detection。 |
| `ulw-loop` | loop steering 和 goal budget protection。 |
| `start-work-continuation` | `$start-work` 执行流程 continuation。 |
| `git-bash` | Bash 调用和 compact 后的 Git Bash MCP 指引。 |
| `codegraph` | CodeGraph bootstrap 和 init guidance。 |
| `teammode` | thread title hygiene。 |
| `lazycodex-executor-verify` | 子智能体 evidence verification。 |
