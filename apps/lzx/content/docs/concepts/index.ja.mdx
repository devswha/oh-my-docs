LazyCodex は [OmO (oh-my-openagent)](https://github.com/code-yeongyu/oh-my-openagent) を Codex で使いやすい形にまとめた配布です。インストールコマンドは短いですが、中には Codex 用 OmO plugin、Skills、hooks、MCP servers、Commands ベースのワークフローが入っています。

> _"LazyVim が Neovim を皆に使えるものにしたように、LazyCodex は Codex に同じことをします。"_

[lazy.nvim](https://github.com/folke/lazy.nvim) に対する [LazyVim](https://github.com/LazyVim/LazyVim) の Codex 版だと考えてください。

## Thin Distribution

LazyCodex 自体は小さな install alias に近いものです。`lazycodex-ai` が Codex platform 向けに OmO install を実行し、実際の機能は `omo` plugin が提供します。

```text
lazycodex/
├── package.json       -> lazycodex-ai install alias
└── plugins/omo/       -> Codex plugin, Skills, hooks, MCP servers
```

インストール手順は [Getting Started](/ja/docs/getting-started) に置き、このセクションではインストール後に Codex へ加わる層だけを説明します。

## What Gets Installed

| Layer | Codex での意味 |
| --- | --- |
| Commands | `$init-deep`、`$ulw-plan`、`$start-work`、`$ulw-loop` などのワークフロー入口 |
| Skills | review、debugging、refactor、frontend、LSP、rules injection などの専門 playbook |
| Hooks | session start、prompt submit、edit 後、compact 後、stop 時点で動く自動補助 |
| MCP Servers | `grep_app`、`context7`、`codegraph`、`git_bash`、`lsp` などの tool 接続 |
