LazyCodex 将 [OmO (oh-my-openagent)](https://github.com/code-yeongyu/oh-my-openagent) 打包成适合在 Codex 中使用的发行层。安装命令很短，但里面包含 Codex 用的 OmO 插件、Skills、hooks、MCP servers，以及 Commands 工作流。

> _"就像 LazyVim 让更多人能轻松使用 Neovim，LazyCodex 对 Codex 做同样的事。"_

可以把它理解为面向 Codex 的 [LazyVim](https://github.com/LazyVim/LazyVim) for [lazy.nvim](https://github.com/folke/lazy.nvim)。

## Thin Distribution

LazyCodex 本身更像一个小型安装 alias。`lazycodex-ai` 针对 Codex 平台运行 OmO 安装，真正的功能由 `omo` 插件提供。

```text
lazycodex/
├── package.json       -> lazycodex-ai install alias
└── plugins/omo/       -> Codex plugin, Skills, hooks, MCP servers
```

安装步骤放在 [Getting Started](/zh/docs/getting-started)。本节只说明安装后 Codex 会增加哪些层。

## What Gets Installed

| Layer | 在 Codex 中的含义 |
| --- | --- |
| Commands | `$init-deep`、`$ulw-plan`、`$start-work`、`$ulw-loop` 等工作流入口 |
| Skills | review、debugging、refactor、frontend、LSP、rules injection 等专家 playbook |
| Hooks | session start、prompt submit、edit 后、compact 后、stop 时自动运行的辅助机制 |
| MCP Servers | `grep_app`、`context7`、`codegraph`、`git_bash`、`lsp` 等工具连接 |
