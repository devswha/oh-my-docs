LazyCodex(LZX)는 설치 과정에서 Codex 설정과 agent 역할에 맞는 모델 프로필을 맞춥니다. 이 값은 플러그인에 포함된 `model-catalog.json`과 사용 중인 Codex 빌드가 지원하는 모델에 따라 적용됩니다.

## Current Baseline

`4.12.1`의 bundled `model-catalog.json`은 현재 기본 프로필을 `gpt-5.5` 중심으로 둡니다.

| 프로필 | 모델 | 추론 설정 |
|---|---|---|
| 기본 | `gpt-5.5` | `high` |
| plan mode | `gpt-5.5` | `xhigh` |
| worker | `gpt-5.5` | `high` |
| verifier | `gpt-5.5` | `high` |

Codex나 OpenAI 쪽 모델 구성이 바뀌면 실제로 보이는 모델명은 달라질 수 있습니다. 그래서 이 문서는 특정 모델의 성능 비교보다, LazyCodex가 모델 프로필을 어디에 쓰는지에 집중합니다.

## Why Role Profiles Exist

역할별 프로필은 작업의 성격을 분리하기 위한 장치입니다.

- 일반 작업은 기본 모델 설정을 따릅니다.
- 계획 모드는 더 강한 추론 설정을 요구할 수 있습니다.
- worker와 verifier는 실행과 검증을 분리해, 같은 결과를 서로 다른 관점에서 확인하게 합니다.

이는 [Agent Roles](/ko/docs/concepts/discipline-agents)와 짝을 이룹니다. 여러 역할이 동시에 움직이더라도, 각 역할이 어떤 모델 프로필로 동작할지 Codex 설정에서 추적할 수 있습니다.

## User Notes

- 설치 후 보이는 모델명이 문서와 다를 수 있습니다. 이 경우 Codex가 지원하는 최신 모델명과 LazyCodex의 `model-catalog.json`이 우선입니다.
- 모델 설정은 품질과 속도를 함께 다루는 영역입니다. 임의로 낮추면 계획, 리뷰, QA 품질이 함께 흔들릴 수 있습니다.
- 문제가 의심되면 먼저 설치 상태와 Codex 설정을 확인하는 쪽이 안전합니다.
