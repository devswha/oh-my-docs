LazyCodex 的作用很简单。它把 OmO 设置接入 Codex，让 Codex 更清楚在项目中要记住什么、按什么顺序工作，以及应该在哪里停下来确认。

LazyCodex（LZX）是面向 Codex 的 OmO Light Edition。它不是把 [OmO（oh-my-openagent）](https://github.com/code-yeongyu/oh-my-openagent) 全部照搬过来，而是把更适合 Codex 的 Hephaestus 中心流程、命令、技能、钩子和辅助工具组合起来。

安装后，Codex 可以读取项目记忆，先做计划，再分步骤推进工作，最后用实际证据确认结果。相同设置之后也可以重新生成，所以一次整理好的工作方式可以反复使用。

> _可以把它理解为给 Codex 加上工作顺序和确认习惯的工具。_

## 能得到什么

| 功能 | 简单来说 |
| --- | --- |
| **项目记忆** | 在 `AGENTS.md` 中留下项目上下文，让 Codex 不必每次都从头猜。 |
| **先做计划** | 在直接改代码前，用 `$ulw-plan` 确定顺序和范围。 |
| **执行计划** | `$start-work` 按计划清单继续推进工作。 |
| **用证据确认** | `$ulw-loop` 和验证流程不是看“好像完成了”，而是看实际证据。 |
| **技能与钩子** | 接入 review、cleanup、frontend、programming 等专项流程和自动检查点。 |

## 先记住这些

一开始只需要记住四步。

1. 用 `$init-deep` 创建项目记忆。
2. 用 `$ulw-plan "要做什么"` 确定工作顺序。
3. 用 `$start-work` 执行计划。
4. 耗时较长或需要仔细确认的工作，用 `$ulw-loop "任务"` 交给它。

LazyCodex 会在这条流程上加入规则加载、技能、钩子、模型路由和验证习惯。更详细的结构，可以在侧边栏文档中按需查看。
