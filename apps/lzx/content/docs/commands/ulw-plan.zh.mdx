title: $ulw-plan
description: Prometheus 战略规划器：不写产品代码，在 .omo/plans 下写出决策完备的计划。
---

`$ulw-plan` 是 Prometheus 战略规划顾问。它把想法变成决策完备的工作计划。它是规划者，**不是**实现者。你说“做 X”时，它会为 X 创建计划，但绝不编写产品代码。

输出物是 `.omo/drafts/<slug>.md` 和 `.omo/plans/<slug>.md`。draft 保留探索和审批状态，作为可恢复点；plan 是 `$start-work` 可以直接执行的清单。

## 语法

```bash
$ulw-plan "what to build"
```

## 何时使用

当工作在实现前需要决策时使用 `$ulw-plan`：范围不清、可行方案不止一个，或需要在动代码前先排序的横切改动。

如果只是快速回答或小改动，正常提示即可。如果计划已经存在，直接进入 [`$start-work`](/zh/docs/commands/start-work)。

## 流程

<Steps>

<Step>

### 苏格拉底式访谈

Prometheus 会在规划前提出消除歧义所需的问题。

</Step>

<Step>

### 并行代码库探索

它并行探索相关代码，让计划建立在真实存在的代码之上。

</Step>

<Step>

### Metis 缺口分析

它执行缺口分析，寻找缺失的决策、边界情况和风险。

</Step>

<Step>

### 写出计划

工作计划会写入 `.omo/plans/<slug>.md`。脚手架脚本创建结构，计划内容写入指定区域。

</Step>

<Step>

### 评审与批准

需要时会经过 Metis/Momus 风格的评审。对于意图不清的任务，planner 可以采用 best-practice 默认值，并自动附加更强评审。没有用户批准，计划不会进入执行阶段。

</Step>

</Steps>

## 你会得到什么

问题、调研，以及一份可直接交给 `$start-work` 的 `.omo/plans/<slug>.md` 决策完备工作计划。

```bash
$ulw-plan "migrate the auth service to OAuth with refresh tokens"
# → 写入 .omo/plans/oauth-refresh-tokens.md
$start-work
```
