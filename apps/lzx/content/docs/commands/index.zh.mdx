LazyCodex 安装的是在 Codex 输入框中使用的 `$command`。它们不是 shell 命令，而是 Codex 会话里的工作流入口。安装后按下 `$` 时看到的项目就属于这里。

## 核心命令

| 命令 | 输入 | 作用 |
| --- | --- | --- |
| [`$init-deep`](/zh/docs/commands/init-deep) | `$init-deep` | 生成分层 `AGENTS.md` 项目记忆。适合大型或刚发生明显变化的代码库。 |
| [`$ulw-plan`](/zh/docs/commands/ulw-plan) | `$ulw-plan "what to build"` | 在实现前探索并提问，然后把决策完备的计划写入 `.omo/plans/<slug>.md`。不写产品代码。 |
| [`$start-work`](/zh/docs/commands/start-work) | `$start-work [plan-name] [--worktree <path>]` | 按复选框执行 `.omo/plans` 中的计划。完成时输出 **ORCHESTRATION COMPLETE**。 |
| [`$ulw-loop`](/zh/docs/commands/ulw-loop) | `$ulw-loop "task"` | 使用 `.omo/ulw-loop` 状态和证据账本持续工作，直到完成被验证。 |

## 如何组合

通常可以按三步理解：先创建项目记忆，需要决策的工作先规划，再执行计划。只有在没有计划、但开放目标必须被证据证明完成时，才使用 `$ulw-loop`。

<Steps>

<Step>

### 创建项目记忆

大型代码库先运行 `$init-deep`。后续命令会读取 `AGENTS.md` 层级，在更好的上下文中开始。

```bash
$init-deep
```

</Step>

<Step>

### 规划并执行

如果实现前需要决策，先写计划，再把计划交给 `$start-work`。

```bash
$ulw-plan "add OAuth login with refresh tokens"
$start-work
```

`$ulw-plan` 会访谈你、探索代码库，并在不触碰产品代码的情况下写入 `.omo/plans/<slug>.md`。随后 `$start-work` 逐个复选框执行该计划。

</Step>

<Step>

### 开放工作的验证完成

当工作必须持续推进，直到证据证明完成，而不是靠乐观状态汇报结束时，运行循环。

```bash
$ulw-loop "make the flaky payment tests pass"
```

智能体会继续工作并输出完成承诺，但循环只会在 Oracle 验证结果之后结束。验证失败则继续。

</Step>

</Steps>
