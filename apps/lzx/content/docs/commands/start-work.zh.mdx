---
title: $start-work
description: 按复选框执行 .omo/plans 中的 Prometheus 计划，并记录 Boulder 状态和证据账本。
---

`$start-work` 会执行 [Prometheus 工作计划](/zh/docs/commands/ulw-plan)，直到顶层复选框全部完成。执行对象是 `.omo/plans/<slug>.md`，进度记录在 `.omo/boulder.json`。

## 语法

```bash
$start-work [plan-name] [--worktree <absolute-path>]
```

- `plan-name` 可选。它可以是 `.omo/plans/<slug>.md` 中计划的文件名或部分名称。
- `--worktree <absolute-path>` 可选。在隔离的 git worktree 中执行工作。

## 工作方式

- `.omo/boulder.json` 中的 **Boulder 状态**会跨 turn 和 session 保留，执行之间不会丢失进度。
- **Stop-hook** 会在计划完成前重新注入下一轮；循环会自行延续。
- **独立子任务会展开到并行 subagents。**
- 在严格 TDD 之外还有**五个证据门**：重读计划、自动验证、手动 QA、对抗式 QA、清理。
- 每个复选框被勾选时，**证据会写入 `.omo/start-work/ledger.jsonl`**。
- 执行者不会直接相信子任务的“完成”报告；只有独立验证通过后才会把复选框标记为完成。

如果用户明确调用 `$start-work` 但没有可用计划，最新流程会先启动 `$ulw-plan` 创建可执行计划，然后进入执行。

## 完成

所有复选框都被勾选后，会输出 `ORCHESTRATION COMPLETE` 块。

```text
ORCHESTRATION COMPLETE
```

## 典型流程

<Steps>

<Step>

### 准备计划

用 `$ulw-plan` 写计划，或把现有 `.omo/plans/<slug>.md` 指给 `$start-work`。

```bash
$ulw-plan "add OAuth login with refresh tokens"
```

</Step>

<Step>

### 执行

```bash
$start-work oauth-refresh-tokens
```

独立工作会展开到并行 subagents，Boulder 进度可以承受中断。

</Step>

<Step>

### 等待完成

`$start-work` 只有在所有复选框完成并输出 `ORCHESTRATION COMPLETE` 时才停止。

</Step>

</Steps>
