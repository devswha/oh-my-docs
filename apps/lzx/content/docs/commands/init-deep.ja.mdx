---
title: $init-deep
description: 階層的な AGENTS.md プロジェクトメモリを生成し、以降のエージェントが完全なコンテキストで始められるようにします。
---

`$init-deep` は LazyCodex のプロジェクトメモリを構築します。リポジトリ全体に階層的な `AGENTS.md` ファイルを生成し、以降のエージェント実行が、これから編集するコードのすぐ隣で必要なコンテキストを持って始められるようにします。

リポジトリが大きすぎて記憶だけでは説明できないときに実行します。

## What It Does

<Steps>

<Step>

### Scores Complex Directories

`$init-deep` はコードベースを走査し、複雑さでディレクトリにスコアを付け、実際に文書化されたガイダンスが必要な箇所を見つけます。

</Step>

<Step>

### Writes Local Guidance Near Code

重要なディレクトリには、そのコードを説明する `AGENTS.md` を近くに書きます。ガイダンスは巨大なルートファイルに埋もれず、エージェントが作業する場所で見つかります。

</Step>

<Step>

### Leaves Landmarks for Future Agents

生成される `AGENTS.md` の階層は、以降のエージェントが編集に入る前の目印になります。ここに何があり、どう噛み合い、何に注意すべきかを示します。

</Step>

</Steps>

## When To Run It

- **コードベースを初めて扱うとき。** 不慣れな、または大規模なリポジトリで LazyCodex に実作業を任せる前に `$init-deep` を実行します。
- **記憶だけで説明するにはリポジトリが大きすぎるとき。** プロジェクトを一文で要約できないなら、`$init-deep` に地図を描かせます。

## When To Run It Again

**コードベースの形が変わったとき**に `$init-deep` を再実行します。新しいトップレベル領域、大きなリファクタリング、既存の `AGENTS.md` の目印を古くする再編成が起きた場合です。メモリを最新に保つことで、以降の実行が安定します。

## Options

デフォルト実行は update mode です。既存の `AGENTS.md` を読み、必要な場所を更新します。

```bash
$init-deep
$init-deep --create-new
$init-deep --max-depth=2
```

- `--create-new` は既存の `AGENTS.md` コンテキストを読み、保存したうえで新しい構造を再生成します。
- `--max-depth=<number>` は探索するディレクトリ深度を制限します。デフォルトは 3 です。

## Where It Fits

`$init-deep` はメモリのステップです。地図ができれば、他の Commands が作業を進めます。

- `$ulw-plan` はコードを変える前に何を作るかを決めます。
- `$start-work` は計画を最後のチェックボックスまで実行します。
- `$ulw-loop` は結果が検証されるまで、開いた形の作業を進め続けます。
