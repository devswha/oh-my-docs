`$start-work`는 [Prometheus 작업 계획](/ko/docs/commands/ulw-plan)을 최상위 체크박스가 모두 끝날 때까지 실행합니다. 계획 우선 흐름의 구현 절반에 해당합니다. 계획을 넘겨주면 그 계획을 완료까지 끌고 갑니다.

## 문법

```bash
$start-work [plan-name] [--worktree <absolute-path>]
```

- `plan-name` — 선택. 실행할 계획(`plans/<slug>.md`). 생략하면 가장 최근 계획을 사용합니다.
- `--worktree <absolute-path>` — 선택. 격리된 git worktree에서 작업을 실행합니다.

## 동작 방식

- `.omo/boulder.json`의 **지속 가능한 Boulder 상태**는 턴과 세션을 넘어 유지되어, 실행 사이에 진행 상태가 사라지지 않습니다.
- **Stop-hook**이 계획이 끝날 때까지 다음 턴을 다시 주입합니다 — 루프가 스스로 이어집니다.
- **독립적인 하위 작업은 병렬 서브에이전트로 펼쳐집니다.**
- **엄격한 TDD에 더해 다섯 가지 증거 게이트**: 계획 재독, 자동 검증, 수동 QA, 적대적 QA, 정리.
- 각 체크박스가 체크될 때마다 **진행 상태가 원장(ledger)에 기록됩니다.**

## 완료

모든 체크박스가 체크되면 `ORCHESTRATION COMPLETE` 블록을 출력합니다.

```text
ORCHESTRATION COMPLETE
```

## 전형적인 흐름

<Steps>

<Step>

### 계획 준비

`$ulw-plan`으로 계획을 쓰거나, 기존 `plans/<slug>.md`를 `$start-work`에 가리킵니다.

```bash
$ulw-plan "add OAuth login with refresh tokens"
```

</Step>

<Step>

### 실행

```bash
$start-work oauth-refresh-tokens
```

독립적인 작업은 병렬 서브에이전트로 펼쳐지고, 지속 가능한 Boulder 진행 상태가 중단을 견딥니다.

</Step>

<Step>

### 완료 대기

`$start-work`는 모든 체크박스가 체크될 때만 멈추고 `ORCHESTRATION COMPLETE`를 출력합니다.

</Step>

</Steps>
