LazyCodex가 제공하는 CLI는 하나, `lazycodex-ai` npm bin입니다. 이것은 얇은 포워더로, 모든 호출은 `npx`로 실행되는 `omo` 커맨드로 확장됩니다. 전역 설치는 없습니다 — 항상 `npx`로 호출합니다.

## OmO로 전달되는 방식

bin은 인자를 읽어 `oh-my-openagent`로 전달합니다. `install` 서브커맨드는 Codex 플랫폼을 고정하도록 특별 처리되고, 나머지는 그대로 통과합니다.

<Tabs items={["install", "기타 커맨드"]}>

<Tab value="install">

```bash
npx lazycodex-ai install
```

은 다음으로 확장됩니다:

```bash
npx --yes --package oh-my-openagent omo install --platform=codex
```

`install` 뒤의 인자는 모두 그대로 덧붙습니다. 예:

```bash
npx lazycodex-ai install --no-tui --codex-autonomous
# → npx --yes --package oh-my-openagent omo install --platform=codex --no-tui --codex-autonomous
```

</Tab>

<Tab value="기타 커맨드">

`install`이 아닌 첫 인자는 플랫폼 고정 없이 `omo`로 바로 전달됩니다:

```bash
npx lazycodex-ai <args...>
# → npx --yes --package oh-my-openagent omo <args...>
```

</Tab>

</Tabs>

## `--dry-run`

`--dry-run`을 **첫** 인자로 두면 커맨드를 실행하는 대신 해석된 `npx` 커맨드를 출력하고 종료합니다:

```bash
npx lazycodex-ai --dry-run install --no-tui --codex-autonomous
```

출력:

```text
npx --yes --package oh-my-openagent omo install --platform=codex --no-tui --codex-autonomous
```

`--dry-run`은 전달 전에 제거되므로, 남은 인자는 실제 실행과 정확히 동일하게 해석됩니다. 실행 전에 정확한 `omo` 호출을 확인할 때 사용하세요.

## 종료 동작

bin은 해석된 커맨드를 상속된 stdio로 실행하고, 그 커맨드의 상태 코드로 종료합니다. `npx` 자체 spawn에 실패하면 에러를 출력하고 0이 아닌 코드로 종료합니다.

## 세션 내 커맨드 기둥

설치되면 LazyCodex는 OmO의 커맨드 기둥을 Codex 안에 등록합니다. 이들은 셸이 아니라 Codex 세션에서 입력합니다. 설치 프로그램이 정확한 `$command` 문법을 보여 줍니다.

| 커맨드 | 입력 | 동작 |
| --- | --- | --- |
| [`$ulw-plan`](/ko/docs/commands/ulw-plan) | `$ulw-plan "what to build"` | Prometheus 전략 플래너. `plans/<slug>.md`에 계획을 작성합니다. 제품 코드는 절대 작성하지 않습니다. |
| [`$start-work`](/ko/docs/commands/start-work) | `$start-work [plan-name] [--worktree <path>]` | 모든 체크박스가 끝날 때까지 계획을 실행합니다. **ORCHESTRATION COMPLETE**를 출력합니다. |
| [`$ulw-loop`](/ko/docs/commands/ulw-loop) | `$ulw-loop "task" [--completion-promise=TEXT] [--strategy=reset\|continue]` | Oracle 검증 완료까지 도는 자기 참조 루프. ultrawork 모드 500회, 일반 모드 100회 상한. |
| [`/init-deep`](/ko/docs/commands/init-deep) | `/init-deep` | 계층형 `AGENTS.md` 프로젝트 메모리를 생성합니다. |

전체 설명은 [커맨드](/ko/docs/commands) 개요를 참고하세요.
