LazyCodex は configuration を最小に保ちます。直接扱うのは install flags と Codex plugin configuration です。その後の provider、model、routing、hooks、Skills、Agent Roles は下の OmO-based setup が担当します。

## Zero-config by Default

LazyCodex は妥当な defaults とともに提供され、install 直後に動作します。defaults が repository に合わないときだけ configuration を触ります。事前に作るべき別 config file はありません。

```bash
npx lazycodex-ai install
```

global install はありません。常に `npx` で呼び出します。これは `npx --yes --package oh-my-openagent omo install --platform=codex` の短縮形です。

> これが Codex light setup です: Agent Roles、Commands、hooks、Skills、model routing、verification defaults。

## Codex Target

LazyCodex は常に Codex を対象にします。`--platform=codex` argument は `lazycodex-ai` bin の `install` path に組み込まれており、harness は他 platform ではなく [OpenAI Codex CLI](https://github.com/openai/codex) に接続されます。自分で `--platform` を渡す必要はありません。

### Prerequisites

- [Node.js](https://nodejs.org): maintained LTS で十分です。`npx` は npm に含まれ、Bun は不要です。
- [OpenAI Codex CLI](https://github.com/openai/codex) または Codex app に login 済み。

## Hooks & Lifecycle

hooks は承認前には実行されません。install 後の初回実行で、Codex startup review が omo hooks の承認を求めます。upgrade 後は plugin files が変わり、以前の trust hash と一致しないため hooks が **Modified** と表示されます。再承認すると、次の session で新 version bootstrap が実行されます。

## Install Flags

<Tabs items={["interactive", "autonomous"]}>

<Tab value="interactive">

default installer は interactive (TUI) です。subscription を検出し、model selection を助け、provider authentication を案内します。

```bash
npx lazycodex-ai install
```

LLM agent に install と setup を進めさせるのが推奨です。subscription detection、model selection、provider authentication を自動で処理します。

</Tab>

<Tab value="autonomous">

完全自律・no-prompt setup では 2 つの flags を一緒に追加します。

```bash
npx lazycodex-ai install --no-tui --codex-autonomous
```

`--no-tui --codex-autonomous` は完全自律 setup のため一緒に転送されます。`lazycodex-ai` bin はこれらの flags を解釈しません。`install` 後のすべての flags は `omo install` にそのまま渡され、意味は `omo install` が与えます。

</Tab>

</Tabs>

## Provider & Model Settings

provider と model setup は LazyCodex ではなく OmO が担当します。install 中、OmO は Codex configuration と bundled `model-catalog.json` を基準に model profiles を合わせます。これが model routing layer です。

- installer が provider authentication をつなぎます。agent に実行させるのが推奨経路です。
- provider keys は environment から読まれます。すべての `*_API_KEY` と OAuth credentials は secret として扱い、log や commit に残さないでください。
- install を超える深い provider/model tuning は OmO conventions に従います。provider env vars と model interpretation rules は OmO docs を参照してください。

> provider keys を勝手に作らないでください。選んだ provider が文書化した key を environment に渡し、routing は installer に解釈させてください。

## Doctor

何かが pending または degraded に見えたら実行します。

```bash
npx lazycodex-ai doctor
```

plugin cache、hooks、MCP Servers、Agent Roles、Codex config state を health report で確認します。何が misconfigured で、なぜそうなり、どの field や component を直すべきかを説明します。
