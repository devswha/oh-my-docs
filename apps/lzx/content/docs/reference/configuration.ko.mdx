LazyCodex는 설정을 최소로 유지합니다. LazyCodex가 직접 노출하는 유일한 손잡이는 설치 플래그뿐입니다. 그 이후의 모든 것 — provider, 모델, 라우팅, 훅 — 은 그 아래의 OmO 엔진에서 설정합니다.

## 기본은 zero-config

LazyCodex는 합리적인 기본값과 함께 제공되며 설치 직후 바로 동작합니다. 기본값이 저장소에 맞지 않을 때만 설정을 만지면 됩니다. 미리 만들어야 할 별도 설정 파일은 없습니다 — 설치 후 바로 작업하세요.

```bash
npx lazycodex-ai install
```

전역 설치는 없습니다. 항상 `npx`로 호출합니다. 이는 `npx --yes --package oh-my-openagent omo install --platform=codex`의 단축형입니다.

## Codex 플랫폼 대상

LazyCodex는 항상 Codex를 대상으로 합니다. `--platform=codex` 인자는 `lazycodex-ai` bin의 `install` 경로에 내장되어 있어, 하니스는 다른 플랫폼이 아니라 [OpenAI Codex CLI](https://github.com/openai/codex)에 연결됩니다. `--platform`을 직접 전달하지 않습니다.

### 사전 요구사항

- [Bun](https://bun.sh)
- [OpenAI Codex CLI](https://github.com/openai/codex)

## 설치 플래그

<Tabs items={["대화형", "자율"]}>

<Tab value="대화형">

기본 설치 프로그램은 대화형(TUI)입니다. 구독을 감지하고, 모델 선택을 돕고, provider 인증을 안내합니다.

```bash
npx lazycodex-ai install
```

LLM 에이전트가 설치와 셋업을 진행하도록 맡기는 것을 강력히 권장합니다 — 구독 감지, 모델 선택, provider 인증을 자동으로 처리합니다.

</Tab>

<Tab value="자율">

완전 자율, 프롬프트 없는 셋업은 두 플래그를 함께 추가합니다:

```bash
npx lazycodex-ai install --no-tui --codex-autonomous
```

| 플래그 | 효과 |
| --- | --- |
| `--no-tui` | 대화형 TUI 없이 설치 프로그램을 실행합니다. |
| `--codex-autonomous` | Codex를 자율 운영용으로 구성합니다. |

`install` 뒤의 모든 플래그는 `omo install`로 그대로 전달됩니다.

</Tab>

</Tabs>

## provider·모델 설정

provider와 모델 설정은 LazyCodex가 아니라 OmO가 담당합니다. 설치 중 OmO는 어떤 모델 구독이 있는지 묻고 각 작업 카테고리를 그에 맞게 라우팅합니다 — 이것이 멀티 모델 라우팅 레이어입니다.

- 설치 프로그램이 provider 인증을 대신 연결해 줍니다. 에이전트가 실행하게 하는 것이 권장 경로입니다.
- provider 키는 환경에서 읽힙니다. 모든 `*_API_KEY`와 OAuth 자격증명은 비밀로 다루세요 — 로깅·커밋 금지.
- 설치를 넘어선 더 깊은 provider·모델 튜닝은 OmO 규약을 따릅니다. provider 환경변수와 모델 해석 규칙은 OmO 문서를 참고하세요.

> provider 키를 임의로 만들지 마세요. 선택한 provider가 문서화한 키를 환경으로 공급하고, 라우팅은 설치 프로그램이 해석하게 하세요.
