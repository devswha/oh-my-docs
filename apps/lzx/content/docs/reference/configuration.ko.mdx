LazyCodex는 설정을 최소로 유지합니다. 직접 다루는 것은 설치 플래그와 Codex 플러그인 설정입니다. 그 이후의 provider, 모델, 라우팅, hooks, Skills, Agent Roles는 OmO 기반 설정이 담당합니다.

## Zero-config by Default

LazyCodex는 합리적인 기본값과 함께 제공되며 설치 직후 바로 동작합니다. 기본값이 저장소에 맞지 않을 때만 설정을 만지면 됩니다. 미리 만들어야 할 별도 설정 파일은 없습니다 — 설치 후 바로 작업하세요.

```bash
npx lazycodex-ai install
```

전역 설치는 없습니다. 항상 `npx`로 호출합니다. 이는 `npx --yes --package oh-my-openagent omo install --platform=codex`의 단축형입니다.

> 이것이 Codex light setup입니다. Agent Roles, Commands, hooks, Skills, 모델 라우팅, 검증 기본값을 설치합니다.

## Codex Target

LazyCodex는 항상 Codex를 대상으로 합니다. `--platform=codex` 인자는 `lazycodex-ai` bin의 `install` 경로에 내장되어 있어, 하니스는 다른 플랫폼이 아니라 [OpenAI Codex CLI](https://github.com/openai/codex)에 연결됩니다. `--platform`을 직접 전달하지 않습니다.

### Prerequisites

- [Node.js](https://nodejs.org) — 유지보수 중인 LTS 버전이면 됩니다. `npx`가 함께 제공되며 Bun은 필요하지 않습니다.
- [OpenAI Codex CLI](https://github.com/openai/codex) 또는 Codex 앱, 로그인 완료 상태.

## Hooks & Lifecycle

훅은 승인 전에는 실행되지 않습니다. 설치 후 첫 실행에서 Codex 시작 리뷰가 omo 훅 승인을 요청합니다. 업그레이드 후에는 플러그인 파일이 바뀌어 이전 trust hash와 달라지므로 훅이 **Modified**로 표시됩니다. 다시 승인하면 다음 세션에서 새 버전 bootstrap이 실행됩니다.

## Install Flags

<Tabs items={["interactive", "autonomous"]}>

<Tab value="interactive">

기본 설치 프로그램은 대화형(TUI)입니다. 구독을 감지하고, 모델 선택을 돕고, provider 인증을 안내합니다.

```bash
npx lazycodex-ai install
```

LLM 에이전트가 설치와 셋업을 진행하도록 맡기는 것을 강력히 권장합니다 — 구독 감지, 모델 선택, provider 인증을 자동으로 처리합니다.

</Tab>

<Tab value="autonomous">

완전 자율, 프롬프트 없는 셋업은 두 플래그를 함께 추가합니다:

```bash
npx lazycodex-ai install --no-tui --codex-autonomous
```

`--no-tui --codex-autonomous`는 완전 자율, 프롬프트 없는 셋업을 위해 함께 전달됩니다. `lazycodex-ai` bin은 이 플래그들을 해석하지 않습니다 — `install` 뒤의 모든 플래그는 `omo install`로 그대로 전달되며, 의미는 `omo install`이 부여합니다.

</Tab>

</Tabs>

## Provider & Model Settings

provider와 모델 설정은 LazyCodex가 아니라 OmO가 담당합니다. 설치 중 OmO는 Codex 설정과 bundled `model-catalog.json`을 기준으로 모델 프로필을 맞춥니다 — 이것이 모델 라우팅 레이어입니다.

- 설치 프로그램이 provider 인증을 대신 연결해 줍니다. 에이전트가 실행하게 하는 것이 권장 경로입니다.
- provider 키는 환경에서 읽힙니다. 모든 `*_API_KEY`와 OAuth 자격증명은 비밀로 다루세요 — 로깅·커밋 금지.
- 설치를 넘어선 더 깊은 provider·모델 튜닝은 OmO 규약을 따릅니다. provider 환경변수와 모델 해석 규칙은 OmO 문서를 참고하세요.

> provider 키를 임의로 만들지 마세요. 선택한 provider가 문서화한 키를 환경으로 공급하고, 라우팅은 설치 프로그램이 해석하게 하세요.

## Doctor

무언가 pending 또는 degraded로 보이면 다음을 실행합니다.

```bash
npx lazycodex-ai doctor
```

플러그인 캐시, hooks, MCP Servers, Agent Roles, Codex config state를 health report로 확인합니다. 무엇이 잘못 설정되었고 왜 그런지, 어떤 필드나 컴포넌트를 고쳐야 하는지 설명합니다.
