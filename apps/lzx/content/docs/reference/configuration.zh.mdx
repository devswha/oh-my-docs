LazyCodex 尽量保持配置最少。你直接处理的部分是安装 flags 和 Codex plugin configuration。之后的 provider、model、routing、hooks、Skills、Agent Roles 都由底层 OmO-based setup 负责。

## Zero-config by Default

LazyCodex 带有合理默认值，安装后即可工作。只有当默认值不适合仓库时才需要修改配置。无需预先准备额外配置文件。

```bash
npx lazycodex-ai install
```

没有全局安装。始终通过 `npx` 调用。这是 `npx --yes --package oh-my-openagent omo install --platform=codex` 的简写。

> 这就是 Codex light setup：Agent Roles、Commands、hooks、Skills、model routing、verification defaults。

## Codex Target

LazyCodex 始终面向 Codex。`--platform=codex` 参数内置在 `lazycodex-ai` bin 的 `install` 路径里，因此 harness 连接的是 [OpenAI Codex CLI](https://github.com/openai/codex)，不是其他平台。你不需要自己传 `--platform`。

### Prerequisites

- [Node.js](https://nodejs.org)：维护中的 LTS 即可。`npx` 随 npm 提供，不需要 Bun。
- [OpenAI Codex CLI](https://github.com/openai/codex) 或 Codex app，并已登录。

## Hooks & Lifecycle

hooks 在批准前不会运行。安装后第一次运行时，Codex startup review 会请求批准 omo hooks。升级后，plugin files 改变并与旧 trust hash 不匹配，所以 hooks 会显示为 **Modified**。再次批准后，下一个 session 会为新版本运行 bootstrap。

## Install Flags

<Tabs items={["interactive", "autonomous"]}>

<Tab value="interactive">

默认安装器是交互式（TUI）。它会检测订阅、帮助选择模型，并引导 provider authentication。

```bash
npx lazycodex-ai install
```

推荐让 LLM agent 运行安装和设置：它会替你处理 subscription detection、model selection、provider authentication。

</Tab>

<Tab value="autonomous">

完全自动、无提示 setup 需要同时传两个 flags：

```bash
npx lazycodex-ai install --no-tui --codex-autonomous
```

`--no-tui --codex-autonomous` 会一起转发，用于完全自动 setup。`lazycodex-ai` bin 不解释这些 flags；`install` 后面的所有 flags 都原样传给 `omo install`，由它赋予含义。

</Tab>

</Tabs>

## Provider & Model Settings

provider 和 model setup 由 OmO 负责，不由 LazyCodex 负责。安装时，OmO 会根据 Codex configuration 和 bundled `model-catalog.json` 对齐模型 profiles；这就是 model routing layer。

- 安装器会连接 provider authentication。推荐让 agent 运行安装流程。
- provider keys 从环境读取。所有 `*_API_KEY` 和 OAuth 凭据都要当作 secret；不要记录日志，也不要提交。
- 安装以外更深的 provider/model tuning 遵循 OmO 约定。provider 环境变量和 model 解释规则请参考 OmO 文档。

> 不要编造 provider keys。提供所选 provider 文档化的 key，并让安装器解释 routing。

## Doctor

如果有什么 pending 或 degraded，运行：

```bash
npx lazycodex-ai doctor
```

它会在 health report 中检查 plugin cache、hooks、MCP Servers、Agent Roles、Codex config state。报告会说明哪里配置错、为什么，以及应该修哪个 field 或 component。
