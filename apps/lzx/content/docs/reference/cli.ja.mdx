LazyCodex が提供する CLI は 1 つ、`lazycodex-ai` npm bin です。OmO-based Codex light setup を install し、診断します。global install はありません。常に `npx` で呼び出します。

## Forwarding to OmO

bin は arguments を読み、`oh-my-openagent` へ転送します。`install` subcommand は Codex platform を固定するために特別扱いされ、それ以外はそのまま通過します。

<Tabs items={["install", "other"]}>

<Tab value="install">

```bash
npx lazycodex-ai install
```

これは次へ展開されます:

```bash
npx --yes --package oh-my-openagent omo install --platform=codex
```

`install` 後の arguments はすべてそのまま追加されます。例:

```bash
npx lazycodex-ai install --no-tui --codex-autonomous
# -> npx --yes --package oh-my-openagent omo install --platform=codex --no-tui --codex-autonomous
```

</Tab>

<Tab value="other">

`install` 以外の最初の argument は、platform 固定なしで `omo` に直接転送されます。

```bash
npx lazycodex-ai <args...>
# -> npx --yes --package oh-my-openagent omo <args...>
```

</Tab>

</Tabs>

## `--dry-run`

`--dry-run` を**最初の** argument に置くと、実行せず解決後の `npx` command を出力します。

```bash
npx lazycodex-ai --dry-run install --no-tui --codex-autonomous
```

出力:

```text
npx --yes --package oh-my-openagent omo install --platform=codex --no-tui --codex-autonomous
```

`--dry-run` は転送前に取り除かれるため、残りの arguments は実行時と同じように解釈されます。実行前に正確な `omo` invocation を確認するときに使います。

## `doctor`

```bash
npx lazycodex-ai doctor
```

何が設定済みで、何が不足し、なぜそうなっているかを示す health report を出力します。plugin cache、hooks、MCP Servers、Agent Roles、Codex config state を確認します。hooks が pending、skills が load されない、routing が変に見える場合はまず実行してください。

## Marketplace Alternative

`npx` install が主経路です。追加の実験的経路として、Codex から plugin を install できます。

```bash
codex plugin marketplace add https://github.com/code-yeongyu/lazycodex
codex plugin add omo@sisyphuslabs
```

次の Codex startup review で omo hooks を承認してください。最初に承認された session は background bootstrap を実行し、完了後の restart を案内します。upgrade は次で実行します。

```bash
codex plugin marketplace upgrade sisyphuslabs
```

upgrade 後は plugin files が変わるため trust hash の再承認が必要で、hooks は **Modified** と表示されます。再承認すると、次の session で新 version の bootstrap が実行されます。

## Exit Behavior

bin は解決された command を inherited stdio で spawn し、その command の status code で終了します。`npx` 自体の spawn に失敗した場合は error を出力し、non-zero で終了します。

## In-session Commands

install 後、LazyCodex は OmO commands を Codex 内に登録します。これは shell commands ではなく、Codex composer で使う `$command` です。syntax と usage flow は [Commands](/ja/docs/commands) にまとめています。
