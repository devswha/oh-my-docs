This section is the LazyCodex (LZX) runtime reference. LazyCodex is a thin distribution that packages [OmO (oh-my-openagent)](https://github.com/code-yeongyu/oh-my-openagent) for Codex. Deep workflows run on top of the OmO plugin's Skills, hooks, MCP Servers, and model profiles.

## What LazyCodex Owns

LazyCodex itself is small. It provides one thing: the `lazycodex-ai` npm bin, invoked through `npx`, that installs and drives OmO for the Codex platform.

```bash
npx lazycodex-ai install
```

This is shorthand for `npx --yes --package oh-my-openagent omo install --platform=codex`. There is no global install; always invoke it with `npx`. The Codex light setup comes with zero-config defaults that can be overridden later.

## What OmO Owns

After installation, the OmO-based setup owns the rest: Agent Roles, parallel exploration, model routing, Skills, hooks, and verified completion. If you tune provider, model, or behavior beyond install flags, you are effectively configuring OmO.
