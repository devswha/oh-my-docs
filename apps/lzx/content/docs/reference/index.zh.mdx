本节是 LazyCodex（LZX）的运行时参考。LazyCodex 是一个轻量分发层，将 [OmO（oh-my-openagent）](https://github.com/code-yeongyu/oh-my-openagent) 打包为 Codex 的代理框架。通过 LazyCodex 运行的一切都会转发给 OmO，因此深度配置沿用 OmO 的约定。

## LazyCodex 负责什么

LazyCodex 本身很小，只提供一件事：用 `npx` 调用、面向 Codex 平台安装并驱动 OmO 的 `lazycodex-ai` npm 可执行文件。

```bash
npx lazycodex-ai install
```

这是 `npx --yes --package oh-my-openagent omo install --platform=codex` 的简写。没有全局安装 —— 始终通过 `npx` 调用。框架自带 zero-config 默认值，之后可覆盖。

## OmO 负责什么

安装后，底层的 OmO 引擎负责其余部分：规律代理（Sisyphus、Hephaestus、Oracle、Librarian）、并行执行、多模型路由、技能、钩子与已验证的完成。若要在安装标志之外调整 provider、模型或行为，本质上就是在配置 OmO。
