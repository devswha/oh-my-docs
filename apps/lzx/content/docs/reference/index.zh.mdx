本节是 LazyCodex (LZX) 运行时 Reference。LazyCodex 是一个为 Codex 打包 [OmO (oh-my-openagent)](https://github.com/code-yeongyu/oh-my-openagent) 的薄发行层。通过 LazyCodex 运行的深层工作流，建立在 OmO 插件的 Skills、hooks、MCP Servers 和 model profiles 之上。

## What LazyCodex Owns

LazyCodex 本身很小。它提供的东西只有一个：通过 `npx` 调用的 `lazycodex-ai` npm bin，用来为 Codex platform 安装并驱动 OmO。

```bash
npx lazycodex-ai install
```

这是 `npx --yes --package oh-my-openagent omo install --platform=codex` 的简写。没有全局安装；始终用 `npx` 调用。Codex light setup 带有之后可以覆盖的 zero-config 默认值。

## What OmO Owns

安装完成后，下面的 OmO-based setup 负责其余部分：Agent Roles、并行探索、model routing、Skills、hooks、verified completion。只要你越过安装 flags 去调整 provider、model 或行为，本质上就是在配置 OmO。
