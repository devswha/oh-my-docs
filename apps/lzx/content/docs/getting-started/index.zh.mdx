---
title: 快速开始
description: 在 Codex 中安装 LazyCodex，并从项目记忆继续到第一个计划和执行。
---

本文介绍从首次安装 LazyCodex 到运行第一个任务的基本流程。先从 Codex 开始安装，用 `$init-deep` 创建项目记忆，然后执行 `$ulw-plan` 和 `$start-work`。

<Steps>

<Step>

### 前置准备

- 可以运行 `npx` 的 Node.js/npm 环境。无需 Bun。
- Codex App 或 [OpenAI Codex CLI](https://github.com/openai/codex)

LazyCodex 是把 OmO 命令、技能和钩子接入 Codex 设置的工具。只要 Codex 已经能正常使用，App 和 CLI 的安装流程可以视为相同。

</Step>

<Step>

### 安装

安装方式可以看作一条流程。打开一个新的 Codex 任务，把 LazyCodex GitHub 链接交给它，然后让它安装。

```text
https://github.com/code-yeongyu/lazycodex

请参考这个仓库安装 LazyCodex。
```

如果需要自己执行命令，请使用：

```bash
npx lazycodex-ai install
```

安装后重新打开 Codex，确认 `$` 菜单中出现 OmO 插件和技能。

下次启动时，需要在 Codex 启动审核中批准 `omo` 钩子。钩子在批准前不会运行。如果状态显示为 `pending` 或 `degraded`，建议先运行诊断。

```bash
npx lazycodex-ai doctor
```

</Step>

<Step>

### 认证

LazyCodex 本身没有单独的登录命令。安装程序，或你委托安装的智能体，会处理订阅检测、模型选择和提供方认证。Codex App 或 Codex CLI 本身需要已登录，但这更接近前置准备，而不是 LazyCodex 专有的认证步骤。

提供方与路由细节见[配置](/zh/docs/reference/configuration)。

</Step>

<Step>

### 运行第一个任务

为了让智能体带着项目上下文开始，先创建分层的 `AGENTS.md` 项目记忆。

```text
$init-deep
```

<figure className="not-prose my-6">
  <img
    src="/images/docs/lazycodex-command-init-deep.gif"
    alt="init-deep 命令运行并创建 AGENTS.md 项目记忆"
    width="960"
    height="540"
    loading="lazy"
    className="w-full rounded-lg border border-fd-border bg-fd-card"
  />
</figure>

然后选择适合当前工作的命令。

如果需要先做计划，可以这样输入：

```text
$ulw-plan "给一个小型任务应用添加完成处理 helper"
```

这个阶段不会编写产品代码。它会读取仓库，留下计划草案和批准请求；批准后，会在 `.omo/plans/<slug>.md` 中生成可执行的计划文件。

<figure className="not-prose my-6">
  <img
    src="/images/docs/lazycodex-command-ulw-plan.gif"
    alt="ulw-plan 命令生成计划草案并请求批准"
    width="960"
    height="540"
    loading="lazy"
    className="w-full rounded-lg border border-fd-border bg-fd-card"
  />
</figure>

如果要执行已有计划，可以这样输入：

```text
$start-work
```

它会按照计划推进，直到所有复选框完成，然后输出 **ORCHESTRATION COMPLETE**。

<figure className="not-prose my-6">
  <img
    src="/images/docs/lazycodex-command-start-work.gif"

    width="960"
    height="540"
    loading="lazy"
    className="w-full rounded-lg border border-fd-border bg-fd-card"
  />
</figure>

如果想把实现到验证的全过程都交出去，可以使用 loop。它会继续工作，直到结果能用证据确认。

```text
$ulw-loop "修复结账流程失败，并端到端验证"
```
