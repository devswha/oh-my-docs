title: 시작하기
description: 한 줄의 npx 명령으로 LazyCodex를 Codex에 설치하고, 인증한 뒤, init-deep, ulw-plan, start-work, ulw-loop로 첫 작업을 실행합니다.
---

LazyCodex는 한 줄의 `npx` 명령으로 OmO 에이전트 하네스를 Codex에 설치합니다. 전역 설치는 필요 없습니다. 먼저 세팅을 마친 다음, 첫 작업을 진행해 봅니다.

<Steps>

<Step>

### 사전 준비

- [Bun](https://bun.sh)
- [OpenAI Codex CLI](https://github.com/openai/codex)

LazyCodex는 OpenAI Codex 안에서 실행되므로, Codex CLI가 설치되어 정상 동작해야 합니다.

</Step>

<Step>

### 설치

한 줄이면 됩니다. 전역 설치나 `npm i -g`는 필요 없으며, 항상 `npx`를 사용합니다.

```bash
npx lazycodex-ai install
```

이는 Codex 플랫폼을 대상으로 하는 OmO 설치 프로그램의 단축형입니다.

```bash
npx --yes --package oh-my-openagent omo install --platform=codex
```

완전 자율, 무 TUI 세팅을 원한다면 자율 모드 플래그를 추가합니다.

```bash
npx lazycodex-ai install --no-tui --codex-autonomous
```

설치 프로그램은 OmO 하네스(커맨드, 스킬, 훅, 모델 라우팅)를 여러분의 Codex 세팅에 연결합니다. LLM 에이전트가 설치를 실행하고 세팅을 대신 진행하도록 맡기는 것을 강력히 권장합니다. 에이전트가 구독 감지, 모델 선택, 제공자 인증을 자동으로 처리합니다.

</Step>

<Step>

### 인증

별도의 LazyCodex 로그인 명령은 없습니다. 설치 프로그램(그리고 이를 대신 실행하도록 맡긴 LLM 에이전트)이 구독 감지, 모델 선택, 제공자 인증을 자동으로 처리합니다. Codex CLI 자체에 로그인되어 있어야 하지만, 이는 사전 준비일 뿐 LazyCodex의 인증 단계는 아닙니다.

제공자와 라우팅 세부 사항은 [설정](/ko/docs/reference/configuration)을 참고하세요.

</Step>

<Step>

### 첫 작업 실행하기

먼저 프로젝트 메모리를 만든 다음, 커맨드 기둥을 사용합니다. 기둥 커맨드는 설치 프로그램이 출력하는 `$command` 구문으로 호출합니다.

먼저 에이전트가 충분한 맥락을 갖고 시작하도록 계층형 `AGENTS.md` 프로젝트 메모리를 생성합니다.

```text
/init-deep
```

그다음 작업에 맞는 기둥을 선택합니다.

- `$ulw-plan "무엇을 만들지"` — Prometheus가 접근 방식을 계획해 `plans/<slug>.md`에 작성합니다. 제품 코드는 절대 작성하지 않습니다.
- `$start-work [plan-name] [--worktree <path>]` — 모든 체크박스가 완료될 때까지 계획을 실행한 뒤 **ORCHESTRATION COMPLETE**를 출력합니다.
- `$ulw-loop "작업" [--completion-promise=TEXT] [--strategy=reset|continue]` — Oracle이 검증한 완료에 이를 때까지 실행되는 자기 참조 루프입니다.

일반적인 흐름은 먼저 계획하고 실행합니다.

```text
/init-deep
$ulw-plan "작업 관리를 위한 REST API를 만든다"
$start-work
```

또는 결과가 증거로 검증될 때까지 계속 진행해야 하는 열린 작업이라면 루프를 바로 사용합니다.

```text
$ulw-loop "결제 흐름의 실패를 수정하고 처음부터 끝까지 검증한다"
```

</Step>

</Steps>

## 스킬

커맨드 기둥 외에도 LazyCodex는 전문 작업을 위한 OmO 스킬 레이어(`review-work`, `remove-ai-slops`, `frontend-ui-ux`, `programming`, `LSP`, `rules`, `comment-checker` 등)를 설치합니다. 커맨드 레이어는 단순하게 유지되고, 스킬이 전문가의 판단을 더합니다. 전체 라이브러리는 [스킬](/ko/docs/skills)을 참고하세요.
