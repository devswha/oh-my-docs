---
title: 시작하기
description: Codex에서 LazyCodex를 설치하고, 프로젝트 메모리부터 첫 계획과 실행까지 이어 가는 과정.
---

LazyCodex를 처음 설치하고 첫 작업까지 이어 가는 기본 흐름을 다룹니다. Codex에서 설치를 시작하고, `$init-deep`로 프로젝트 메모리를 만든 뒤 `$ulw-plan`과 `$start-work`를 실행하는 구조입니다.

<Steps>

<Step>

### 사전 준비

- `npx`를 실행할 수 있는 Node.js/npm 환경. Bun은 필요하지 않습니다.
- Codex App 또는 [OpenAI Codex CLI](https://github.com/openai/codex)

LazyCodex는 Codex 설정에 OmO 커맨드, 스킬, 훅을 연결하는 도구입니다. Codex가 정상 동작하는 상태라면 App과 CLI의 설치 흐름은 같다고 보면 됩니다.

</Step>

<Step>

### 설치

설치 방식은 하나로 보면 됩니다. Codex에 새 작업을 열고 LazyCodex GitHub 링크를 준 다음, 설치해 달라고 말하면 됩니다.

```text
https://github.com/code-yeongyu/lazycodex

이 저장소를 참고해서 LazyCodex를 설치해줘.
```

직접 명령을 실행해야 하는 환경이라면 아래 명령을 사용합니다.

```bash
npx lazycodex-ai install
```

설치 후에는 Codex를 다시 열고 `$` 메뉴에서 OmO 플러그인과 스킬이 뜨는지 확인합니다.

다음 실행 때 Codex 시작 리뷰에서 `omo` 훅을 승인해야 합니다. 훅은 승인 전에는 실행되지 않습니다. 상태가 `pending` 또는 `degraded`로 보이면 먼저 진단을 실행하는 편이 좋습니다.

```bash
npx lazycodex-ai doctor
```

</Step>

<Step>

### 인증

LazyCodex 자체에는 별도의 로그인 명령이 없습니다. 설치 프로그램, 또는 설치를 맡긴 에이전트가 구독 감지, 모델 선택, 제공자 인증을 처리합니다. Codex App 또는 Codex CLI 자체에는 로그인되어 있어야 하지만, 이는 LazyCodex 고유의 인증 단계라기보다는 사전 준비에 가깝습니다.

제공자와 라우팅 세부 사항은 [설정](/ko/docs/reference/configuration)을 참고하면 됩니다.

</Step>

<Step>

### 첫 작업 실행하기

에이전트가 프로젝트 맥락을 알고 시작하도록, 먼저 계층형 `AGENTS.md` 프로젝트 메모리를 만듭니다.

```text
$init-deep
```

그다음 작업에 맞는 커맨드를 고릅니다.

계획부터 세워야 한다면 이렇게 입력합니다. 이 단계에서는 제품 코드를 작성하지 않고, 리포지토리를 읽은 뒤 초안과 승인 요청을 남깁니다. 승인하면 실행 가능한 계획 파일이 `.omo/plans/<slug>.md`에 만들어집니다.

```text
$ulw-plan "작은 작업 앱에 완료 처리 helper를 추가한다"
```

이미 만든 계획을 실행하려면 이렇게 입력합니다. 모든 체크박스가 완료될 때까지 계획을 따라간 뒤 **ORCHESTRATION COMPLETE**를 출력합니다.

```text
$start-work
```

