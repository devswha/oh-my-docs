このページでは、LazyCodex を初めてインストールして最初の作業まで進める基本の流れを扱います。Codex からインストールを始め、`$init-deep` でプロジェクトメモリを作り、その後 `$ulw-plan` と `$start-work` を実行します。

<Steps>

<Step>

### 事前準備

- `npx` を実行できる Node.js/npm 環境。Bun は不要です。
- Codex App または [OpenAI Codex CLI](https://github.com/openai/codex)

LazyCodex は Codex 設定に OmO コマンド、スキル、フックを接続するツールです。Codex が正常に動いている状態なら、App と CLI のインストールの流れは同じものとして見ればよいです。

</Step>

<Step>

### インストール

インストール方法は 1 つの流れとして見れば十分です。Codex で新しい作業を開き、LazyCodex の GitHub リンクを渡して、インストールを依頼します。

```text
https://github.com/code-yeongyu/lazycodex

このリポジトリを参考に LazyCodex をインストールしてください。
```

自分でコマンドを実行する必要がある場合は、次を使います。

```bash
npx lazycodex-ai install
```

インストール後は Codex を開き直し、`$` メニューに OmO プラグインとスキルが表示されるか確認します。

次回起動時、Codex のスタートアップレビューで `omo` フックを承認してください。フックは承認前には実行されません。状態が `pending` または `degraded` に見える場合は、先に診断を実行するのがよいです。

```bash
npx lazycodex-ai doctor
```

</Step>

<Step>

### 認証

LazyCodex 自体には専用のログインコマンドはありません。インストーラー、またはインストールを任せたエージェントが、サブスクリプション検出、モデル選択、プロバイダー認証を処理します。Codex App または Codex CLI 自体にはログイン済みである必要がありますが、これは LazyCodex 固有の認証ステップというより事前準備に近いものです。

プロバイダーとルーティングの詳細は[設定](/ja/docs/reference/configuration)を参照してください。

</Step>

<Step>

### 最初の作業を実行する

エージェントがプロジェクトの文脈を知った状態で始められるよう、まず階層型の `AGENTS.md` プロジェクトメモリを作ります。

```text
$init-deep
```

  <img

次に、作業に合うコマンドを選びます。

先に計画を立てる必要がある場合は、次のように入力します。

```text
$ulw-plan "小さなタスクアプリに完了処理 helper を追加する"
```

この段階ではプロダクトコードを書きません。リポジトリを読んだうえで計画の草案と承認依頼を残し、承認されると `.omo/plans/<slug>.md` に実行可能な計画ファイルが作成されます。

</figure>

既存の計画を実行する場合は、次のように入力します。

```text
$start-work
```

すべてのチェックボックスが完了するまで計画に沿って進み、最後に **ORCHESTRATION COMPLETE** を出力します。

