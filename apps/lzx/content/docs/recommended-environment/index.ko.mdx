LazyCodex는 별도의 복잡한 실행 환경을 요구하지 않습니다. Codex 안에 OmO 명령, 스킬, 훅을 연결하는 쪽에 가깝습니다. 먼저 Codex가 제대로 열리고, `npx`를 실행할 수 있고, 작업할 저장소가 준비되어 있으면 됩니다.

## 운영체제

가장 무난한 환경은 Ubuntu 또는 macOS입니다. LazyCodex는 셸, Git, Node.js/npm, Codex 설정 파일, 훅을 자주 다루기 때문에 Unix 계열 환경에서 훨씬 덜 삐걱거립니다.

| 운영체제 | 권장 정도 | 메모 |
| --- | --- | --- |
| <span className="not-prose inline-flex items-center gap-2 whitespace-nowrap align-middle"><img src="/images/docs/os-ubuntu.svg" alt="" width="20" height="20" className="h-5 w-5 shrink-0" /> Ubuntu</span> | 가장 권장 | 서버와 개발 환경 모두에서 경로, 셸, 패키지 관리가 예측 가능합니다. |
| <span className="not-prose inline-flex items-center gap-2 whitespace-nowrap align-middle"><img src="/images/docs/os-macos.svg" alt="" width="20" height="20" className="h-5 w-5 shrink-0" /> macOS</span> | 권장 | 로컬 개발용으로 쓰기 좋습니다. Homebrew와 Node.js/npm만 정리되어 있으면 충분합니다. |
| <span className="not-prose inline-flex items-center gap-2 whitespace-nowrap align-middle"><img src="/images/docs/os-windows.svg" alt="" width="20" height="20" className="h-5 w-5 shrink-0" /> Windows</span> | 적극 비추천 | 네이티브 Windows 셸과 경로 차이 때문에 훅, CLI, 파일 권한, 스크립트 실행에서 불필요한 마찰이 생기기 쉽습니다. |

Windows를 꼭 써야 한다면 네이티브 Windows에서 바로 쓰기보다 WSL2의 Ubuntu 안에서 Codex와 LazyCodex를 쓰는 편이 낫습니다. 프로젝트도 가능하면 WSL2 파일시스템 안에 두는 쪽이 덜 흔들립니다.

## 설치 전에 확인할 것

| 항목 | 권장 상태 |
| --- | --- |
| Codex | Codex App 또는 Codex CLI가 설치되어 있고 로그인되어 있습니다. |
| Node.js/npm | 유지보수 중인 Node.js LTS를 사용합니다. `npx`는 npm에 포함되어 있습니다. |
| 프로젝트 | LazyCodex를 쓸 저장소를 Codex에서 열어 둡니다. |
| Git | 변경 사항을 확인하고 되돌릴 수 있도록 Git 저장소에서 쓰는 편이 좋습니다. |
| 비밀값 | 제공자 키가 필요하다면 셸이나 Codex 환경에 둡니다. 프로젝트 파일에 붙여 넣지 않습니다. |

LazyCodex를 소스에서 빌드하는 경우가 아니라면 Bun은 설치하지 않아도 됩니다. 일반 설치와 사용은 Node.js/npm의 `npx` 경로를 기준으로 합니다.

`npx`는 npm에 포함되어 있습니다. 먼저 버전을 확인합니다.

```bash
node -v
npm -v
npx -v
```

없다면 Node.js/npm을 설치합니다.

```bash
# Ubuntu 또는 WSL2 Ubuntu
sudo apt update
sudo apt install -y nodejs npm

# macOS
brew install node
```

<Callout title="Node.js/npm이 뭐예요?">
  Node.js는 컴퓨터에서 JavaScript 도구를 실행하게 해 주는 환경입니다. npm은 Node.js와 함께 설치되는 패키지 관리 도구이고, `npx`는 npm에 포함된 실행 도구입니다. LazyCodex 설치에서는 이 `npx`로 설치 명령을 한 번 실행합니다.
</Callout>

## 저자의 주관적인 추천

- **Codex App으로 시작하기**: 설치, 로그인, 세션 확인, 스킬 호출 흐름을 눈으로 보기 쉬워서 처음 쓰는 사람에게 가장 무난합니다.
- **ChatGPT Pro 이상 사용하기**: 문서의 예시는 ChatGPT Pro 이상에서 제공되는 Codex 사용 환경을 기준으로 이해하면 됩니다.
- **처음이면 네이티브 Codex부터 경험하기**: 바이브코딩이 처음이라면 바로 LazyCodex를 얹기보다, 먼저 Codex가 기본적으로 어떻게 응답하고 파일을 고치는지 경험해 보는 편이 좋습니다.
- **다른 모델 조합은 피하기**: GLM, Kimi, Mimo 같은 조합은 권장하지 않습니다. LazyCodex 문서는 Codex와 OmO 스킬 흐름을 기준으로 설명하므로, 다른 모델까지 같은 품질로 맞춰 안내하지 않습니다.
