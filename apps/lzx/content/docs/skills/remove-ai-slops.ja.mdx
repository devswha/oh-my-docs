title: remove-ai-slops
description: 挙動を保ったまま、AI が作ったコード臭を整理します。
---

`remove-ai-slops` は、ブランチ差分または明示されたファイル一覧から、AI が作ったコード臭を挙動を厳密に保ったまま取り除きます。中核となる安全不変式は、1 行を消す前に通過する回帰テストで現在の挙動を固定することです。チェックリストは安全網ではありません。通るテストが安全網です。

トリガーは "remove ai slops"、"clean ai code"、"deslop"、"cleanup AI generated" などです。

`remove-ai-slops` はオーケストレーターです。範囲を決め、まず回帰テストで挙動を固定し、カテゴリ別の整理を並列 worker に分け、品質ゲートで検証します。Codex では OpenCode 風の `deep` 例をそのまま呼びません。自己完結したメッセージを持つ worker sub-agent に翻訳して実行します。

## 範囲

- **デフォルト**: `merge-base main` に対するブランチ差分。引数なしで動きます。
- **任意**: 呼び出し元が渡した明示的なファイル一覧。たとえば loop ワークフローの変更ファイル集合。

削除ファイル、バイナリ、生成物、vendor ファイルは除外されます。

## Slop カテゴリ

この Skill は主要な slop カテゴリをグループごとに評価します。

| グループ | カテゴリ |
| --- | --- |
| スタイル | 明らかなコメント、過剰な防御コード、過剰な複雑さ。 |
| 構造 | 不要な抽象化、境界違反、死んだコード。 |
| 隠れたコスト | 重複、挙動等価の性能整理。 |
| 挙動カバレッジ | 不足しているテスト。 |
| 構造、サイズ | 純粋な 250 LOC 以上の巨大モジュール。 |

各カテゴリには明示的な KEEP ルールがあります。理由を説明するコメント、システム境界での検証、実際の境界を作る抽象化、意図されたパターンは残します。性能変更は挙動等価が明らかな場合だけ適用し、そうでなければ SKIP します。巨大モジュールは単なる警告ではなく、本格的なモジュール分割を引き起こします。新しいファイルは担当する概念で命名し、`utils`、`helpers`、`common`、`part_1` は使いません。

## 手順

<Steps>

<Step>

### 範囲を決める

ファイルパスが渡されていれば、それが範囲です。そうでなければブランチ差分を使います。

```bash
git diff $(git merge-base main HEAD)..HEAD --name-only
```

</Step>

<Step>

### 回帰テストで挙動を固定する

範囲内の各ファイルについて、観測可能な挙動を特定し、既存カバレッジを探し、編集前に現在の挙動を固定する最小の回帰テストを書きます。ベースラインは green でなければなりません。green ベースラインを作れない場合は停止して報告します。

</Step>

<Step>

### 計画してから並列整理する

安全な順から危険な順に、コメント、死んだコード、防御コード、重複、複雑度、抽象化/境界、性能、テスト、巨大モジュールの整理計画を書きます。その後、ファイル単位 worker を一度に最大五つずつ並列実行します。ファイルが失敗したら一度だけ再試行し、それでも失敗すればエスカレーションします。

</Step>

<Step>

### 品質ゲートで検証する

回帰テスト、lint、LSP 診断とプロジェクト型チェッカーによる typecheck、単体/統合テスト、静的/セキュリティスキャンを実行します。本当に該当しない gate は `N/A` として報告し、黙って飛ばしません。その後、安全、挙動、品質のチェックリストを確認します。

</Step>

<Step>

### 問題を修正する

gate が失敗したら、問題のある変更をターゲット編集または checkout で戻し、安全を証明できる部分だけを再適用して gate を再実行します。同じファイルで三回失敗したら停止してエスカレーションします。

</Step>

</Steps>

## アンチパターン

- 回帰テスト手順を飛ばすこと。カバーされていない場所でコードを消すのは、遅れて起きる挙動変更です。
- 無関係なリファクタリングを整理コミットに混ぜること。
- アルゴリズム変更を性能改善に見せかけること。
- 黙ってスキップすること。`N/A` と理由を書く。
- 理由を説明するコメントを消すこと。
- 合意した範囲外のファイルを触ること。
