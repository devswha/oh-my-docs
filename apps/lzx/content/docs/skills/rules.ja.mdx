`rules` Skill は、LazyCodex (LZX) をプロジェクト固有の指示に固定します。Codex で OmO (oh-my-openagent) プラグインが有効になると、プロジェクト rules が自動的に注入されます。毎回のセッションで貼り直す必要はありません。Codex は対応している rules と instruction ファイルをコンテキストに置いたまま作業します。

## 一行まとめ

Codex Rules は自動です。各セッション開始時と各 prompt で静的プロジェクト指示を注入し、編集後にはファイル別の一致 rules を注入します。実行するコマンドはありません。プラグインを有効にすると、リポジトリの規約がどこでも Codex についてきます。

## 注入対象

- `SessionStart` と `UserPromptSubmit` での**静的プロジェクト指示**。
- デフォルトでは、Codex `apply_patch` 後の**一致するファイル別 rules**。

動的な `PostToolUse` 出力は追加コンテキストとして注入され、プラグインデータはセッション単位で重複排除されます。Codex Rules はツール出力を書き換えません。その横に指示を追加するだけです。

## 対応するプロジェクトソース

| ソース | 備考 |
|---|---|
| `CONTEXT.md` | 一般的なプロジェクトコンテキスト。 |
| `.omo/rules/**/*.md` | OmO/LazyCodex rules ツリー。 |
| `.claude/rules/**/*.md` | Claude rules ディレクトリ。 |
| `.cursor/rules/**/*.md` | Cursor rules。 |
| `.github/instructions/**/*.md` | GitHub instruction ファイル。 |
| `.github/copilot-instructions.md` | Copilot 指示。 |

## 設定

環境変数で挙動を調整します。

| 変数 | 効果 |
|---|---|
| `CODEX_RULES_DISABLED=1` | Skill 全体を無効化。 |
| `CODEX_RULES_MODE=both\|static\|dynamic\|off` | static、dynamic、both、off を選択。 |
| `CODEX_RULES_MAX_RULE_CHARS=<number>` | 注入される各 rule のサイズ制限。 |
| `CODEX_RULES_MAX_RESULT_CHARS=<number>` | 注入される結果コンテキストのサイズ制限。 |
| `CODEX_RULES_ENABLED_SOURCES=CONTEXT.md,.omo/rules` | 読み取るソースを制限。 |

レガシーな `PI_RULES_*` 変数は、`pi-rules` から移行するユーザー向けの fallback として受け付けられます。

## `$init-deep` との組み合わせ

`$init-deep` は階層型プロジェクトメモリを作成し、`rules` Skill は対応する rules と instruction ファイルを各セッション・各 prompt で Codex の前に保ちます。一度メモリを作れば、その後は `rules` が継続して注入します。
