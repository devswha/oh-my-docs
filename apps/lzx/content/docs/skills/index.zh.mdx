LazyCodex 会把 OmO 插件的 Skills 安装进 Codex。Commands（`$ulw-plan`、`$start-work`、`$ulw-loop`、`$init-deep`）负责工作流，Skills 则提供工作流中需要的专业判断。

Skills 会在 Codex session 中加载。用户可以直接要求“使用某个 Skill”，当任务内容匹配 Skill 描述时，Codex 也会跟随对应 playbook。

## Documented Skills

| Skill | 用途 |
| --- | --- |
| [`review-work`](/zh/docs/skills/review-work) | 多角度实现后评审 |
| [`remove-ai-slops`](/zh/docs/skills/remove-ai-slops) | 保持行为不变，清理 AI 味代码 |
| [`debugging`](/zh/docs/skills/debugging) | 系统化根因调试 |
| [`frontend`](/zh/docs/skills/frontend-ui-ux) | 设计过的 UI 工作，而不是普通填布局 |
| [`programming`](/zh/docs/skills/programming) | 严格 TypeScript、Rust、Python、Go 纪律 |
| [`refactor`](/zh/docs/skills/refactor) | 由测试验证的重构，包含 AST-grep 结构搜索与重写 |
| [`LSP`](/zh/docs/skills/lsp) | diagnostics、definitions、references、symbols、rename |
| [`rules`](/zh/docs/skills/rules) | 来自 AGENTS、rules、instruction 文件的项目指令 |
| [`comment-checker`](/zh/docs/skills/comment-checker) | 编辑类工作后的反馈 |

## Other Installed Skills

以 `4.12.1` 为准，插件还包含下面的 Skills。本站还没有为每个 Skill 单独建页，但它们可以在 Codex session 中加载。

| Skill | 用途 |
| --- | --- |
| `ast-grep` | AST-aware 结构搜索和重写 |
| `git-master` | commit、历史调查、rebase/squash/fixup 等 git 工作 |
| `lsp-setup` | language server 安装和配置 |
| `lcx-doctor` | LazyCodex/Codex 安装健康诊断 |
| `lcx-report-bug` | LazyCodex/Codex bug issue 编写 |
| `lcx-contribute-bug-fix` | 已验证 bug fix 贡献流程 |
| `teammode` | 像团队一样协调多个 Codex threads |
| `ultraresearch` | 明确要求的深度研究 |
| `visual-qa` | UI/TUI 变更后的截图和视觉验证 |

Commands 本身也作为 Skills 安装，但面向用户的文档把它们单独放在 [Commands](/zh/docs/commands)。

## Where Skills Fit

- Commands 拥有流程：规划、执行、验证完成。
- Skills 在流程周围增加专家判断：review、cleanup、UI work、语言纪律、code intelligence。
- Skills 只在相关任务中加载，用完就退出，让 agent prompt 保持较轻。
