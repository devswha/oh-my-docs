---
title: 스킬
description: LazyCodex가 명령 기둥 위에 올려 쓰는 전문가 플레이북입니다.
---

LazyCodex는 OmO(oh-my-openagent)를 Codex 에이전트 하니스로 패키징합니다. 명령 기둥(`$ulw-plan`, `$start-work`, `$ulw-loop`, `$init-deep`) 위에 OmO 플러그인이 여러 스킬을 설치합니다. 스킬은 재사용 가능한 지시문 묶음으로, 집중 워커를 도메인 전문가로 바꿔 줍니다.

스킬은 Codex 세션 안에서 호출됩니다. 규율 에이전트(Sisyphus, Hephaestus, Oracle, Librarian)가 작업과 도메인이 겹치는 스킬을 로드한 뒤 그 플레이북을 따릅니다.

## 설치되는 스킬

| 스킬 | 쓰임새 |
| --- | --- |
| [`review-work`](/ko/docs/skills/review-work) | 다각도 구현 후 리뷰 |
| [`remove-ai-slops`](/ko/docs/skills/remove-ai-slops) | 동작을 보존하면서 AI 티 나는 코드 정리 [^1] |
| [`debugging`](/ko/docs/skills/debugging) | 체계적인 근본 원인 디버깅 |
| [`refactor`](/ko/docs/skills/refactor) | 테스트로 검증하는 리팩터링. `ast_grep_search` / `ast_grep_replace`를 통한 구조적 검색·재작성 포함 |
| [`frontend-ui-ux`](/ko/docs/skills/frontend-ui-ux) | 일반적인 레이아웃 채우기 대신 디자인된 UI 작업 |
| [`programming`](/ko/docs/skills/programming) | 엄격한 TypeScript, Rust, Python, Go 규율 |
| [`LSP`](/ko/docs/skills/lsp) | 진단, 정의, 참조, 심볼, 이름 변경 |
| [`rules`](/ko/docs/skills/rules) | AGENTS, rules, 지시 파일에서 가져온 프로젝트 지시문 |
| [`comment-checker`](/ko/docs/skills/comment-checker) | 편집류 작업 후 피드백 |
| [`ultragoal`](/ko/docs/skills/ultragoal) | 목표를 고정한 증거 기반 완료 |

[^1]: `ai-slop-remover`는 `remove-ai-slops`가 병렬로 호출하는 단일 파일 워커이며, 별도의 사용자용 스킬이 아닙니다.

## 스킬이 들어가는 자리

- 명령 기둥이 루프를 가집니다: 계획, 실행, 검증된 완료.
- 스킬은 그 기둥 둘레에 전문가 판단을 더합니다: 리뷰, 정리, UI 장인 작업, 언어 규율, 코드 인텔리전스.
- 스킬은 해당 작업에 대해서만 로드되고 끝나면 내려가므로, 에이전트 프롬프트가 가볍게 유지됩니다.

## 스킬이 사는 곳

OmO는 `.opencode/skills`, `~/.config/opencode/skills`, `.claude/skills`, `.agents/skills`, `~/.agents/skills` 같은 프로젝트·사용자 위치에서 스킬을 로드할 수 있습니다.
