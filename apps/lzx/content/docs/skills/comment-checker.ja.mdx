`comment-checker` スキルは、LazyCodex（LZX）が書くコメントを正直に保ちます。編集の後、フックが触れたコメントをチェックし、問題があれば Codex にブロッキングのフィードバックを渡して、次へ進む前に直させます。LazyCodex が Codex にインストールする OmO（oh-my-openagent）プラグインに同梱されています。

## 一行要約

`comment-checker` は編集系操作の後に自動で動きます。警告を報告すると Codex はブロッキングのフィードバックを受け取り、フラグされたコメントを修正するか、その正しさを説明してから続行しなければなりません。呼び出すコマンドも MCP ツールもなく、ただ動きます。

## 動作方法

プラグインは成功した編集系呼び出しに `PostToolUse` フックを登録します。

- `apply_patch`
- `write`
- `edit`
- `multi_edit`
- `multiedit`

これらのいずれかの呼び出しの後にチェッカーがコメントをフラグすると、Codex はブロッキングのフィードバックを受け取ります。期待はシンプルです —— フラグされたコメントを直すか、なぜ正しいのかを説明してから次のステップへ進みます。

## スコープ

| 観点 | 挙動 |
|---|---|
| MCP ツール | 公開なし —— 呼び出すものはありません。 |
| 非編集ツール | このプラグインは無視します。 |
| チェッカーバイナリの欠落 | フック出力を出さず、通常の Codex 作業が中断なく続きます。 |

## なぜ重要か

コメントドリフト（下のコードと食い違うコメント）は、じわじわ蓄積する混乱の典型的な原因です。チェックはコメントを入れた当の編集で発火するため、フィードバックは変更がまだ新鮮なうちに届き、後のレビューに回りません。`programming` スキルの書いた後のレビューループと自然に組み合わさります。
