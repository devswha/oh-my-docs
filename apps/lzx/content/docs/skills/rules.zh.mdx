`rules` Skill 会把 LazyCodex (LZX) 固定到项目自己的指令上。当 Codex 中启用 OmO (oh-my-openagent) 插件时，项目规则会自动注入。你不需要每个会话都重新粘贴。Codex 会在工作时把受支持的 rules 和 instruction 文件放进上下文。

## 一句话总结

Codex Rules 是自动的。它会在每次会话开始和每次 prompt 时注入静态项目指令，并在编辑后注入匹配的文件级规则。没有需要运行的命令。启用插件后，仓库约定会在各处跟随 Codex。

## 注入对象

- `SessionStart` 和 `UserPromptSubmit` 时的**静态项目指令**。
- 默认情况下，Codex `apply_patch` 之后的**匹配文件级规则**。

动态 `PostToolUse` 输出会作为额外上下文注入，并按会话对插件数据去重。Codex Rules 不会改写工具输出，只是在旁边补充指令。

## 支持的项目来源

| 来源 | 说明 |
|---|---|
| `CONTEXT.md` | 通用项目上下文。 |
| `.omo/rules/**/*.md` | OmO/LazyCodex rules 树。 |
| `.claude/rules/**/*.md` | Claude 规则目录。 |
| `.cursor/rules/**/*.md` | Cursor 规则。 |
| `.github/instructions/**/*.md` | GitHub instruction 文件。 |
| `.github/copilot-instructions.md` | Copilot 指令。 |

## 配置

通过环境变量调整行为。

| 变量 | 效果 |
|---|---|
| `CODEX_RULES_DISABLED=1` | 禁用整个 Skill。 |
| `CODEX_RULES_MODE=both\|static\|dynamic\|off` | 选择 static、dynamic、both、off。 |
| `CODEX_RULES_MAX_RULE_CHARS=<number>` | 限制每条注入规则的大小。 |
| `CODEX_RULES_MAX_RESULT_CHARS=<number>` | 限制注入结果上下文的大小。 |
| `CODEX_RULES_ENABLED_SOURCES=CONTEXT.md,.omo/rules` | 限制读取的来源。 |

旧的 `PI_RULES_*` 变量仍作为从 `pi-rules` 迁移用户的 fallback 被接受。

## 与 `$init-deep` 配合

`$init-deep` 创建层级项目记忆，而 `rules` Skill 会在每个会话和 prompt 中持续把受支持的 rules 和 instruction 文件放到 Codex 面前。记忆创建一次之后，`rules` 会继续注入。
