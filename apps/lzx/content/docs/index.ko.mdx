LazyCodex의 역할은 단순합니다. Codex가 프로젝트 안에서 무엇을 기억하고, 어떤 순서로 일하고, 어디서 멈춰 확인해야 하는지 덜 헷갈리도록 OmO 설정을 연결합니다.

LazyCodex(이하 LZX)는 Codex용 OmO Light Edition입니다. [OmO(oh-my-openagent)](https://github.com/code-yeongyu/oh-my-openagent) 전체를 그대로 옮기는 도구가 아니라, Codex에서 쓰기 좋은 Hephaestus 중심 흐름과 명령, 스킬, 훅, 보조 도구를 묶어 줍니다.

설치 후에는 Codex가 프로젝트 메모리를 읽고, 먼저 계획을 세운 뒤, 일을 나눠 진행하고, 마지막에는 실제 증거로 결과를 확인하는 흐름을 쓸 수 있습니다. 같은 설정을 다시 만들 수 있어서, 한 번 맞춰 둔 작업 방식을 반복해서 쓰기 좋습니다.

> _Codex에 일하는 순서와 확인하는 습관을 붙여 주는 도구라고 보면 됩니다._

## 무엇을 얻나요

| 기능 | 쉽게 말하면 |
| --- | --- |
| **프로젝트 메모리** | Codex가 매번 처음부터 추측하지 않도록 `AGENTS.md`에 프로젝트 맥락을 남깁니다. |
| **계획 먼저** | 코드를 바로 바꾸기 전에 `$ulw-plan`으로 순서와 범위를 정합니다. |
| **계획 실행** | `$start-work`가 계획 체크리스트를 따라 작업을 진행합니다. |
| **증거로 확인** | `$ulw-loop`와 검증 흐름이 “끝난 것 같아요”가 아니라 실제 증거를 기준으로 봅니다. |
| **스킬과 훅** | 리뷰, 정리, 프론트엔드, 프로그래밍 같은 전문 흐름과 자동 확인 지점을 붙입니다. |

## 일단 이것만 기억하면 됩니다

처음엔 네 단계만 알면 됩니다.

1. `$init-deep`로 프로젝트 메모리를 만듭니다.
2. `$ulw-plan "무엇을 만들지"`로 작업 순서를 정합니다.
3. `$start-work`로 계획을 실행합니다.
4. 오래 걸리거나 꼼꼼히 확인해야 하는 일은 `$ulw-loop "작업"`으로 맡깁니다.

LazyCodex는 이 흐름에 규칙 로딩, 스킬, 훅, 모델 라우팅, 검증 습관을 붙여 줍니다. 자세한 구조는 사이드바 문서에서 필요한 항목만 하나씩 보면 됩니다.
