LazyCodex가 하는 일은 단순합니다. Codex가 프로젝트 안에서 무엇을 기억해야 하는지, 어떤 순서로 일해야 하는지 헷갈리지 않게 OmO 설정을 붙여 줍니다.

LazyCodex(이하 LZX)는 Codex용 OmO Light Edition입니다. [OmO(oh-my-openagent)](https://github.com/code-yeongyu/oh-my-openagent)를 일일이 만져서 맞추지 않아도, Codex에서 쓸 수 있는 명령, 스킬, 훅, 보조 도구를 바로 연결해 줍니다.

설치하고 나면 Codex가 프로젝트 메모리를 읽고, 먼저 계획을 세운 뒤, 일을 나눠 진행하고, 마지막에는 실제로 잘 됐는지 증거를 보고 확인하는 흐름을 쓸 수 있습니다. 필요할 때 같은 설정을 다시 만들 수 있게 되어 있어서, 한 번 맞춰 둔 작업 방식을 반복해서 쓰기 좋습니다.

> _Codex에게 일하는 순서와 확인하는 습관을 붙여 주는 도구라고 보면 됩니다._

## 그래서 뭐가 달라지나

| 기능 | 설명 |
| --- | --- |
| **Project Memory** | `rules`와 `AGENTS.md`를 통해 Codex가 프로젝트 규칙과 맥락을 기억하게 합니다. |
| **Planning** | `$ulw-plan`으로 바로 만들기 전에 작업 순서와 범위를 먼저 정합니다. |
| **Execution** | `$start-work`와 `start-work-continuation`으로 정한 계획을 이어서 실행합니다. |
| **Verified Completion** | `ulw-loop`와 `ultrawork` 흐름으로 결과가 증거로 확인될 때까지 점검합니다. |
| **Codex Light Components** | `comment-checker`, `git-bash`, `lsp`, `telemetry` 같은 Codex용 OmO 구성요소가 붙습니다. |
| **MCP & Search Helpers** | `grep_app`, `context7`, `git_bash`, `lsp`, `ast-grep` 같은 보조 도구로 코드와 문서를 더 잘 찾습니다. |

## 일단 이것만 기억하면 됨

명령어가 많아 보여도 처음엔 딱 네 가지만 기억하면 됩니다. 프로젝트를 읽고, 계획하고, 실행하고, 끝났는지 확인하는 순서입니다.

| 커맨드 | 하는 일 |
| --- | --- |
| `$init-deep` | 프로젝트를 훑어보고, Codex가 기억해 둘 내용을 `AGENTS.md`로 정리합니다. |
| `$ulw-plan` | 바로 만들기 전에 먼저 작업 계획을 세웁니다. 결과는 `plans/<slug>.md`에 남습니다. |
| `$start-work` | 정한 계획대로 일을 시작합니다. 체크박스와 확인 절차를 따라 진행합니다. |
| `$ulw-loop` | 목표를 정해 두고, 증거로 확인될 때까지 실행과 점검을 반복합니다. |

처음 프로젝트에 붙일 때는 `$init-deep`를 한 번 돌려주면 됩니다. 그다음 `$ulw-plan "무엇을 만들지"`로 접근 방식을 정하고, 계획이 괜찮다면 `$start-work`로 넘깁니다. 오래 걸리거나 꼼꼼히 확인해야 하는 일은 `$ulw-loop "작업"`으로 맡기는 편이 속 편합니다.

설치 직후 프로젝트를 읽는 단계부터 마지막 확인까지, 네 가지 명령이 이어집니다.
