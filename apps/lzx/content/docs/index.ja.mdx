LazyCodex の役割はシンプルです。Codex がプロジェクト内で何を記憶し、どの順序で作業し、どこで止まって確認すべきかを迷いにくくするために、OmO 設定を接続します。

LazyCodex（LZX）は Codex 向けの OmO Light Edition です。[OmO（oh-my-openagent）](https://github.com/code-yeongyu/oh-my-openagent) 全体をそのまま移すものではなく、Codex で使いやすい Hephaestus 中心の流れ、コマンド、スキル、フック、補助ツールをまとめます。

インストール後は、Codex がプロジェクトメモリを読み、先に計画を立て、作業を段階に分けて進め、最後に実際の証拠で結果を確認する流れを使えます。同じ設定を再作成できるため、一度整えた作業方法を繰り返し使いやすくなります。

> _Codex に作業の順序と確認する習慣を付けるための道具です。_

## 何が得られるか

| 機能 | 簡単に言うと |
| --- | --- |
| **プロジェクトメモリ** | Codex が毎回最初から推測しなくてよいように、`AGENTS.md` にプロジェクトの文脈を残します。 |
| **先に計画** | コードをすぐ変更する前に、`$ulw-plan` で順序と範囲を決めます。 |
| **計画の実行** | `$start-work` が計画チェックリストに沿って作業を進めます。 |
| **証拠で確認** | `$ulw-loop` と検証フローが、「終わった気がする」ではなく実際の証拠を基準に見ます。 |
| **スキルとフック** | review、cleanup、frontend、programming などの専門フローと自動確認ポイントを追加します。 |

## まず覚えること

最初は 4 つの手順だけで十分です。

1. `$init-deep` でプロジェクトメモリを作ります。
2. `$ulw-plan "何を作るか"` で作業順序を決めます。
3. `$start-work` で計画を実行します。
4. 長い作業や細かい確認が必要な作業は `$ulw-loop "タスク"` に任せます。

LazyCodex はこの流れに、ルール読み込み、スキル、フック、モデルルーティング、検証の習慣を追加します。詳しい構造は、サイドバーの文書から必要な項目だけ確認してください。
