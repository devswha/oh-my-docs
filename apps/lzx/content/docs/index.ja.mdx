**LazyCodex（LZX）** は、複雑なコードベース向けの Codex エージェントハーネスとして [OmO（oh-my-openagent）](https://github.com/code-yeongyu/oh-my-openagent) をパッケージ化したものです。lazy.nvim に対する LazyVim のようなもの、ただし Codex 向け、と考えてください。

OmO は Codex に完全なエージェントハーネスを提供します。規律エージェント、並列実行、マルチモデルルーティング、スキルシステム、フックです。LazyCodex はそのハーネスを、一行の `npx` で導入できる再現可能な Codex セットアップとしてパッケージ化します。グローバルインストールは不要です。

> _「LazyVim は Neovim を一般の人にも使えるものにしました。LazyCodex は Codex に同じことをします。」_

## 得られるもの

| 機能 | 説明 |
| --- | --- |
| **規律エージェント** | Sisyphus が Hephaestus、Oracle、Librarian をオーケストレーションします。完全な AI 開発チームです。 |
| **並列実行** | 複数のエージェントが独立したサブタスクを同時に処理します。 |
| **マルチモデルルーティング** | タスクカテゴリごとに最適なモデルを自動選択します。 |
| **スキルシステム** | 専門作業のための拡張可能なスキルライブラリです。 |
| **フックとライフサイクル** | すべてのエージェント動作に対する前後フックを提供します。 |
| **ゼロ設定** | 妥当なデフォルトをそのまま提供し、必要に応じて上書きできます。 |

## コマンドの柱

LazyCodex は Codex 向けに 3 つのコマンドの柱とプロジェクトメモリ用コマンドを導入します。柱コマンドはインストーラーが表示する `$command` 構文で呼び出します。

| コマンド | 機能 |
| --- | --- |
| `$init-deep` | 階層型の `AGENTS.md` プロジェクトメモリを生成します。 |
| `$ulw-plan` | Prometheus 戦略プランナー。計画を `plans/<slug>.md` に書き出し、プロダクトコードは決して書きません。 |
| `$start-work` | すべてのチェックボックスが完了するまで計画を実行し、**ORCHESTRATION COMPLETE** を出力します。 |
| `$ulw-loop` | Oracle が検証する完了に達するまで実行される自己参照ループです。 |

典型的な流れは次のとおりです。プロジェクトメモリのために `$init-deep` を一度実行し、`$ulw-plan "何を作るか"` で方針を決め、`$start-work` で実行します。結果が証拠で検証されるまで作業を進め続けたい場合は `$ulw-loop "タスク"` を使います。

## なぜ異なる GPT モデルが現れるのか

OmO はすべてのサブタスクに最強モデルを使うわけではありません。タスクカテゴリを定義し、各カテゴリを最適なモデルへルーティングします。`quick` は `gpt-5.4-mini` のような小型モデルへ、`ultrabrain` は難しいロジックのために高推論 GPT モデルへ、エージェント型コーディング経路は Codex 調整済み GPT モデルを使います。要点はクォータ管理です。深い推論には強力なモデルを、それで十分なときはより速いモデルを使います。
