LazyCodex は一行の `npx` コマンドで OmO ベースの Codex light セットアップを導入します。グローバルインストールは不要です。前提条件を確認し、初回起動でフックを承認し、`doctor` で確認します。

<Steps>

<Step>

### 前提条件

- [Node.js](https://nodejs.org) — メンテナンス中の LTS。`npx` が同梱されています。Bun は不要です。
- [OpenAI Codex](https://github.com/openai/codex)：**Codex アプリ**（推奨）または **Codex CLI**

LazyCodex は OpenAI Codex の*内部*で動作するため、先に Codex をインストールしてサインインしておく必要があります。使いたい環境を選んでください：

<Callout title="作者は Codex アプリを推奨します">
どちらでも動きますが、Codex アプリの方が使い勝手がいいです。迷ったらアプリを入れてください。
</Callout>

<Tabs items={["Codex アプリ（推奨）", "Codex CLI"]}>

<Tab value="Codex アプリ（推奨）">

**OpenAI Codex アプリ**をインストールし、サインインしてプロジェクトを開きます。LazyCodex は Codex 内で動作するため、インストールもその後の `$command` 実行もアプリ内で完結します。下の**インストール**へ進んでください。

</Tab>

<Tab value="Codex CLI">

[OpenAI Codex CLI](https://github.com/openai/codex) をインストールし、起動してサインインできる状態にしておきます。LazyCodex は Codex 内で動作するため、まず CLI が正常に動く必要があります。下の**インストール**へ進んでください。

</Tab>

</Tabs>

</Step>

<Step>

### インストール

一行で済みます。グローバルインストールや `npm i -g` は不要で、常に `npx` を使います。

```bash
npx lazycodex-ai install
```

これは Codex プラットフォームを対象とする OmO インストーラーの短縮形です。

```bash
npx --yes --package oh-my-openagent omo install --platform=codex
```

完全自律・TUI なしのセットアップには、自律モードのフラグを追加します。

```bash
npx lazycodex-ai install --no-tui --codex-autonomous
```

インストーラーは Codex light セットアップ（コマンド、スキル、フック、モデルルーティング、検証デフォルト）をあなたの Codex セットアップに組み込みます。インストールとセットアップは LLM エージェントに任せるのがお勧めです。サブスクリプション検出、モデル選択、プロバイダー認証を自動でやってくれます。

インストール後の次回 Codex 起動時に、スタートアップレビューが omo フックの承認を求めます。フックは承認前には実行されません。

自分でコマンドを打つのが面倒なら、Codex（CLI またはアプリ）を開いて [lazycodex.ai](https://lazycodex.ai) のリンクを渡し、LazyCodex をインストールしてほしいと頼むだけです。エージェントがドキュメントを読んで、セットアップ全体を代行します。

<Step>

### 認証

認証するのは **Codex 本体**であって、LazyCodex ではありません。**Codex CLI** ではサインインは Codex 自身のセットアップの一部です（初回起動時に案内されます）。**Codex アプリ** では UI からサインインします。LazyCodex 専用のログインコマンドはありません。

Codex にサインインしていれば、`npx lazycodex-ai install` がサブスクリプション検出・モデル選択・プロバイダールーティングを自動で処理するため、追加で認証することはありません。LLM エージェントにインストールを任せれば、このあたりも全部代行してもらえます。

セットアップ状態を確認するには次を実行します。

```bash
npx lazycodex-ai doctor
```

`doctor` は何が設定済みで、何が不足し、なぜ問題なのかを説明します。

プロバイダーとルーティングの詳細は[設定](/ja/docs/reference/configuration)を参照してください。
