LazyCodex는 `npx` 명령 한 줄로 OmO 기반 Codex light 설정을 설치합니다. 전역 설치는 필요 없습니다. Node.js와 Codex를 먼저 준비하고, 첫 실행 때 훅을 승인한 뒤 `doctor`로 확인합니다.

<Steps>

<Step>

### 사전 준비

- [Node.js](https://nodejs.org) — 유지보수 중인 LTS 버전이면 됩니다. `npx`가 함께 제공되며 Bun은 필요하지 않습니다.
- [OpenAI Codex](https://github.com/openai/codex): **Codex 앱**(권장) 또는 **Codex CLI**

LazyCodex는 OpenAI Codex *안에서* 실행됩니다. 그래서 Codex가 설치되어 있고 로그인까지 끝나 있어야 합니다. 작업할 환경은 둘 중 하나를 고르면 됩니다.

<Callout title="저자는 Codex 앱을 권장합니다">
둘 다 쓸 수 있습니다. 그래도 처음 설치한다면 Codex 앱이 더 편합니다. 화면에서 프로젝트를 열고 바로 `$command`를 실행할 수 있어서 덜 헷갈립니다.
</Callout>

**OpenAI Codex 앱**을 설치하고 로그인한 뒤 프로젝트를 엽니다. LazyCodex는 Codex 안에서 돌아가므로, 앱을 쓰면 설치부터 이후 모든 `$command` 실행까지 GUI에서 처리할 수 있습니다. 준비가 끝났으면 아래 **설치**로 넘어가세요.

### 설치

명령은 이것 하나입니다. 전역 설치나 `npm i -g`는 쓰지 않습니다.

```bash
npx lazycodex-ai install
```

아래 OmO 설치 명령의 단축 형태입니다.

```bash
npx --yes --package oh-my-openagent omo install --platform=codex
```

TUI를 띄우지 않고 설치를 맡기려면 자율 모드 플래그를 붙입니다.

```bash
npx lazycodex-ai install --no-tui --codex-autonomous
```

설치 프로그램은 Codex light 설정의 커맨드, 스킬, 훅, 모델 라우팅, 검증 기본값을 Codex 설정에 연결합니다. 직접 만지기보다 LLM 에이전트에게 설치를 맡기는 쪽이 편합니다. 구독 감지, 모델 선택, 제공자 인증까지 에이전트가 이어서 처리합니다.

설치 후 다음 Codex 실행에서 시작 리뷰가 omo 훅 승인을 요청합니다. 훅은 승인 전에는 실행되지 않습니다.

설치 과정이 번거롭게 느껴지면 명령을 직접 실행하지 않아도 됩니다. Codex CLI나 앱을 열고 [lazycodex.ai](https://lazycodex.ai) 링크를 준 뒤 LazyCodex를 설치해 달라고 요청하세요. 에이전트가 문서를 읽고 설정을 진행합니다.

### 인증

인증 대상은 LazyCodex가 아니라 **Codex 자체**입니다. **Codex CLI**는 Codex 셋업 과정에서 로그인하며, 첫 실행 때 로그인 안내가 나옵니다. **Codex 앱**은 앱 UI에서 로그인합니다. LazyCodex용 로그인 명령은 따로 없습니다.

Codex에 로그인돼 있다면 `npx lazycodex-ai install`이 구독 감지, 모델 선택, 제공자 라우팅을 처리합니다. LLM 에이전트에게 설치를 맡긴 경우에도 같은 과정을 대신 진행합니다.

설정 상태를 확인하려면 다음을 실행하세요.

```bash
npx lazycodex-ai doctor
```

`doctor`는 무엇이 설정되었고, 무엇이 빠졌고, 왜 문제가 되는지 설명합니다.

제공자와 라우팅 설정은 [설정](/ko/docs/reference/configuration)을 참고하세요.
