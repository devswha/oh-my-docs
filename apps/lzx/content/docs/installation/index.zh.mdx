LazyCodex 用一行 `npx` 命令把 OmO 智能体框架装进 Codex，不需要全局安装。前置条件装好、安装跑完、登录 Codex，就能用了。

<Steps>

<Step>

### 前置条件

- [Bun](https://bun.sh)
- [OpenAI Codex](https://github.com/openai/codex)：**Codex 应用**（推荐）或 **Codex CLI**

LazyCodex 跑在 OpenAI Codex *内部*，所以 Codex 必须装好且已登录。选你想用的界面：

<Callout title="作者推荐使用 Codex 应用">
两者都行，但 Codex 应用体验更顺。拿不定主意就装应用。
</Callout>

<Tabs items={["Codex 应用（推荐）", "Codex CLI"]}>

<Tab value="Codex 应用（推荐）">

装好 **OpenAI Codex 应用**，登录后用它打开项目。LazyCodex 跑在 Codex 里，应用全程有 GUI，安装和后续的每个 `$command` 都能在界面里操作。继续下面的**安装**。

</Tab>

<Tab value="Codex CLI">

装好 [OpenAI Codex CLI](https://github.com/openai/codex)，确认能启动、已登录。LazyCodex 跑在 Codex 里，CLI 得先通才行。继续下面的**安装**。

</Tab>

</Tabs>

</Step>

<Step>

### 安装

一行搞定。别用 `npm i -g`，始终用 `npx`：

```bash
npx lazycodex-ai install
```

它是 Codex 平台 OmO 安装程序的简写：

```bash
npx --yes --package oh-my-openagent omo install --platform=codex
```

需要完全无 TUI 的自主配置，加上这个标志：

```bash
npx lazycodex-ai install --no-tui --codex-autonomous
```

安装程序把 OmO 框架（命令、技能、钩子、模型路由）接入你的 Codex 配置。推荐让 LLM 智能体直接跑安装——订阅检测、模型选择、提供方认证，全自动处理。

懒得手动跑的话，也可以这样：打开 Codex（CLI 或应用），把 [lazycodex.ai](https://lazycodex.ai) 链接丢给它，让它自己装。文档它会看，配置它会做。

<Step>

### 认证

认证的对象是 **Codex 本身**，不是 LazyCodex。用 **Codex CLI** 的话，登录在 Codex 首次启动时就会提示；用 **Codex 应用** 的话，在应用 UI 里登录。没有单独的 LazyCodex 登录命令。

Codex 登录好了，`npx lazycodex-ai install` 会自动处理 OmO 这边（订阅检测、模型选择、提供方路由），不需要额外认证。让智能体跑安装也一样，流程相同。

提供方与路由细节见[配置](/zh/docs/reference/configuration)。
