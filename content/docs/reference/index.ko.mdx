## 패키지와 이름 호환성

upstream rename은 전환 중입니다. 설치는 upstream에 현재 배포된 명령을 우선하고, OpenAgent 이름은 product/public identity로 이해하면 안전합니다.

| 이름 | 용도 |
|---|---|
| `oh-my-opencode` | 현재 upstream npm package와 primary installer command. |
| `oh-my-openagent` | rename transition 중 제공되는 compatibility binary/product name. |
| `oh-my-openagent` plugin entry | installer가 등록할 수 있을 때 선호하는 OpenCode plugin entry. |
| `oh-my-opencode` plugin entry | 전환 기간 동안 계속 인식되는 legacy entry. |

## CLI

| 명령 | 용도 |
|---|---|
| `bunx oh-my-opencode install` | 대화형 installer. |
| `bunx oh-my-opencode install --no-tui ...` | agent-driven install에 적합한 비대화형 setup. |
| `bunx oh-my-opencode doctor` | 시스템, 설정, 도구, 모델 해석 검증. |
| `bunx oh-my-opencode run <message>` | 완료 강제 기능이 포함된 OpenCode session runner. |
| `bunx oh-my-opencode version` | 버전 정보 표시. |
| `bunx oh-my-opencode mcp oauth ...` | remote MCP server용 OAuth helper. |

## 설정 파일 위치

사용자 설정을 먼저 읽고, 프로젝트 설정이 덮어쓸 수 있습니다. JSONC를 지원합니다.

| 범위 | 경로 |
|---|---|
| 프로젝트 | `.opencode/oh-my-openagent.json[c]`, `.opencode/oh-my-opencode.json[c]` |
| 사용자 | `~/.config/opencode/oh-my-openagent.json[c]`, `~/.config/opencode/oh-my-opencode.json[c]` |

## 빠른 설정 예시

```jsonc
{
  "$schema": "https://raw.githubusercontent.com/code-yeongyu/oh-my-openagent/dev/assets/oh-my-opencode.schema.json",
  "agents": {
    "oracle": { "model": "openai/gpt-5.4", "variant": "high" },
    "explore": { "model": "github-copilot/grok-code-fast-1" }
  },
  "categories": {
    "quick": { "model": "opencode/gpt-5-nano" },
    "visual-engineering": { "model": "google/gemini-3.1-pro" }
  }
}
```

설치된 package가 renamed schema를 노출한다면 `doctor`가 알려주는 경로를 우선하세요.

## Agent 옵션

| 옵션 | 설명 |
|---|---|
| `model` | `provider/model` 형식의 명시적 override. |
| `variant` | `max`, `high`, `medium`, `low`, `xhigh` 같은 변형. |
| `fallback_models` | model/provider 실패 시 사용하는 순서 있는 fallback chain. |
| `prompt`, `prompt_append` | agent prompt 교체/추가. `file://` URI를 지원합니다. |
| `tools` | agent가 사용할 수 있는 도구 제한. |
| `disable` | 특정 agent 비활성화. |
| `permission` | edit, bash, webfetch 같은 권한 제어. |

## Category

| Category | 목적 |
|---|---|
| `visual-engineering` | Frontend, UI/UX, design, styling, animation. |
| `ultrabrain` | Hard logic, architecture, deep reasoning. |
| `deep` | Autonomous research + end-to-end implementation. |
| `artistry` | 창의적/비정형 작업. |
| `quick` | 단순 변경, typo, 단일 파일 수정. |
| `unspecified-low`, `unspecified-high` | 일반 작업의 저/고노력 fallback. |
| `writing` | 문서와 prose. |

## Install Flags

| Flag | 의미 |
|---|---|
| `--claude=no\|yes\|max20` | Claude provider mode. |
| `--openai=no\|yes` | OpenAI/ChatGPT provider mode. |
| `--gemini=no\|yes` | Gemini provider mode. |
| `--copilot=no\|yes` | GitHub Copilot fallback provider. |
| `--opencode-go=no\|yes` | OpenCode Go subscription path. |
| `--opencode-zen=no\|yes` | `opencode/` model catalog path. |
| `--zai-coding-plan=no\|yes` | Z.ai Coding Plan path. |
| `--kimi-for-coding=no\|yes` | Kimi for Coding path. |
| `--vercel-ai-gateway=no\|yes` | Vercel AI Gateway path. |
| `--skip-auth` | 설치 후 auth 안내 생략. |

## 환경 변수

| 변수 | 의미 |
|---|---|
| `OMO_SEND_ANONYMOUS_TELEMETRY=0` | 익명 활성 설치 telemetry 비활성화. |
| `OMO_DISABLE_POSTHOG=1` | PostHog telemetry 비활성화. |
| `OPENCODE_CONFIG_DIR` | OpenCode 설정 디렉토리 override. |
