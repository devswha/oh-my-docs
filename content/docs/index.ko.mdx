**OpenCode를 위한 멀티 에이전트 오케스트레이션. 목표만 말하면 됩니다.**

_OpenCode 사용법을 더 외우지 마세요. OmO가 흐름을 잡습니다._

Oh My OpenAgent(OmO)는 OpenCode 위에서 동작하는 에이전트 레이어입니다. 사용자가 원하는 결과를 말하면, OmO가 의도를 읽고 계획, 위임, 실행, 검증까지 이어갑니다.

작은 일은 평소처럼 요청하면 됩니다. 복잡한 일은 `ulw` 또는 `ultrawork`로 시작하세요. 정확한 설계가 먼저 필요하면 Prometheus로 계획을 만들고 `/start-work`로 실행합니다.

> **이름 안내:** 제품 이름은 **Oh My OpenAgent**지만, upstream npm 패키지는 아직 `oh-my-opencode`로 배포됩니다. 설치 문서에서는 이 전환 상태를 그대로 적습니다.

```bash
bunx oh-my-opencode install
```

## 왜 OmO인가

- **의도 먼저** — 설명인지, 조사인지, 구현인지 먼저 가릅니다.
- **에이전트 분업** — Sisyphus, Hephaestus, Prometheus, Atlas, Oracle 등이 역할을 나눕니다.
- **카테고리 라우팅** — 모델명을 외우는 대신 `visual-engineering`, `deep`, `writing` 같은 작업 성격으로 보냅니다.
- **스킬 주입** — `playwright`, `git-master`, project skill이 필요한 순간에만 들어옵니다.
- **실제 검증** — 브라우저, tmux, API 요청, 빌드처럼 사용자가 만지는 표면으로 확인합니다.
- **계속 밀기** — todo, background task, Ralph/ulw loop가 남은 일을 다시 이어갑니다.
