**Oh My OpenAgent(OmO)** 는 OpenCode 위에 올라가는 멀티 모델 에이전트 하네스입니다. 단순한 프롬프트 모음이 아니라 의도 파악, 계획, 위임, 실행, 검증, 세션 연속성을 한 루프로 묶습니다.

OmO의 전제는 명확합니다. 사람은 원하는 결과를 말하고, 하네스가 방법을 찾아야 합니다. 작은 일은 평소처럼 요청하면 되고, 복잡한 일은 `ulw` 또는 `ultrawork`라고 쓰면 됩니다. 정확도가 중요한 일은 Prometheus로 계획을 만든 뒤 `/start-work`로 실행합니다.

현재 배포 패키지와 바이너리 이름은 여전히 `oh-my-opencode`입니다. OpenCode plugin entry는 `oh-my-openagent`를 선호하지만, 전환 기간에는 legacy entry와 config 파일명도 계속 인식합니다.

```bash
bunx oh-my-opencode install
```

## OmO가 더하는 것

| 층 | 에이전트에게 주는 것 |
|---|---|
| Intent Gate | 답변, 조사, 구현, 계획, 위임 중 사용자의 진짜 의도를 먼저 읽습니다. |
| Discipline agents | Sisyphus, Hephaestus, Prometheus, Atlas, Oracle, Librarian, Explore, Metis, Momus, Sisyphus-Junior가 서로 다른 일을 맡습니다. |
| Category routing | 모델명을 직접 고르지 않고 `visual-engineering`, `ultrabrain`, `deep`, `quick`, `writing` 같은 작업 성격으로 위임합니다. |
| Skills | `playwright`, `git-master`, `frontend-ui-ux`, `review-work`, 프로젝트 스킬이 필요한 순간에만 지식과 MCP를 주입합니다. |
| Agent-grade tools | LSP, AST-grep, tmux, 브라우저 QA, MCP 검색, session tools, hash-anchored edit를 제공합니다. |
| Continuation hooks | Todo continuation, Ralph/ulw loop, session recovery, runtime fallback, reminder가 중단된 작업을 다시 밀어 올립니다. |
