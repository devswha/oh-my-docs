## 도구 경계

이 제약은 하네스의 일부입니다. "똑똑한" 에이전트가 잘못된 역할에서 위험한 일을 하는 걸 막습니다.

| Agent | 제약 |
|---|---|
| Oracle | read-only consultation. write, edit, delegation 없음. |
| Librarian | research-only. write, edit, delegation 없음. |
| Explore | search-only. write, edit, delegation 없음. |
| Multimodal-Looker | media analysis 중심, 읽기 위주. |
| Atlas | 계획 실행을 오케스트레이션하지만 구현 단위는 위임. |
| Sisyphus-Junior | 한 단위 구현 후 돌아가기 전 검증 필수. |

## Category Worker

`task`에 `category`가 들어오면, OmO가 그 도메인에 맞춘 Sisyphus-Junior 워커를 띄웁니다. category가 모델 프로필을 정하고, skill이 지시와 MCP를 더합니다.

| Category | 워커 동작 |
|---|---|
| `visual-engineering` | Frontend/UI/design. `frontend-ui-ux`와 브라우저 검증을 같이 씁니다. |
| `ultrabrain` | hard reasoning과 architecture. 프롬프트는 목표 중심으로 유지하세요. |
| `deep` | 자율 조사 + 구현. 복잡하지만 일관된 작업에 좋습니다. |
| `artistry` | 창의적이거나 비관습적인 접근. |
| `quick` | 작은 변경. diff가 작다고 시각 작업을 여기로 보내지 마세요. |
| `writing` | 문서와 prose. |

## Background Agents

메인 세션이 결과 없이도 진행 가능하면 specialist를 background로 돌리세요.

```ts
task({
  subagent_type: "explore",
  load_skills: [],
  run_in_background: true,
  prompt: "Find all auth policy implementations and summarize file paths.",
});
```

독립적인 조사면 background, 다음 구현 결정을 막는 결과면 동기 모드로 기다리세요.
