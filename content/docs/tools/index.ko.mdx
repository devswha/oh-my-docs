## 도구 스택

OmO에서 도구는 편의 기능이 아니라 증거 표면입니다. 완료를 말하기 전에 실제 사용자가 만지는 표면으로 확인해야 합니다.

| 레이어 | 도구 | 증명하는 것 |
|---|---|---|
| 코드 지능 | LSP diagnostics, definitions, references, workspace rename | 전체 build 전에 type/symbol 문제가 없는지. |
| 구조 검색 | AST-Grep search/replace | 문자열 검색보다 안전한 code-pattern 변경. |
| 파일 안전성 | Hashline-aware read/edit | 공유 worktree에서 읽은 뒤 파일이 바뀌지 않았는지. |
| Runtime surface | tmux, browser, HTTP request, CLI driver | 사용자가 실제로 쓰는 표면에서 기능이 동작하는지. |
| 외부 지식 | Context7, web search, GitHub code search, Librarian | 최신 문서, OSS 예제, remote repo 사실. |
| Continuation | todos, background task notification, boulder state | 긴 작업이 조용히 멈추지 않고 이어지는지. |

## LSP와 AST-Grep

| 도구 | 쓰임 |
|---|---|
| `lsp_diagnostics` | 변경 파일의 language-server error/warning을 build 전에 확인합니다. |
| `lsp_goto_definition` | 참조한 symbol의 정의 위치를 확인합니다. |
| `lsp_find_references` | 동작을 바꾸기 전에 call site를 찾습니다. |
| `lsp_rename` | `lsp_prepare_rename` 뒤 프로젝트 전체 rename을 적용합니다. |
| `ast_grep_search` | 함수 호출이나 component shape 같은 완전한 AST pattern을 찾습니다. |
| `ast_grep_replace` | capture한 변수를 보존하며 AST node를 바꿉니다. |

Symbol 질문에는 LSP, code shape 질문에는 AST-Grep을 우선합니다.

## 수동 QA 표면

| 산출물 | 필요한 표면 |
|---|---|
| Web UI | browser automation: 페이지 열기, click/fill, console 확인, 시각 결과 확인. |
| CLI/TUI | tmux: `--help`, happy path, 잘못된 입력 하나 이상. |
| HTTP API | live process와 `curl` 또는 작은 driver. |
| Library/SDK | 변경 경로를 import하고 end-to-end 실행하는 최소 script. |
| 문서 사이트 | build와 변경 route smoke check. |

“소스상 될 것 같다”는 완료 증거가 아닙니다.

## MCP와 검색

| Source | 적합한 일 |
|---|---|
| Context7 | 공식 package docs와 API 예시. |
| Web search | 최신 공개 사실, release, incident, page. |
| GitHub code search | 공개 repo의 실제 사용 pattern. |
| Librarian | Multi-repo 또는 외부 library 종합. |

## Hook과 Continuation

OmO에는 많은 lifecycle hook이 있지만, 대부분 workflow는 아래 보장에 의존합니다.

| 계열 | 목적 |
|---|---|
| Session recovery | tool result 누락, empty content, malformed session state 복구. |
| Todo continuation | pending todo가 있으면 계속 진행. |
| Background notification | delegated work 완료/실패를 main agent에 알림. |
| Interactive shell | CLI/TUI를 one-shot shell이 아니라 tmux에서 유지. |
| Directory agents | directory-scoped instruction과 convention 주입. |
