## 시작하기

OmO를 처음 쓴다면 아래 순서대로 가면 됩니다.

1. [설치](#설치) - upstream 패키지와 provider 선택
2. [인증](#provider-인증) - OpenCode auth로 provider 연결
3. [검증](#검증) - `doctor`로 설정 확인
4. [첫 작업](#첫-작업) - `ulw`로 한 번 돌려보기

### 사전 요구사항

| 항목 | 요구사항 |
|---|---|
| OpenCode | `opencode` 설치 필요. upstream guide는 `1.0.150` 이상을 기준으로 합니다. |
| Runtime | bootstrap에는 `bunx` 또는 `npx`가 가장 편합니다. |
| Provider | Claude, OpenAI/ChatGPT, Gemini, Copilot, OpenCode Go/Zen, Kimi, Z.ai, Vercel AI Gateway 중 실제로 쓸 것. |
| OS | macOS, Linux, Windows용 binary가 배포됩니다. |

## 설치

현재 upstream npm 패키지는 `oh-my-opencode`입니다.

```bash
bunx oh-my-opencode install
```

전환기 호환 binary도 노출될 수 있습니다.

```bash
bunx oh-my-openagent install
```

installer는 OpenCode plugin entry, OmO config, provider 인증 안내를 한 번에 처리합니다.

## 비대화형 설치

에이전트에게 설치를 맡길 때는 `--no-tui`를 쓰세요.

```bash
bunx oh-my-opencode install --no-tui \
  --claude=yes \
  --openai=yes \
  --gemini=no \
  --copilot=no
```

주요 flag는 아래만 기억하면 됩니다.

| Flag | 의미 |
|---|---|
| `--claude=no\|yes\|max20` | Claude 구독 모드. `max20`은 Claude Max 20x. |
| `--openai=no\|yes` | OpenAI/ChatGPT provider. Oracle 계열에 자주 씁니다. |
| `--gemini=no\|yes` | Gemini provider. |
| `--copilot=no\|yes` | GitHub Copilot fallback. |
| `--opencode-go=no\|yes` | OpenCode Go 구독 경로. |
| `--opencode-zen=no\|yes` | `opencode/` model catalog 경로. |
| `--zai-coding-plan=no\|yes` | Z.ai Coding Plan 경로. |
| `--kimi-for-coding=no\|yes` | Kimi for Coding 경로. |
| `--vercel-ai-gateway=no\|yes` | Vercel AI Gateway 경로. |
| `--skip-auth` | config만 쓰고 인증 안내는 건너뜁니다. |

OmO는 전역 provider priority 하나로 움직이지 않습니다. agent와 category마다 fallback chain이 다릅니다.

## Provider 인증

설치 후 provider를 로그인합니다.

```bash
opencode auth login
```

Gemini Antigravity OAuth는 현재 `opencode-antigravity-auth` plugin 설정을 따릅니다. Copilot, Z.ai, OpenCode Go/Zen, Kimi는 installer flag에 맞춰 기본 model이 잡힙니다.

## 검증

```bash
opencode --version
bunx oh-my-opencode doctor
```

`doctor`가 보는 항목은 네 가지입니다.

| 그룹 | 확인 내용 |
|---|---|
| System | OpenCode binary, version, plugin 등록, legacy package warning |
| Config | JSONC parsing, schema validation, config file detection |
| Tools | AST-Grep, LSP server, GitHub CLI, MCP server |
| Models | Capability cache, model normalization, override, fallback availability |

Provider나 model이 이상하면 `doctor --verbose`로 한 번 더 봅니다.

## 첫 작업

설치가 끝났으면 바로 작은 일을 맡겨보세요.

```text
ulw fix the failing login flow and verify it in the browser
```

설계를 먼저 잠그고 싶다면:

```text
@plan refactor the auth module without behavior changes
/start-work
```

깊은 GPT식 추론이 필요하면 agent selector에서 Hephaestus를 고르고, 절차가 아니라 목표를 주세요.

## Telemetry

익명 active-install telemetry는 기본값으로 켜져 있습니다. UTC 하루 한 번 이하로 hashed installation id만 보냅니다. 끄려면 둘 중 하나를 설정하세요.

```bash
OMO_SEND_ANONYMOUS_TELEMETRY=0
OMO_DISABLE_POSTHOG=1
```
