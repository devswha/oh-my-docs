## 준비물

| 항목 | 요구사항 |
|---|---|
| OpenCode | `opencode`가 설치되어 있어야 합니다. upstream guide는 OpenCode `1.0.150` 이상을 기준으로 합니다. |
| Runtime | 설치 후 CLI는 주요 플랫폼용 standalone binary를 사용하지만, bootstrap은 `bunx` 또는 `npx`가 가장 쉽습니다. |
| Providers | Claude, OpenAI/ChatGPT, Gemini, GitHub Copilot, OpenCode Go/Zen, Kimi for Coding, Z.ai, Vercel AI Gateway 중 실제로 쓰는 것. |
| OS | macOS ARM64/x64, Linux x64/ARM64 및 musl/Alpine, Windows x64 binary가 배포됩니다. |

## 설치

upstream 패키지는 아직 `oh-my-opencode`로 배포됩니다.

```bash
bunx oh-my-opencode install
```

호환 binary도 사용할 수 있습니다.

```bash
bunx oh-my-openagent install
```

installer는 가능하면 OpenCode plugin entry를 `oh-my-openagent`로 등록하고, OmO config 파일을 만들며, 선택한 provider에 필요한 인증 단계를 출력합니다.

## 비대화형 설치

에이전트가 설치를 대신할 때는 `--no-tui`를 쓰는 편이 안전합니다.

```bash
bunx oh-my-opencode install --no-tui \
  --claude=yes \
  --openai=yes \
  --gemini=no \
  --copilot=no
```

주요 provider flag:

| Flag | 의미 |
|---|---|
| `--claude=no\|yes\|max20` | Claude 구독 모드. `max20`은 Claude Max 20x입니다. |
| `--openai=no\|yes` | Oracle 같은 GPT 기반 agent를 위한 OpenAI / ChatGPT 구독. |
| `--gemini=no\|yes` | Gemini 통합. |
| `--copilot=no\|yes` | GitHub Copilot fallback provider. |
| `--opencode-go=no\|yes` | GLM, Kimi, MiniMax를 제공하는 OpenCode Go. |
| `--opencode-zen=no\|yes` | `opencode/` model catalog 접근. |
| `--zai-coding-plan=no\|yes` | 주로 GLM fallback을 제공하는 Z.ai Coding Plan. |
| `--kimi-for-coding=no\|yes` | Kimi for Coding 구독. |
| `--vercel-ai-gateway=no\|yes` | Vercel AI Gateway provider 경로. |
| `--skip-auth` | config만 쓰고 인증 안내를 건너뜁니다. |

Provider 선택은 agent별입니다. OmO는 하나의 전역 provider priority를 쓰지 않고, agent와 category마다 다른 fallback chain을 사용합니다.

## Provider 인증

설치 후 활성화한 provider는 OpenCode auth에서 인증합니다.

```bash
opencode auth login
```

Gemini Antigravity OAuth는 현재 `opencode-antigravity-auth` plugin과 해당 README의 model config가 필요합니다. GitHub Copilot, Z.ai, OpenCode Go, OpenCode Zen, Kimi는 installer가 flag에 맞춰 기본 model을 설정합니다.

## 검증

```bash
opencode --version
bunx oh-my-opencode doctor
```

`doctor`는 system, config, tools, model resolution을 확인합니다.

| 그룹 | 예시 |
|---|---|
| System | OpenCode binary, version, plugin 등록, legacy package warning. |
| Config | JSONC parsing, schema validation, config file detection. |
| Tools | AST-Grep, LSP server, GitHub CLI, MCP server. |
| Models | Capability cache, model normalization, agent/category override, fallback availability. |

Provider/model resolution이 이상하면 `doctor --verbose`를 사용하세요.

## 첫 작업

자율 실행:

```text
ulw fix the failing login flow and verify it in the browser
```

정확한 다단계 작업:

```text
@plan refactor the auth module without behavior changes
/start-work
```

GPT-native deep work가 필요하면 agent selector에서 Hephaestus로 전환하고, 절차가 아니라 목표를 주세요.

## Telemetry

익명 active-install telemetry는 기본 활성화입니다. UTC 하루에 한 번 이하로 hashed installation id를 보내며 PostHog person profile은 만들지 않습니다. 끄려면 둘 중 하나를 설정합니다.

```bash
OMO_SEND_ANONYMOUS_TELEMETRY=0
OMO_DISABLE_POSTHOG=1
```
