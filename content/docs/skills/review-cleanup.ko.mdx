리뷰 skill은 구현이 끝난 뒤에 돕니다. 위험과 빠진 증거를 잡아내고, 워커를 구체적인 수정으로 되돌려 보냅니다.

## 리뷰 레인

| Skill | 확인하는 것 |
|---|---|
| `review-work` | 목표 부합, 코드 품질, 보안, QA 증거, 맥락 채굴. |
| `ai-slop-remover` | 불필요한 추상화, 중복 코드, 시끄러운 주석 같은 AI-generated 냄새. |
| `pre-publish-review` | 변경별 deep review, holistic review, Oracle synthesis로 본 릴리스 준비도. |
| `github-triage` | 증거 기반 read-only issue/PR triage. |
| `work-with-pr` | worktree setup부터 CI/review loop와 merge까지 PR lifecycle. |

## 원칙

리뷰 skill은 구현 다음에 돌리는 것이지 테스트의 대체가 아닙니다. 구현자가 놓친 것을 표면화할 뿐, 작업 자체를 대신하지는 않습니다.
